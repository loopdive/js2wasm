ECMAScript to WebAssembly

Compile ordinary JavaScript into small Wasm modules.

js2wasm is an ahead-of-time compiler for JavaScript and TypeScript that targets WebAssembly GC. The goal is to keep the source language familiar, keep the output small, and move execution into a capability-limited Wasm module instead of shipping a full JavaScript runtime inside every artifact. A further goal is to bring real npm package compatibility into the Wasm ecosystem, so existing JavaScript modules can move into Wasm hosts without being rewritten into a different language or packaging model.

Why

Small modules Target Wasm GC directly without embedding a JavaScript engine per module.
Sandboxed host access Filesystem, network, DOM, and globals stay behind explicit imports instead of ambient access.
Real browser tooling Inspect WAT, benchmark Wasm against JS, browse test262 progress, and explore example programs.

Compiler Model

js2wasm parses and checks JavaScript or TypeScript, lowers supported semantics into typed Wasm operations, and leaves only explicit host boundaries as imports.

  • Closed-world optimization for code the compiler can fully see
  • Host effects isolated at imports and exports
  • Readable WAT output for debugging and inspection

Current Surface

The project already supports numeric code, control flow, classes, arrays, strings, modules, browser DOM access, and a growing subset of ECMAScript semantics.

  • Interactive playground with real Wasm execution
  • Benchmark suite comparing Wasm to JavaScript
  • test262 tracking and sprint-based issue planning
