# Development Method

This folder collects the repo's working method and operating conventions.

Contents:

- `team-setup.md` — current agent/team operating model
- `team.md` — older team/spec process doc kept for history
- `definition-of-ready.md` — issue entry criteria
- `definition-of-done.md` — issue exit criteria
- `pre-commit-checklist.md` — local pre-commit gate
- `pre-completion-checklist.md` — developer completion gate
- `pre-merge-checklist.md` — merge gate
- `session-start-checklist.md` — tech-lead session bootstrap
- `velocity-template.md` — sprint velocity section template
- `file-locks.md` — active file-claim coordination table
- `agent-sessions.md` — session registry conventions

These are method/process docs. They are distinct from:

- `plan/issues/` — issue records and sprint records
- `plan/log/` — archived historical artifacts, investigations, and reviews
- `plan/goals/` — product and compiler goals
- `plan/design/` — design decisions, architecture notes, and specifications
