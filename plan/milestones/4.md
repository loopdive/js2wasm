# Milestone 4: Pure Wasm Architecture

**Goal**: Eliminate JS host imports to enable standalone Wasm execution (WASI, Component Model). This is the project's core architectural principle.

**Target**: Remove 15+ JS host imports. Enable standalone WASI execution for real programs.

**Estimated pass rate improvement**: Indirect — enables WASI deployment rather than test count improvement.

## Rationale

The project's mission is "never delegate to JS host." Generators and iterators are the two biggest remaining host-import clusters. Fixing them enables:

1. **WASI target works for real programs** — currently generators crash under WASI
2. **Component Model** — clean exports with no JS glue needed
3. **Edge deployment** — Fastly, Fermyon, Shopify Functions

#680 (generators) is XL but the phased approach makes it manageable:
- Phase 1 (simple yields) covers 60% of cases
- Phase 2 (yields in loops) covers 85%
- Phase 3 (yield*, return, throw) covers 95%

## Success criteria

- Generator-using programs run under WASI without JS runtime
- Iterator protocol uses WasmGC structs, no host imports
- Component Model WIT generation produces valid interfaces
- At least one edge platform deployment demonstrated (Fastly or Fermyon)

- #639 depends on stable function exports
