# Milestone 1: Crash & Compile Error Elimination

**Goal**: Maximize compilable + non-crashing test count. Fix the issues that block the most tests from even running.

**Target**: Reduce CEs from 4,443 → <2,000. Reduce null-pointer traps by 1,600.

**Estimated pass rate improvement**: +3,000–4,000 tests (30% → 38%)

## Issues (priority order)

| # | Title | Type | Impact | Complexity |
|---|-------|------|--------|------------|
| **759** | Insert missing extern.convert_any at call sites | CE | 1,299 CE | Easy |
| **728** | Null pointer dereference → TypeError, not trap | FAIL | 1,604 FAIL | Medium |
| **765** | Compile error triage (umbrella) | CE | 4,443 CE | Small+spawns |
| **730** | Missing exception paths (tests expect throws) | FAIL | 708 FAIL | Large |
| **736** | SyntaxError detection at compile time | FAIL | 316 FAIL | Medium |

## Rationale

These are the highest-leverage fixes. #759 is marked easy and eliminates 29% of all compile errors with a mechanical fix (insert `extern.convert_any` coercion). #728 is the single highest-impact runtime fix. #765 triages the remaining CEs to identify the next batch of mechanical fixes.

## Success criteria

- CE count < 2,000
- No more than 500 tests crash with Wasm traps (down from ~1,600)
- All CE patterns with >100 occurrences have dedicated sub-issues

## Dependencies

None — all issues are independently startable.
