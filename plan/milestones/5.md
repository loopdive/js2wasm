# Milestone 5: Built-in Completeness & Test Coverage

**Goal**: Fill in missing built-in method support and close equivalence test coverage gaps. Move from "compiles most code" to "runs most code correctly."

**Target**: Pass rate from ~46% toward 55%+. All implemented features have equivalence tests.

**Estimated pass rate improvement**: +3,000–4,000 tests (46% → 55%)

## Issues (priority order)

| # | Title | Type | Impact | Complexity |
|---|-------|------|--------|------------|
| **763** | RegExp runtime methods (exec, match, replace, split) | FAIL | ~400 FAIL | Large |
| **764** | Promise chain methods (.then, .catch, .finally) | FAIL | ~300 FAIL | Large |
| **696** | Classify "other fail" runtime errors (4,649) | Triage | 4,649 FAIL | Small+spawns |
| **767** | Equivalence test coverage gaps | Tests | Quality gate | Medium |
| **687** | Live-streaming test262 report with run selector | Infra | DX | High |

## Rationale

After Milestones 1–4 fix crashes, core language, iterators, and architecture, the remaining failures are in built-in method edge cases. #696 is the key triage issue — 4,649 failures need classification to identify the next round of targeted fixes.

#767 (test coverage) ensures we don't regress. Currently RegExp, Promise, and async features have no equivalence tests, meaning regressions go undetected.

## Success criteria

- `RegExp.prototype.exec()` and string regex methods work
- Promise `.then()/.catch()/.finally()` chains resolve correctly
- 4,649 "other fail" errors classified into actionable sub-issues
- Equivalence tests exist for all major feature areas
- Test262 pass rate exceeds 55%

## Dependencies

- #763 benefits from #682 (RegExp Wasm engine) but can use host imports initially
- #764 may need event loop / microtask support
- #696 is independent triage work
- #767 is independent
