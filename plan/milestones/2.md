# Milestone 2: Core Language Correctness

**Goal**: Fix the most common runtime assertion failures. These tests compile and run but produce wrong values.

**Target**: Fix ~3,000 assertion failures across class features, built-in methods, and property semantics.

**Estimated pass rate improvement**: +2,500–3,000 tests (38% → 44%)

## Issues (priority order)

| # | Title | Type | Impact | Complexity |
|---|-------|------|--------|------------|
| **729** | Class feature codegen gaps | FAIL | 1,161 FAIL | Hard |
| **731** | Function/class .name property | FAIL | 558 FAIL | Easy |
| **732** | hasOwnProperty correctness | FAIL | 520 FAIL | Medium |
| **733** | RangeError validation in built-ins | FAIL | 442 FAIL | Medium |
| **734** | Array method correctness (edge cases) | FAIL | 343 FAIL | Medium |

| **737** | Undefined-handling edge cases | FAIL | 276 FAIL | Medium |

## Rationale

After Milestone 1 removes crashes and compile errors, these are the next biggest failure buckets. #731 (Function.name) is marked easy — it's a property that just needs to be populated. #729 (class features) is the biggest single item but can be broken into sub-tasks (field initializers, static blocks, computed keys).

## Success criteria

- Class field initializers and static blocks compile correctly
- Function.name returns the correct value for all declaration forms
- hasOwnProperty/Object.defineProperty match spec behavior
- Array methods handle edge cases (sparse arrays, negative indices, type coercion)

## Dependencies

- #729 depends on #678 (prototype chain) — should be done by now
- Rest are independent
