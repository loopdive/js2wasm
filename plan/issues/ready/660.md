# #660 — Capture first failing assert in test output for returned-0 tests

## Status: open

12,974 tests fail with "returned 0" but no information about WHICH assertion failed. The wrapTest function wraps the body in try/catch and sets __fail=1, but doesn't record the failing assert.

### Fix
In the wrapTest assert shims, record the first failure message:

```typescript
let __fail_msg: string = "";
function assert_sameValue(a: any, b: any, msg?: string): void {
  if (a !== b) {
    __fail = 1;
    if (!__fail_msg) __fail_msg = msg || "sameValue failed";
  }
}
```

Then in the test function return, encode the message into the return value or store it in a global that the runner can read.

Simplest approach: instead of `return 0`, return a negative number that encodes which assert failed (e.g., -1 for first assert, -2 for second, etc.). The runner reads the return value and maps it back to the assert.
