# #361 -- Runtime `in` operator for property checks

## Status: open

21 tests use the `in` operator to check property existence at runtime. Need to check struct fields at runtime.

## Details

The `in` operator checks if a property exists on an object:
```javascript
const obj = { a: 1, b: 2 };
'a' in obj; // true
'c' in obj; // false
0 in [10, 20]; // true (array index check)
```

Implementation challenges:
1. Struct-based objects have fixed fields known at compile time -- need a runtime check mechanism
2. For string keys, need to compare the key against known field names
3. For array indices, check bounds
4. Symbol keys (e.g., `Symbol.iterator in obj`) need symbol support

Approach: emit a sequence of string comparisons against known field names for the object's type, returning true if any match.
