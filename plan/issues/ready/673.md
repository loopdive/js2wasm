# #673 — Reflect API support

## Status: open

~391 tests use Reflect methods.

### Approach
Compile each Reflect method to its equivalent operation:
- `Reflect.get(obj, prop)` → `obj[prop]` (property access)
- `Reflect.set(obj, prop, val)` → `obj[prop] = val`
- `Reflect.has(obj, prop)` → `prop in obj`
- `Reflect.ownKeys(obj)` → `Object.keys(obj)` (already implemented)
- `Reflect.apply(fn, this, args)` → `fn.apply(this, args)`
- `Reflect.construct(C, args)` → `new C(...args)`
- `Reflect.defineProperty` → `Object.defineProperty`
- `Reflect.getPrototypeOf` → `Object.getPrototypeOf`

Most are direct rewrites to patterns we already compile. Add `Reflect` to lib types and rewrite calls at compile time.

## Complexity: M
