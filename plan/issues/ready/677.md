# Full property descriptor support

## Problem

Object.defineProperty with writable/configurable flags had no effect -- all properties were always writable.
Object.getOwnPropertyDescriptor returned undefined (null extern) -- test262 tests couldn't query descriptor attributes.
~1,000 test262 tests depend on property descriptor semantics.

## Approach

Added a per-struct i32 `__descriptors` bitfield field. 2 bits per field:
- Bit `2*i` = writable
- Bit `2*i+1` = configurable
- Default value: -1 (all bits set = all writable + configurable)

Supports up to 16 fields per struct (i32 has 32 bits / 2 bits per field).

## Implementation

### index.ts
- Added `descriptorFieldIdx: Map<string, number>` and `propDescTypeIdx: number` to CodegenContext
- Added `__descriptors` i32 field to all user structs: anonymous, class (root+child), interface, widened, and Object type structs
- For class structs: `__descriptors` goes right after `__tag`; child classes inherit from parent
- Default init value: `i32.const -1` (all writable+configurable)
- Registered `__PropDesc` struct type lazily via `ensurePropDescType()`

### expressions.ts
- `compileObjectDefineProperty`: parses `writable`, `configurable`, `enumerable` boolean literals from descriptor and emits bitfield manipulation (AND to clear, OR to set)
- `compilePropertyAssignment`: checks writable bit before `struct.set`; silently ignores assignment if non-writable
- `compilePropertyCompoundAssignment`: same writable check for `+=`, `-=`, etc.
- `compileGetOwnPropertyDescriptor`: reads field value and descriptor bits, constructs a `__PropDesc` struct with value/writable/enumerable/configurable
- All struct.new emission points updated to emit `-1` for `__descriptors` (including `compileWidenedEmptyObject`, `Object.create` path)

## Files changed
- `src/codegen/index.ts`
- `src/codegen/expressions.ts`
- `tests/equivalence/property-descriptors.test.ts` (new)
