# #362 -- typeof on member expressions

## Status: open

16 tests use typeof on member expressions (e.g., `typeof obj.prop`). Currently typeof only works on identifiers.

## Details

```javascript
var obj = { fn: function() {} };
typeof obj.fn; // "function"
typeof obj.missing; // "undefined"
```

Currently, the typeof compilation only handles simple identifiers. It needs to be extended to:
1. Compile the member expression to get its value
2. Apply the same type-tag logic used for identifiers
3. Handle the case where the property doesn't exist (return "undefined")

Since the compiler knows the type of struct fields at compile time, this can often be resolved statically.
