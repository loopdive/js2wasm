# #652 — Borrow checker / ownership semantics for TypeScript (research)

## Status: open (low priority — after JS compatibility)

## Feasibility Analysis

A full Rust-equivalent borrow checker for TypeScript is **not realistic**. The languages are too different (TS has structural types, no lifetime annotations, erasable types). But the *performance benefits* of ownership — eliminating GC pauses, stack allocation — are achievable through simpler mechanisms.

### Why WasmGC changes the equation
WasmGC structs are GC-managed by definition. There's no way to allocate a WasmGC struct outside the GC heap. To get actual stack allocation, we'd need linear memory alongside WasmGC — two memory models. Because GC handles safety, a borrow checker here is purely an **optimization**, not a correctness requirement. A partial checker that misses some cases just means those values fall back to GC.

### Recommended phased approach

**Phase 0 — Escape analysis (free, no annotations)**
During codegen, if a struct is only used locally and doesn't escape the function, emit locals instead of a heap allocation. No user-facing changes needed. This is a standard compiler optimization.

**Phase 1 — Arena/region API (moderate)**
Define an `Arena` type; the compiler emits bump-pointer allocations in linear memory and resets at scope exit:
```typescript
const arena = new Arena();
const points = arena.alloc<Point[]>(1000); // linear memory, no GC
// ... use points ...
// arena auto-freed at scope exit
```
Simpler analysis than ownership: just prove the arena doesn't escape its scope.

**Phase 2 — Branded ownership types (large)**
Use `Owned<T>`, `Borrowed<T>` brands in the TS type system. A pre-codegen analysis pass checks ownership rules for annotated values. Restricted subset — not full Rust semantics.

**Phase 3 — Lifetime annotations (speculative)**
May never be worth doing. TypeScript's type system would need extensions.

### Prior art
- AssemblyScript, Kotlin/Wasm, Dart/Wasm: all use GC, no borrow checker
- Only Rust and C++ (unique_ptr) have compile-time ownership
- Linear Haskell has linear types but no borrow checker

## Complexity: L (Phase 0: M, Phase 1: L, Phase 2: XL)
