# #416 -- Compound assignment on element access (non-ref targets)

## Status: ready

11 tests fail when compound assignment operators (`+=`, `-=`, `*=`, etc.) are applied to element access expressions like `arr[i] += value`. The compiler cannot resolve the target type for the read-modify-write sequence.

## Root cause

`compileCompoundAssignment` handles property access (`obj.prop += val`) but does not fully support element access (`obj[expr] += val`). When the target is an array element or a dynamically-keyed property, the compiler fails to emit the correct get-modify-set instruction sequence.

Related to #404 (done) which fixed compound assignment on unresolvable property types, but this issue is specific to bracket-notation element access targets.
