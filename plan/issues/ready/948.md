# #948 — Systematic WAT analysis of all passing equivalence tests

## Problem

Manual WAT inspection of the calendar example (#947) found 6 codegen inefficiencies. These likely exist across all compiled output. A systematic analysis of ALL passing equivalence tests would identify:

1. **Universal patterns** — inefficiencies that appear in every compiled module
2. **Frequency data** — which patterns are most common (prioritize fixes by impact)
3. **Binary size impact** — how many bytes each pattern adds per module
4. **Performance impact** — which patterns affect hot loops vs cold paths

## What to do

### Phase 1: Build analysis corpus
1. Compile all ~1,155 passing equivalence tests to WAT
2. Parse the WAT to extract:
   - Function count, local count, instruction count per function
   - Duplicate local declarations (same name, different type)
   - Dead instruction sequences (local.set + drop, local.tee + drop)
   - Infinity guards on modulo with constant operands
   - String concat chains (consecutive concat calls)
   - If-chain patterns that could be br_table
   - TDZ flag locals that are never checked at runtime
   - f64↔i32 conversion roundtrips

### Phase 2: Generate report
Output `benchmarks/results/wat-analysis-report.json` with:
```json
{
  "corpus_size": 1155,
  "patterns": {
    "duplicate_locals": { "count": N, "affected_functions": N, "example": "..." },
    "dead_drops": { "count": N, "bytes_wasted": N },
    "infinity_guard_on_constant": { "count": N },
    "string_concat_chain": { "count": N, "max_chain_length": N },
    "if_chain_br_table_candidate": { "count": N },
    "tdz_flag_unused": { "count": N },
    "f64_i32_roundtrip": { "count": N }
  }
}
```

### Phase 3: Create targeted issues
For each pattern with significant frequency, create or update an issue with:
- Exact count across the corpus
- Binary size impact estimate
- Sample WAT before/after the fix
- Which codegen file to modify

## Acceptance criteria

- All passing equiv tests compiled and analyzed
- Report generated with pattern frequencies
- Top 5 patterns have issues created with fix proposals
- No manual WAT reading required — fully automated analysis

## Implementation

Create `scripts/analyze-wat-patterns.ts` that:
1. Iterates equivalence test sources
2. Compiles each to WAT via `compile(source, { emitWat: true })`
3. Parses WAT text with regex/string matching (no full WAT parser needed)
4. Aggregates pattern counts
5. Writes JSON report
