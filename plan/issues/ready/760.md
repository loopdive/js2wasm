# #760 -- Monomorphize functions with call-site types

## Problem

Functions are compiled with generic externref parameters when the compiler doesn't know the argument type. At many call sites, the exact struct type IS known — forcing unnecessary extern.convert_any / any.convert_extern round-trips.

## Approach

1. Pre-pass: scan all call sites, collect concrete argument types per function
2. Specialize: if all call sites pass same struct type, type parameter as struct ref directly
3. Monomorphize: if 2-3 different types, compile one variant per type combination
4. Generic fallback: only use externref for exported/callback/highly-polymorphic functions

## Acceptance criteria

- Functions called with known struct types use struct ref parameters directly
- Net improvement in test262 pass count and CE count
- No regressions in equivalence tests
