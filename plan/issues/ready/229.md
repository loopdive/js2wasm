# Issue #229: Tagged template cache: array out of bounds on top-level repeated calls

## Status: in-progress

## Summary

1 test (`cache-same-site-top-level.js`) fails with `RuntimeError: array element access out of bounds`. The test calls a tag function in a loop at the top level and checks that the template object is cached (same reference across calls). 5 other tagged template cache tests fail with `returned 0` (wrong caching behavior).

## Root Cause

Sprint 2 (#212) added per-call-site global caching for tagged templates. The top-level case may have an off-by-one in the global cache array index, or the cache array is not properly sized. The `returned 0` failures in the other 5 cache tests suggest the caching comparison itself is not working -- the template objects are not being recognized as identical across calls.

## Scope

- `src/codegen/expressions.ts` -- tagged template caching logic
- Tests affected: 6 tagged template cache tests

## Expected Impact

Fixes 6 runtime failures.

## Suggested Approach

1. Fix the array bounds issue in the top-level cache by ensuring the cache array is sized correctly
2. For the `returned 0` failures, the issue is likely that `assert.sameValue(templates[0], templates[1])` compares refs and the cached template object is being recreated instead of reused
3. Verify that the global cache lookup correctly returns the previously stored template array ref on subsequent calls to the same call site

## Acceptance Criteria

- [ ] No array out of bounds on top-level tagged template calls
- [ ] Template objects are reference-identical across calls to the same site
- [ ] All 6 tagged template cache tests pass
