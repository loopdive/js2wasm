# #646 — Residual undefined .kind: more AST node handlers (5,329 CE)

## Status: review

## Root Cause

The "Cannot read properties of undefined (reading 'kind')" crash was caused by
a brace nesting error in `compilePropertyAccess` (expressions.ts). The
`accessWasm` fallback code (which emits default values for f64/i32/externref
property access) was incorrectly nested inside an `if (typeName && !ctx.classSet.has(typeName))`
block. When `typeName` was falsy (e.g., for IArguments type from `arguments.length`,
unknown types, or any objects without a resolved struct name), the entire fallback
was skipped and the function returned `undefined` instead of a valid ValType.

This `undefined` result was then accessed as `result.kind` in `compileExpression`
line 508, causing the crash that was caught by the try/catch and reported as
"Internal error compiling expression/statement: Cannot read properties of
undefined (reading 'kind')".

## Fix

1. **Brace fix**: Added closing `}` to properly close the `if (typeName)` block
   before the `accessWasm` fallback code, and removed the duplicate `}` at the
   end of the function.

2. **Safety guard**: Added a guard in `compileExpression` after the try/catch to
   detect when `compileExpressionInner` returns a non-null value without a valid
   `.kind` property, and emit a default value instead of crashing.
