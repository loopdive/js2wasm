# Issue #238: Class expression new -- `new (class { ... })()` pattern

## Status: in-progress

## Summary

158 tests fail with "Unsupported new expression for class: __class". These tests use inline class expressions in `new` expressions, like `new (class { constructor() { ... } })()`. The class compiles but the `new` expression does not recognize the inline class as a valid constructor target.

## Root Cause

The `new` expression codegen checks for known class names but does not handle anonymous/inline class expressions. When TypeScript types the class expression as `typeof __class`, the codegen does not find it in the class registry.

## Scope

- `src/codegen/expressions.ts` -- NewExpression handling
- Tests affected: ~158 compile errors

## Expected Impact

Fixing inline class expression instantiation could resolve ~100 compile errors (the cases where this is the sole error; ~58 combine with other errors).

## Suggested Approach

1. In NewExpression codegen, when the constructor expression is a ClassExpression:
   - Compile the class expression first (creating the struct type and constructor)
   - Use the resulting class info to emit the `new` call
2. Handle the case where the class expression is wrapped in parentheses: `new (class C { ... })()`
3. Ensure constructor arguments are properly passed

## Acceptance Criteria

- [ ] `new (class { ... })()` compiles and instantiates correctly
- [ ] Named inline class expressions work: `new (class C { ... })()`
- [ ] At least 80 compile errors resolved
- [ ] No regression in existing class/new tests
