# #391 -- Numeric index signature on object types (30 CE)

## Status: open

30 tests fail with "Type '{}' has no matching index signature for type 'number'" because the compiler does not handle numeric index access on plain object types.

## Details

Some JavaScript patterns use numeric indices on non-array objects:

```javascript
var obj = {0: "a", 1: "b", length: 2};
var val = obj[0];  // numeric index on object
```

The compiler currently requires an explicit index signature or array type for numeric access.

Fix:
1. For object literal types with numeric property names, allow numeric index access
2. For array-like objects (with `length` and numeric keys), support bracket access
3. Consider coercing numeric index to string for property lookup on structs

## Complexity: S

- [ ] `obj[0]` works when obj has numeric property names
