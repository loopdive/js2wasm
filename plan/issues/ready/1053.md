# #1053 — arguments.length reflects formal parameter count instead of runtime args length

## Real Root Cause (dev-1053, 2026-04-11)

The harvester-reported hypothesis ("trailing comma double-counted") is **wrong**. Trailing comma is incidental: all failing tests in the cluster happen to use `f(42, 'a',)` syntax, but the underlying bug is orthogonal to the parser.

**Actual bug**: `emitArgumentsObject` in `src/codegen/statements/nested-declarations.ts:713` and `src/codegen/function-body.ts:283` builds the `arguments` vec struct by iterating the **callee's formal parameter list**, not the actual runtime args passed at the call site. In every failing test, the method is declared `method()` (zero formal params) and called with two runtime arguments, so we emit `arguments.length === 0`, the first assertion fails, and `__fail = 2` is returned.

**Reproduced on 6 variants**, all FAIL with the same root cause:
- `cls-decl-meth-args-trailing-comma-multiple.js`
- `cls-decl-gen-meth-args-trailing-comma-multiple.js`
- `cls-decl-private-meth-args-trailing-comma-multiple.js`
- `meth-args-trailing-comma-multiple.js`
- `gen-meth-args-trailing-comma-multiple.js`
- `func-decl-args-trailing-comma-multiple.js`

Extra confirmation: `method(a)` called with `method(42, 'TC39')` returns `arguments.length === 1`, not 2. `method(a, b)` called with `method(42, 'TC39')` correctly returns 2 — so the breakage is strictly "runtime args > formal params".

## Fix (Option B: synthetic trailing rest-vec param)

Append a synthetic `vec_externref` Wasm parameter to any function/method whose body uses `arguments`. Caller builds a vec of extra args (beyond formal count) and passes it as the last Wasm argument. `emitArgumentsObject` reads formal params + extras vec to construct `arguments`.

- Metadata: track `usesArguments: boolean` per funcMap entry (callers consult it).
- Call sites: in `calls.ts`, when the callee has the synthetic param, emit an extras vec before calling.
- Callee: adjust `emitArgumentsObject` to concat formal params with the extras vec contents.
- Narrow scope: the synthetic param is emitted **only** when `bodyUsesArguments === true`.

## Original harvester evidence (kept for audit trail)

Class methods (gen, async, static, private) called with trailing-comma argument lists (e.g. `f(42, 'a',)`) observe the wrong `arguments.length` and `arguments[k]` values. The trailing comma should be ignored and not count as an extra argument.

## Problem

Class methods (gen, async, static, private) called with trailing-comma argument lists (e.g. `f(42, 'a',)`) observe the wrong `arguments.length` and `arguments[k]` values. The trailing comma should be ignored and not count as an extra argument.

## Root cause hypothesis

The call-site argument-count plumbing incorrectly propagates a phantom slot for the trailing comma, or the method prologue reads the pre-trim argument-count from a frame location that has not accounted for the trailing comma elision.

## Fix

Audit call-site argument-list lowering to drop trailing-comma sentinels before populating the `arguments` object. Ensure `arguments.length` is derived from the actual argument count, not the raw parse-tree slot count.
