# #816 -- String/prototype compilation hang (1,200+ tests skipped)

## Problem
Multiple String.prototype method tests cause infinite compilation loops in the wrapped test262 context. Patterns: lastIndexOf, isWellFormed, indexOf (wrapped-values), and others. The valueOf re-entrancy guard in type-coercion.ts fixes some but not all loops.

## Root cause (suspected)
The compilation loop is NOT in coerceType valueOf coercion. It may be in:
- compileCallExpression → property dispatch for String methods
- compilePropertyAccess when the receiver is a boxed string wrapper
- The test262 wrapper creates String objects (Object("foo")) that trigger a different compilation path than string primitives

## Investigation approach
1. Add a compilation call tracer (log function entry/exit) to find the loop
2. Compile a hanging test with the tracer enabled
3. The trace will show which functions are calling each other in the loop
4. Add a re-entrancy guard at the right level

## Affected tests
- String/prototype/lastIndexOf/S15.5.4.8_*
- String/prototype/isWellFormed/*
- String/prototype/indexOf/searchstring-tostring-wrapped-values.js
- Likely more String/prototype tests
