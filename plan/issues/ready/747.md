# #747 -- Object.defineProperty throws TypeError on null property access (~1,931 tests)

## Problem

1,931 test262 tests fail with TypeError because `Object.defineProperty` and related property descriptor operations hit null when accessing properties on objects that don't have the expected struct shape.

Current implementation is a no-op stub that returns the object. Tests that rely on actual property descriptor behavior (writable, configurable, enumerable checks) fail when subsequent code tries to read descriptor properties.

## Fix approach

Implement `Object.defineProperty` as a host import that actually sets properties, or compile it to struct field mutations when the target type is known at compile time.

## Acceptance criteria

- Object.defineProperty sets properties on objects
- Property descriptors (writable, configurable, enumerable) are respected where possible
- ~1,931 tests improve
