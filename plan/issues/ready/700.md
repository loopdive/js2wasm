# #700 — Reuse ts.CompilerHost across compilations (~25% speedup)

## Status: open

Each `compile()` call creates a new `ts.createProgram` which re-parses lib.d.ts (~50ms). In the compiler pool, workers compile thousands of tests sequentially — keeping the host warm saves ~50ms per compile.

### Profile
- Full compile: 241ms
- With emitWat:false + sourceMap:false: 190ms (current)
- With cached host (estimated): ~140ms (-25%)

### Approach
1. Add `reuseHost?: ts.CompilerHost` to CompileOptions
2. In compileSource(), if reuseHost provided, use it instead of creating new
3. The host caches: lib file contents, default lib path, directory structure
4. In compiler-worker.mjs: create host once, pass to every compile()
