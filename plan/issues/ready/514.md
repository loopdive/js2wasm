# #514 -- Generator/async-gen "options is not defined" (~684 CE)

## Status: ready

### Root cause

The error "options is not defined" is a runtime error during compilation (not a Wasm validation error). The compiler's internal `options` variable is not in scope when processing certain code paths -- likely in generator function compilation, async generator methods, or when processing certain class method patterns.

The `options` variable in `src/codegen/index.ts` (line ~368) is passed to `compile()` / `compileFile()` and used for configuration. When the compiler recursively processes nested functions (generators, async generators), the `options` reference may be lost.

### Fix approach

1. Trace the code path that produces "options is not defined"
2. Ensure the `options` variable is accessible in all nested compilation contexts
3. If it's a FunctionContext issue, propagate options through the context chain
