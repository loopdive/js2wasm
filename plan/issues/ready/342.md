# Issue #342: Array.prototype.method.call/apply patterns

Test262 uses patterns like `Array.prototype.forEach.call(arrayLike, fn)`. The compiler needs to recognize `.call()` dispatch on array methods and rewrite them to direct method calls.

## Implementation Summary

The core implementation was already in place via two paths in `src/codegen/expressions.ts`:

1. **`compileArrayPrototypeCall` (line ~21130)** -- Detects `Array.prototype.METHOD.call(obj, ...args)` pattern, resolves array info from shape map or TypeScript types, and either dispatches to dedicated inline implementations (indexOf, includes, every, some, forEach) or creates a synthetic `obj.METHOD(args)` AST node and routes through `compileArrayMethodCall`.

2. **Case 2a fallback (line ~10265)** -- Generic `Type.prototype.method.call(receiver, ...args)` rewrite that handles Array, String, Number, Boolean by creating a synthetic call expression.

### What was done
- Added comprehensive tests for `forEach.call`, `every.call`, `some.call`, `includes.call`, `push.call`, and `slice.call` patterns
- All tests verify parity between `Array.prototype.X.call(arr, ...)` and `arr.X(...)`
- 21 total tests, all passing
