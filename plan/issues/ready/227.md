# Issue #227: BigInt comparison with Infinity causes float-unrepresentable trap

## Status: review

## Summary

4 tests fail with `RuntimeError: float unrepresentable in integer range` when comparing BigInt values with `Infinity` or `-Infinity`. Tests like `bigint-and-non-finite.js` compare `0n > -Infinity` which should return `true`, but the i64-to-f64 conversion traps when encountering non-finite floats.

## Root Cause

BigInt is compiled as i64. When comparing BigInt with Number, the codegen converts f64 to i64 via `i64.trunc_f64_s`, which traps on Infinity/NaN. The correct approach is to convert the i64 to f64 first (`f64.convert_i64_s`) and then do f64 comparison, since any finite f64 comparison with i64-range values is representable.

## Scope

- `src/codegen/expressions.ts` -- BigInt comparison paths
- Tests affected: 4 (bigint-and-non-finite in greater-than, less-than, >=, <=)

## Expected Impact

Fixes 4 runtime failures (BigInt + Infinity comparisons).

## Suggested Approach

1. In comparison codegen, when one operand is BigInt (i64) and the other is Number (f64):
   - Convert the i64 operand to f64 via `f64.convert_i64_s`
   - Compare using f64 comparison operators
   - This handles Infinity/NaN naturally (NaN comparisons return false, Infinity compares correctly)
2. For precision: values outside i64 range are always greater/less than any i64, and within range f64 has enough precision for comparison purposes

## Acceptance Criteria

- [ ] BigInt vs Infinity/NaN comparisons do not trap
- [ ] All 4 bigint-and-non-finite tests pass
- [ ] Existing BigInt comparison tests still pass

## Implementation Notes

Modified `compileBinaryExpression` in `src/codegen/expressions.ts` to handle mixed BigInt (i64) + Number (f64) operands:
- When TS types detect one BigInt and one Number, compile each with its natural type hint (i64 or f64)
- Convert i64 to f64 via `f64.convert_i64_s` before comparison
- Added fallback at compiled-type level: when leftType/rightType are mixed i64/f64, convert i64 side to f64
- This avoids `i64.trunc_f64_s` which traps on Infinity/NaN

Added 2 equivalence tests: BigInt vs Infinity comparison, BigInt comparison operators with numbers.
