# #662 — For/for-of/destructuring timeouts (3,330 FAIL)

## Status: open

3,330 non-Temporal tests timeout at 90s. Three root causes:

### 1. Missing iterator return() protocol (965 timeouts)
Tests like `for (const [x] = customIterator; ...)` expect the destructuring to call `iterator.return()` when it only needs the first element. Our iterator implementation calls `next()` but never `return()`, so iterators that return `done: false` forever cause infinite loops.

**Fix:** After array destructuring from an iterator, call `return()` if it exists.

### 2. Collection mutation during for-of (770 timeouts)
`for (const x of set) { set.delete(x); set.add(x + 1); }` — mutating Set/Map during iteration can cause our implementation to loop forever if it re-reads the collection.

**Fix:** Snapshot the collection at the start of for-of, iterate over the snapshot.

### 3. Compiler compilation hangs (1,395 timeouts)
Complex patterns (deeply nested destructuring, generator chains) cause the compiler itself to loop during codegen. Not a runtime issue — compilation never finishes.

**Fix:** Add a per-test COMPILATION timeout (separate from the runtime timeout). If compilation takes >30s, abort with CE instead of waiting 90s.
