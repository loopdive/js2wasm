# #686 — Closure capture type preservation

## Status: open

### Partial progress
Commit e4a2da95 ("fix: closure type mismatches — boxed capture propagation + return coercion") fixes a subset of this: non-mutable captures of boxed variables now correctly register in `liftedFctx.boxedCaptures`, and return coercion is generalized via `coercionInstrs()`. This fixes ~800+ closure compile errors in test262. The broader optimization (concrete-typed ref cells, read-only capture elimination) remains open.

When a variable is captured by a closure, it's often widened to externref even if it has a concrete type (f64, ref $Struct). This forces boxing/unboxing at every access.

### Root cause
The ref cell for mutable captures uses `(field $value (mut externref))`. If the captured variable is f64, every read unboxes and every write boxes.

### Approach
1. When creating a ref cell for a capture, use the variable's actual type: `(field $value (mut f64))` for numbers
2. Only widen to externref if the variable is reassigned with a different type inside the closure
3. For read-only captures: just pass the value directly, no ref cell needed

### Impact
Eliminates boxing overhead for captured loop counters, accumulator variables, and other typed locals. Common in test262 generator and iterator tests.
