# Issue #570: Wasm validation -- stack not empty at fallthrough

## Problem

When `addUnionImports` (or `addStringImports`) is called during closure/method/getter/setter
compilation, it shifts function indices (`call`, `ref.func`) in already-compiled function bodies.
However, it only shifts the CURRENT function context's body and savedBodies -- it does
NOT shift the PARENT function's body, which already contains `call` and `ref.func`
instructions targeting the pre-shift indices.

This causes:
1. `call` instructions in the parent to reference wrong functions (e.g., an import
   instead of the intended function), which can cause type mismatches and stack
   validation failures.
2. `ref.func` instructions to reference wrong function indices, breaking closure
   struct creation.

The error manifests as:
- "expected 0 elements on the stack for fallthru, found N" -- when a misrouted call
  leaves unexpected values on the stack
- "call[0] expected type X, found Y" -- when a misrouted call has wrong parameter types

## Root Cause

When entering nested function compilation (closures, methods, etc.), `ctx.currentFunc`
is swapped to the nested function's context. The parent's body is saved in a local
`savedFunc` variable but is NOT tracked by `addUnionImports` for index shifting.

The parent's `func.body` in `ctx.mod.functions` is a DIFFERENT array from the parent's
`fctx.body` (per the `body: []` pattern), so the `ctx.mod.functions` iteration in
`addUnionImports` shifts the wrong (empty) array.

## Implementation Summary

### What was done
- Added `activeParentBodies: Instr[][]` to `CodegenContext` to track parent function
  bodies that need index shifting during late import additions.
- Added `enterNestedFunc(ctx)` / `leaveNestedFunc(ctx, pushed)` helper functions in
  `index.ts` that push/pop the current function's body and savedBodies onto the stack.
- Updated all 11 sites in `expressions.ts` and `statements.ts` where `ctx.currentFunc`
  is saved/restored for nested function compilation to call these helpers.
- Updated `addUnionImports`, `addStringImports`, and `shiftLateImportIndices` to also
  shift all entries in `ctx.activeParentBodies`.

### Files changed
- `src/codegen/index.ts` -- added `activeParentBodies` field, `enterNestedFunc`/`leaveNestedFunc` helpers, and parent body shifting in `addUnionImports` and `addStringImports`
- `src/codegen/expressions.ts` -- added `enterNestedFunc`/`leaveNestedFunc` calls at all 9 nested function sites, and parent body shifting in `shiftLateImportIndices`
- `src/codegen/statements.ts` -- added `enterNestedFunc`/`leaveNestedFunc` calls at 2 nested function sites

### Verification
- The specific reproducer (`1n + 1` in closures mixed with `Object(1n) + 1`) now compiles
  and validates correctly -- previously failed with "expected 0 elements on stack for fallthru, found 2"
- 139 tests pass with no new regressions (only 1 pre-existing failure in closure capture test)
