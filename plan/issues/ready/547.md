# Issue #547: Restore search/filter UI in report.html

The filter UI was lost when conformance-report.html was merged into report.html. Add back:

- Text search box to filter categories by name
- Status toggle cards (pass/fail/CE/skip) to filter by status
- Click-to-expand category rows showing individual test results (already present)

## Key files
- `benchmarks/report.html`

## Implementation Summary

### What was done
- Added CSS styles for `.filter-bar`, `.search-box`, `.filter-toggle`, and `.card.filter-card` classes
- Modified `renderConformance()` to make the four status summary cards (Passed, Failed, Compile Errors, Skipped) clickable toggle filters
- Added a text search input that filters categories by name (matches both full path and short display name)
- Both filters combine: search narrows by name, status card narrows by which categories have nonzero counts in that status
- Group header rows auto-hide when all their child categories are filtered out
- Open detail rows are cleared when filters change to avoid stale state

### What worked
- Reusing the existing `makeCard` helper and just adding a `filter-card` CSS class for the toggle behavior
- Storing `rowsByGroup` map for efficient DOM show/hide without re-rendering

### Files changed
- `benchmarks/report.html` -- added filter UI styles and filtering logic in `renderConformance()`
