# #791 -- SyntaxError detection gaps: code compiles when it should not

## Status: in-progress

2,690 test262 tests expect a SyntaxError (negative parse phase) but ts2wasm successfully compiles the code.

## Implementation

Extended `detectEarlyErrors()` in `src/compiler.ts` with new checks:

1. **"use strict" + non-simple parameters**: Functions with destructuring, default, or rest params cannot have "use strict" directive in body
2. **Labeled declarations**: `label: let x`, `label: const x`, `label: class C {}` are SyntaxErrors
3. **eval/arguments as binding names in strict mode**: `var eval = 42` in strict mode
4. **Switch case duplicate lexical declarations**: Duplicate let/const/function names across case clauses
5. **Static prototype**: `class C { static prototype() {} }` is a SyntaxError
6. **Duplicate private names**: Duplicate private field/method names (except get+set pairs)
7. **let/const in single-statement positions**: `if (x) let y;`, `do const z; while (false)`

Also fixed `handleNegativeTest()` in test runner to add `"use strict"` prefix for `onlyStrict` tests.

## Impact
Estimated 2,690 test262 improvements (100% detection rate on sampled tests).
