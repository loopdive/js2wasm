# #759 -- Insert missing extern.convert_any at call sites (1,299 CE)

## Problem

1,299 tests fail with "call[0] expected type externref, found ref" — the compiler pushes a struct ref argument but the function parameter is typed as externref. Missing `extern.convert_any` coercion at the call site.

## Fix

In the call expression compilation, after pushing each argument, check if the pushed type is a struct ref but the parameter expects externref. If so, emit `extern.convert_any`.

## Acceptance criteria

- 1,299 compile errors resolved
- No new regressions
