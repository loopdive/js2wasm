# #615 — Assignment to non-array (181 CE)

## Status: review

181 tests fail with "Assignment to non-array" at various line positions (L32:5, L31:5, L43:5). These are destructuring patterns where the RHS is not recognized as an array — likely iterator results, generator yields, or custom iterable objects.

### Example tests
- `dstr/private-gen-meth-dflt-ary-ptrn-elem-id-iter-val-array-prototype.js`
- `dstr/ary-ptrn-elem-id-iter-val-array-prototype.js`

These modify `Array.prototype[Symbol.iterator]` which changes how arrays destructure — the compiler doesn't handle custom iterator overrides.

## Complexity: M

## Implementation Summary

### What was done
Added externref fallback paths for array destructuring in both assignment expressions and variable declarations. When the RHS of a destructuring pattern compiles to externref (e.g., any-typed parameters, iterator results), instead of erroring with "Cannot destructure: not an array type", the compiler now:

1. Stores the externref value in a temp local
2. Ensures `__extern_get` import is registered (via new exported `ensureExternGetImport` helper)
3. Ensures `__box_number` is available for boxing numeric indices to externref
4. For each destructuring target, emits `__extern_get(obj, i)` to extract elements by index
5. Handles identifier targets, holes (omitted expressions), default values, and nested patterns

### Files changed
- `src/codegen/expressions.ts`: Added `ensureExternGetImport()` exported helper, `compileExternrefArrayDestructuringAssignment()` for assignment destructuring, updated `compileArrayDestructuringAssignment` to use externref fallback
- `src/codegen/statements.ts`: Added `compileExternrefArrayDestructuringDecl()` for variable declaration destructuring, updated `compileArrayDestructuring` to use externref fallback, added import for `ensureExternGetImport`
- `tests/equivalence/externref-array-destructuring.test.ts`: New test file with 5 tests covering assignment destructuring, declaration destructuring, holes, and function returns

### What worked
- Using `__extern_get(obj, boxed_index)` to extract elements from externref array-like values
- Boxing numeric indices via `__box_number` (ensured via `addUnionImports`)
- `ensureExternGetImport` as reusable helper with proper late import index shifting

### What didn't
- Default value detection via `ref.is_null` doesn't catch JS `undefined` (externref undefined is not wasm null). This is a known limitation that could be addressed in a follow-up issue.
