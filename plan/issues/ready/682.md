# #682 — RegExp: pure Wasm engine for standalone mode, host for js-host mode

## Status: open

#676 proposed host imports for RegExp. But in standalone mode (wasmtime/WASI), there's no JS host to delegate to.

### Two modes (like strings #679)

**js-host mode**: Delegate to host RegExp (current approach, fast, complete)
- `new RegExp(pattern, flags)` → host `__regexp_new`
- `.test()`, `.exec()` → host imports
- Full spec compliance via JS engine

**standalone mode**: Pure Wasm RegExp engine
- Compile RegExp patterns to Wasm state machines at compile time
- `/abc/` → Wasm function that scans i16 string arrays
- Simple patterns (literal match, character classes, quantifiers) → direct Wasm loops
- Complex patterns (backreferences, lookahead) → backtracking DFA in Wasm

### Phased approach
1. **Literal patterns**: `/hello/` → string indexOf. Covers ~30% of test262 RegExp tests
2. **Character classes**: `/[a-z0-9]/` → range checks. +20%
3. **Quantifiers**: `/a+/`, `/a{2,4}/` → loop counters. +15%
4. **Anchors**: `/^..$/` → position checks. +10%
5. **Groups + captures**: `/(a)(b)/` → save positions. +15%
6. **Backtracking**: Lookahead, backreferences. +10%

### Prior art
- RE2 (Google): Compiles to DFA, no backtracking. Could port to Wasm.
- Rust regex crate: NFA-based. Fast, no backtracking.
- For Phase 1-4, a simple NFA is sufficient and ~500 lines of Wasm codegen.

## Complexity: XL (full), M (phases 1-3)
