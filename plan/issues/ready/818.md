# #818 -- Internal error: fctx is not defined during compilation (~251 tests)

## Problem

~251 test262 tests fail with `Internal error compiling expression: fctx is not defined`. This means the compiler accesses `fctx` (FunctionContext) in a code path where it's not in scope.

## Reproduction

```
test/language/expressions/arrow-function/cannot-override-this-with-thisArg.js
```

This test uses `this` inside module scope: `[1].forEach(value => { ... }, usurper)`. The module-level code has no FunctionContext, so `fctx` is undefined when compiling the arrow function body or `this` reference.

## Root Cause

Likely in expression compilation paths that assume a FunctionContext exists but can be reached from module-level code. The `fctx` variable is passed through most compilation functions but some code paths reference it without null-checking.

## Fix Approach

1. Find all compilation paths where `fctx` can be undefined:
   - Module-level expressions that get compiled without a function context
   - Arrow function `this` resolution at module scope
2. Add guards or ensure a valid FunctionContext exists for module-level code

## Acceptance Criteria

- `fctx is not defined` errors eliminated
- Module-level arrow functions with `this` compile without error
