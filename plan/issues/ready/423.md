# #423 — Invalid field index in struct access (16 CE)

## Problem

16 tests fail with "invalid field index" errors when accessing struct fields. The compiler generates a struct.get or struct.set with a field index that is out of bounds for the struct type.

Root causes:
- Class inheritance where child accesses parent field with wrong offset
- Dynamic property addition creating fields not in the static struct definition
- Mismatch between struct type definition and field access index

## Priority: medium (16 tests)

## Complexity: S

## Acceptance criteria
- [ ] Field index validation before struct.get/set emission
- [ ] Correct field offset calculation for inherited fields
- [ ] Reduce "invalid field index" CEs to zero
