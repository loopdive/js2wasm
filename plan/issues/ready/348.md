# #348 -- Null/undefined arithmetic coercion

## Status: review

362 tests need null-to-0 and undefined-to-NaN coercion in arithmetic/unary contexts. Quick win: emit `f64.const 0` for null, `f64.const NaN` for undefined in numeric expressions.

## Details

JavaScript coerces null and undefined in numeric contexts:
```javascript
+null === 0          // true
+undefined !== +undefined  // true (NaN)
null + 1 === 1       // true
undefined + 1        // NaN
null * 5 === 0       // true
```

Current behavior: these cases either fail to compile or produce incorrect results.

Fix:
1. In `compileUnaryExpression` for `+` and `-`: detect null/undefined operands, emit appropriate f64 constants
2. In `compileBinaryExpression` for arithmetic ops: when one operand is null, coerce to `f64.const 0`; when undefined, coerce to `f64.const NaN`
3. In `coerceType` when target is f64: handle null -> 0, undefined -> NaN

## Implementation Summary

### What was done
1. **`tryStaticToNumber` (expressions.ts)**: Added type assertion/parenthesized expression unwrapping at function entry, so `+(undefined as any)` correctly resolves `undefined` through the `as any` cast and returns NaN.

2. **Unary minus detection (compilePrefixUnary)**: Extended the null/undefined operand detection to unwrap `AsExpression`, `NonNullExpression`, `TypeAssertion`, and `ParenthesizedExpression` -- matching the same pattern used in the fast-path at the top of `compileExpression`. This ensures `-(undefined as any)` correctly emits `f64.const NaN` instead of going through externref.

3. **test262-runner.ts**: Removed the "unary +/- on null/undefined" skip filter (lines 353-360) since the compiler now handles these cases correctly.

### What already worked
- Binary arithmetic with null/undefined (e.g., `null + 1`, `undefined * 5`) was already handled by the `compileExpression` fast-path which detects literal null/undefined when `expectedType` is f64/i32.
- Unary `+null` was already handled by `tryStaticToNumber`.
- Unary `-null` (without type cast) was already handled by the existing null detection in compilePrefixUnary.

### Files changed
- `src/codegen/expressions.ts` -- `tryStaticToNumber` unwrapping, unary minus detection unwrapping
- `tests/test262-runner.ts` -- removed null/undefined arithmetic skip filter
- `tests/issue-348.test.ts` -- 12 unit tests for null/undefined arithmetic coercion
