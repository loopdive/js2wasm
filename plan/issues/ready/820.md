## Problem

## Root causes

1. **Missing property descriptors on built-in methods** (~500 tests) â€” `Object.getOwnPropertyDescriptor(Array.prototype, "map")` etc. returns null because built-in methods are host imports without descriptor metadata
2. **Prototype chain not resolved at runtime** (~1,000 tests) â€” property lookup on inherited properties traps because we only check the immediate struct
3. **Null guards too aggressive or missing** (~2,000 tests) â€” `ref.cast` traps when the value is null or wrong type, instead of returning undefined or throwing TypeError
4. **Built-in `this` validation missing** (~500 tests) â€” methods like `TypedArray.prototype.sort.call(notTypedArray)` should throw TypeError but we don't validate `this`
5. **General type dispatch** (~3,000 tests) â€” methods called on externref values where we lost the concrete type
