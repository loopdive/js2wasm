# #820 -- TypeError / null dereference failures (6,077 tests)

## Problem

6,077 tests fail with type_error (dereferencing null/undefined, calling non-functions, accessing properties on wrong types). All 6,077 are classified as "null/undefined access" -- Wasm traps from `ref.cast` or `struct.get` on null references where JavaScript would throw TypeError or return undefined.

### Sample test files with exact errors

**1. Class expression accessor with computed property (null deref)**
File: `test/language/expressions/class/accessor-name-inst-computed-in.js`
Error: `TypeError (null/undefined access): AssignmentExpression may contain 'in' keyword regardless of outer context`
```js
// Line 33-35:
var C = class {
  get [key]() { return 'get string'; }
};
```
Root cause: Class expression with computed accessor key dereferences null when the class struct is not fully initialized.

**2. Async generator method default params (null deref on arguments)**
File: `test/language/expressions/class/async-gen-method-static/dflt-params-arg-val-not-undefined.js`
Error: `TypeError (null/undefined access): Use of initializer when argument value is not 'undefined'`
Root cause: Static async generator method compilation produces null struct reference.

**3. Array.prototype method called on non-array (for-of iteration)**
File: `test/language/statements/for-of/Array.prototype.entries.js`
Error: `TypeError (null/undefined access): The method should return a valid iterator that can be traversed using a 'for...of' loop.`
```js
// Line 5-7:
for (var value of [].entries()) {
  // ...
}
```
Root cause: `Array.prototype.entries()` returns an iterator object that is null at Wasm level.

**4. TypedArray Symbol.species check (null deref)**
File: `test/built-ins/TypedArray/Symbol.species/length.js`
Error: `TypeError (null/undefined access): get %TypedArray% [ @@species ].length is 0.`
Root cause: TypedArray constructor object does not expose `Symbol.species` getter.

**5. arguments.length property (null deref)**
File: `test/language/arguments-object/10.6-7-1.js`
Error: `TypeError (null/undefined access): Arguments Object has length as its own property`
```js
(function () { assert.sameValue(arguments.length, 0); })();
```
Root cause: Arguments object struct is null when function has zero parameters.

## Root causes

1. **Class expression struct initialization order** (~1,420 tests) -- class expressions produce null struct references when methods/accessors reference the class before it is fully set up. File: `src/codegen/index.ts`
2. **Built-in prototype method dispatch** (~500 tests) -- calling built-in methods on non-standard receivers (`.call()`, `.apply()`) dereferences null because the method lookup falls through to null. File: `src/codegen/expressions.ts`
3. **TypedArray/DataView incomplete implementation** (~650 tests) -- TypedArray constructors and prototype methods are partially stubbed, returning null for unsupported operations. File: `src/codegen/expressions.ts`
4. **Iterator/async iterator protocol** (~345 tests) -- iterator result objects are null, `next()` returns null instead of `{value, done}`. File: `src/codegen/statements.ts`
5. **Object property reflection** (~440 tests) -- `Object.getOwnPropertyDescriptor`, `Object.keys`, etc. return null for Wasm struct properties. File: `src/codegen/expressions.ts`
