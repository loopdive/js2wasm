# #650 — Stack fallthrough errors (300 CE)

## Status: open

300 tests fail with "expected N elements on the stack for fallthru". Wasm validation requires all branches of an if/block to leave the stack in the same state.

### Root cause
Conditional expressions or if statements where one branch produces a value and the other doesn't (or produces a different type).

### Fix
After compiling if/else branches, verify both sides produce the same stack type. Push a default value for the missing side.
