# Static eval() and new Function() support

## Problem
~2,500 test262 tests use `eval()` or `new Function()`. Currently all are skipped.

## Solution
When `eval()` is called with a string literal argument, parse it at compile time
using `ts.createSourceFile` and compile the resulting statements inline. The return
value is the completion value (last expression statement's value).

For `new Function("p1","p2","body")`, synthesize a function expression source,
parse it, and compile through `compileArrowAsClosure`.

## Scope
- `eval("string literal")` -- static compile-time eval
- `(0, eval)("string literal")` -- indirect eval (same codepath)
- `new Function("params...", "body")` -- static compile-time function creation
- String literals inside eval'd code are auto-registered via `collectEvalStringLiterals`

## Limitations
- Dynamic eval (non-literal argument) is NOT supported -- would need runtime compilation
- `new Function()` compiled functions have no type checker info (all params are externref/any)
