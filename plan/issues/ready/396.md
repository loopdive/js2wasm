# #396 -- Runtime null pointer dereference (64 FAIL)

## Status: open

64 tests compile but fail at runtime with null pointer dereference errors when accessing fields or methods on null struct references.

## Details

The Wasm runtime traps when `struct.get` or `struct.set` is called on a null reference. In JavaScript, this would throw a TypeError, but in our compiled output it causes an unrecoverable trap.

Common scenarios:
- Accessing properties on a variable that was conditionally assigned
- Method chains where an intermediate result is null/undefined
- Constructor patterns that may not initialize all fields
- Optional chaining (`?.`) not properly guarding downstream access

Fix:
1. Insert null checks (ref.is_null + br_if) before struct access in hot paths
2. Ensure optional chaining emits proper null guards
3. For method chains, check intermediate results before field access

## Complexity: M

## Acceptance criteria
- [ ] Null struct references produce graceful behavior instead of traps
- [ ] Optional chaining properly short-circuits on null
- [ ] Reduce null dereference runtime failures by ~64
