# #488 — Property introspection: hasOwnProperty / propertyIsEnumerable (1,617 tests)

## Status: open

1,617 tests skipped because they use `hasOwnProperty`, `propertyIsEnumerable`, or `Object.prototype.hasOwnProperty.call()`.

## Approach

In WasmGC, struct fields are statically known. Property introspection can be resolved at compile time:

### hasOwnProperty
```javascript
obj.hasOwnProperty("x")  // → does struct type have field "x"? compile-time boolean
Object.prototype.hasOwnProperty.call(obj, "x")  // same
```

For statically known property names: emit `i32.const 1` or `i32.const 0` at compile time.

For dynamic property names (`obj.hasOwnProperty(varName)`): need a runtime lookup table mapping string → field index per struct type. Compile a static array of field names per struct type, check membership at runtime via string comparison.

### propertyIsEnumerable
Same as hasOwnProperty for our purposes — all struct fields are enumerable (we don't support property descriptors).

### Implementation
1. For literal string arguments: resolve at compile time (zero cost)
2. For dynamic arguments: emit a field name table per struct type and do runtime string lookup
3. `Object.prototype.hasOwnProperty.call(obj, key)` — detect this pattern and compile as above
4. Narrow skip filter to only skip tests that use `Object.getOwnPropertyDescriptor` or `Object.getPrototypeOf`

## Complexity: M

## Acceptance criteria
- [ ] `obj.hasOwnProperty("x")` returns true/false based on struct field existence
- [ ] `Object.prototype.hasOwnProperty.call(obj, key)` works
- [ ] Dynamic key lookup works via field name table
- [ ] Unlock 1,000+ tests
