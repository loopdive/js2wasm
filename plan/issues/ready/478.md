# #478 — assert.throws side-effect assertions (342 tests)

342 tests are skipped because they use `assert.throws` with side-effect-dependent assertions that our current shim doesn't handle. The #397 shim calls the function and catches errors but doesn't verify the error type or check side effects after the throw. Narrow the skip filter or enhance the shim.

## Implementation Notes

### Finding: skip filter already removed

The skip filter for "assert.throws with side-effect-dependent assertions" was already removed in issue #397. The 342 count in this issue came from a stale test262 report (commit b944e61b, before the filter was removed). These tests are already being attempted by the runner -- they are no longer skipped for this reason.

The old filter checked whether `assert.sameValue` or `assert()` appeared after the last `assert.throws` call, and skipped those tests. That filter was removed when `transformAssertThrows` was implemented, which properly transforms `assert.throws(ErrorType, fn)` into `assert_throws(fn)` with a try/catch shim.

### Fix: assert_throws shim now fails on missing throw

The `assert_throws` shim was missing failure detection: when `fn()` completed without throwing, the test silently passed even though `assert.throws` semantically requires that the function throws. Fixed by adding `__fail = 1` after the try/catch block, so tests that expect a throw but don't get one are correctly reported as failures.

### Files changed
- `tests/test262-runner.ts` -- added `__fail = 1` to `assert_throws` shim when function does not throw
