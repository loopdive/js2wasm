# #746 -- Test262 regression analysis: stale cache + test runner changes

## Problem

Comparing the Mar 21 run (15,232 pass) to the current run (14,045 pass) shows ~1,187 apparent regressions. Analysis reveals these are NOT compiler regressions.

## Root cause analysis

### 1. Stale disk cache (PRIMARY CAUSE: ~2,000+ false positives)

The `buildCompilerHash()` function in `tests/test262-vitest.test.ts` only hashes 7 specific codegen files. After the expressions.ts refactoring (#688), code was extracted into 8 NEW files:
- `property-access.ts`, `binary-ops.ts`, `string-ops.ts`, `closures.ts`
- `literals.ts`, `object-ops.ts`, `typeof-delete.ts`, `array-methods.ts`, `shared.ts`

Changes to these files do NOT invalidate the cache. The checker (`src/checker/index.ts`) is also missing from the hash, so lib file changes don't invalidate either.

**Impact**: The Mar 21 run used cached Wasm binaries compiled by older compiler versions that:
- Didn't have null-check throws (pre-#728)
- Had different type resolution
- Produced different codegen for property access, string ops, etc.

This inflated the Mar 21 pass count by an unknown amount (likely 2,000+).

### 2. Test runner changes (error reporting, not actual behavior)

Between the runs, the test runner changed significantly:
- `a73c2cc0`: Added WebAssembly.Exception handler that surfaces null-payload exceptions as "TypeError (null/undefined access)" -- previously these were reported with `execErr.message ?? String(execErr)` which produced different error text
- `5d345935`: Re-enabled `throw e` in the test wrapper catch block -- previously exceptions were swallowed and tests returned -1, recorded as "fail, returned -1"
- `ed7c96ec`: Stopped stripping comments from test source

These changes cause existing failures to be reported DIFFERENTLY (different error messages), not new failures.

### 3. Actual compiler changes (minor)

The only real codegen changes between baseline and HEAD:
- `#728` reverts (89b061b4, 673d73c6, 20e73e08): These RESTORE baseline behavior
- `#697` block type widening: Fixes ref/ref_null mismatches
- `#698` argument type coercion: Fixes call dispatch paths
- `#721` early error checks: Adds SyntaxError for delete, with, etc.
- `#723` TDZ checks: Adds ReferenceError for let/const before init
- `#725` ref.cast fallback: Improves property access

None of these should cause previously-passing tests to fail.

## Fix applied

Updated `buildCompilerHash()` to hash ALL `.ts` files in `src/codegen/` directory (using `readdirSync`) plus `src/checker/index.ts`. This ensures cache invalidation when any codegen file changes.
