# #764 -- Promise chain methods (.then, .catch, .finally)

## Status: ready

## Problem

Promise support is limited to `Promise.all()` and `Promise.race()`. The core chaining methods are missing:

1. **Promise.prototype.then(onFulfilled, onRejected)** — creates a new promise that resolves when the callback completes
2. **Promise.prototype.catch(onRejected)** — sugar for `.then(undefined, onRejected)`
3. **Promise.prototype.finally(onFinally)** — runs callback regardless of outcome
4. **Promise.resolve(value)** — wraps a value in a resolved promise
5. **Promise.reject(reason)** — creates a rejected promise
6. **Promise.allSettled()** — waits for all promises to settle
7. **Promise.any()** — resolves with first fulfilled promise

### Impact

Promise chains are fundamental to async JavaScript. Without `.then()`, async/await desugaring produces promises that can't be composed. This blocks real-world async code patterns.

### Test262 categories affected

- `built-ins/Promise`: ~300 tests
- Indirect impact on `language/expressions/async-*` tests

### Fix approach

Promises in Wasm need a microtask queue or event loop model. Options:

1. **Host-backed** (short-term): delegate `.then()` chaining to JS host Promise
2. **Pure Wasm** (long-term): implement promise state machine with callback queues as WasmGC structs, microtask scheduling via a run loop

## Complexity: L

## Acceptance criteria

- `.then()` chains resolve in correct order
- `.catch()` handles rejections
- `.finally()` runs regardless of outcome
- `Promise.resolve/reject` create correctly settled promises
- Async/await code that uses promise chains works end-to-end
