# #561 — Math.hypot closure captures ref instead of f64 (1 CE)

## Status: open

`built-ins/Math/hypot/Math.hypot_ToNumberErr.js` fails with "f64.add[0] expected type f64, found local.tee of type (ref null 8)". A closure captures a variable that should be f64 but is stored as a ref cell. The `local.tee` returns the ref cell type instead of the unwrapped f64.

### Fix
After reading from a ref cell capture in arithmetic context, ensure the value is unboxed to f64 before use in `f64.add`.
