# #721 — Residual negative test false-pass (2,564 FAIL)

## Status: open

## Problem

2,564 tests with `negative: { phase: parse }` metadata still compile successfully when they should fail. #703 added strict-mode validation (generator in statement position, private names outside class, var/lexical redeclaration) but many patterns remain uncaught.

## Top uncaught patterns

Based on test file analysis:

- **`with` in strict mode** — modules are strict, `with` is forbidden (~500)
- **Duplicate parameter names** in strict mode functions (~400)
- **Octal literals** in strict mode (`0777` syntax) (~200)
- **Assignment to `eval`/`arguments`** in strict mode (~200)
- **Invalid regex patterns** that TS accepts (~192 from RegExp tests)
- **Invalid label/break/continue** targets (~150)
- **Invalid destructuring targets** (~100)

## Approach

Add these checks to `detectEarlyErrors()` in `src/compiler.ts`:

1. `with` statement detection (AST walk)
2. Duplicate parameter name detection
3. Octal literal detection
4. `eval`/`arguments` assignment detection
