# #394 — Destructuring produces wrong return values (326 FAIL)

## Status: open

326 tests compile successfully but return wrong results due to destructuring bugs. This is the single largest runtime failure category (51% of all "returned 0" failures).

## Details

Tests use destructuring patterns that compile but produce incorrect bindings at runtime:

```javascript
let [a, b, c] = [1, 2, 3];     // a might be 0 instead of 1
let {x, y} = {x: 10, y: 20};   // x might be 0 instead of 10
function f([a, b]) { return a; } // returns 0 instead of expected value
```

Categories affected:
- `language/statements/class` (101 tests — class method destructuring params)
- `language/expressions/class` (79 tests)
- `language/statements/for-of` (51 tests — for-of with destructuring)
- `language/expressions/assignment` (40 tests)
- `language/statements/function` (37 tests — function param destructuring)

Also related: #396 (64 null pointer failures, all from destructuring) and the 829 "Unsupported call expression" CEs in destructuring context.

## Complexity: L

## Acceptance criteria
- [ ] Array destructuring extracts correct values from iterables
- [ ] Object destructuring extracts correct property values
- [ ] Nested destructuring patterns work in function params, for-of, assignment
- [ ] Reduce "returned 0" destructuring failures by 200+
