# #513 -- returned 0: wrong return value (~3,436 FAIL)

## Status: ready

3,436 tests compile and run but the test function returns 0 (falsy) instead of a truthy value, meaning assertions failed silently. This is the single largest failure category -- 78% of all runtime failures. Previously estimated at 480; the actual count is 7x higher.

### Root cause (umbrella)

This is a catch-all for any test where computed values are wrong. Multiple independent root causes:

1. **Type coercion gaps**: Operations on mixed types produce wrong results (e.g., `"3" + 1` should be `"31"` not `4`)
2. **Missing built-in behavior**: `assert.throws(TypeError, ...)` tests that expect TypeError but the code runs successfully (not-a-constructor checks, etc.)
3. **Scope/hoisting bugs**: Variables resolved to wrong values due to scope issues
4. **Missing `arguments` object**: Many tests reference `arguments` which the compiler may not support
5. **Generator protocol**: `g.next().value` / `.done` not returning correct iterator result shape
6. **Implicit type conversions**: `valueOf`/`toString` not called when needed

### Triage approach

This issue is too large to fix as one unit. Sub-triage needed:
1. Sample 50 tests from each major category
2. Classify root causes into sub-buckets
3. Create targeted sub-issues for each bucket >50 tests
