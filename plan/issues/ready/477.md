# #477 — propertyHelper.js harness support (647 tests)

647 tests are skipped because they require `propertyHelper.js`. Issue #309 added stubs for some harness includes but propertyHelper.js was not included. Add no-op stubs for `verifyProperty`, `verifyEnumerable`, `verifyWritable`, `verifyConfigurable` etc. — tests that also check values will still be meaningful even without descriptor verification.

## Implementation Summary

**What was found**: propertyHelper.js was already in the allowed includes list and the seven no-op stubs were already present (added by #309). However, all stub parameters used `number` types, which would cause TypeScript compilation errors when tests pass string property names or object descriptors (the typical test262 usage is `verifyProperty(obj, "propName", { value: 42 })`).

**What was done**: Updated all seven propertyHelper stub functions to use `any` parameter types instead of `number`:
- `verifyProperty(obj: any, prop: any, desc: any): void`
- `verifyEnumerable(obj: any, prop: any): void`
- `verifyNotEnumerable(obj: any, prop: any): void`
- `verifyWritable(obj: any, prop: any): void`
- `verifyNotWritable(obj: any, prop: any): void`
- `verifyConfigurable(obj: any, prop: any): void`
- `verifyNotConfigurable(obj: any, prop: any): void`

**Files changed**: `tests/test262-runner.ts`
