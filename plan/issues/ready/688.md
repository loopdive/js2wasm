# #688 — Refactor codebase into smaller modules per language feature

## Status: open

expressions.ts is 26K lines — the largest file. It mixes every language feature into one monolithic switch. Each feature (classes, generators, destructuring, closures, array methods, binary ops, property access) should be its own module.

### Proposed modules
- `src/codegen/classes.ts` — class compilation, constructors, methods, inheritance
- `src/codegen/generators.ts` — generator functions, yield, state machines
- `src/codegen/destructuring.ts` — array/object destructuring (already deduplicated in #587)
- `src/codegen/closures.ts` — arrow functions, closure structs, captures
- `src/codegen/array-methods.ts` — forEach, map, filter, etc. (already deduplicated in #586)
- `src/codegen/binary-ops.ts` — binary expressions, type coercion for operators
- `src/codegen/property-access.ts` — property get/set, element access, optional chaining
- `src/codegen/call-expression.ts` — function calls, method dispatch, .call/.apply

### Approach
Extract one module at a time. Each extraction: move functions, update imports, run tests. #591 already extracted type-coercion.ts as proof of concept.
