# #884 -- CI: GitHub Actions test262 on every PR

## Problem

No automated testing on PRs. Regressions only caught manually after merge. The STF application needs to show engineering rigor.

## Requirements

1. GitHub Actions workflow triggered on PR to main
2. Run equivalence tests (fast gate, ~2 min)
3. Run test262 subset (~5K representative tests, ~3 min) as smoke test
4. Full test262 on merge to main (nightly or on-demand)
5. Report pass count in PR comment or check status
6. Block merge if equiv tests fail or pass count drops significantly

## Implementation notes

- Use the sharded runner (#882) for parallel execution
- Container needs 16GB RAM — use a self-hosted runner or large GitHub runner
- Cache the .test262-cache between runs (keyed by compiler hash)
- Store results as GitHub Actions artifacts

## Acceptance criteria

- PR gets a check status with test262 pass count
- Equiv test failure blocks merge
- Conformance number visible in PR without manual testing
