# #826 -- Illegal cast accessing private members from inner functions (592 failures)

## Problem

592 tests fail with "illegal cast" at runtime. All involve accessing private class members (fields, methods, getters, setters) from inner arrow functions or nested regular functions inside class methods.

```js
class C {
  #field = 42;
  method() {
    const inner = () => this.#field; // illegal cast here
    return inner();
  }
}
```

The Wasm `ref.cast` instruction traps because the closure captures `this` as `externref` but the private field access expects a concrete class struct type.

## Root cause

In `src/codegen/expressions.ts`, when compiling closures (arrow functions / nested functions), the `this` capture is stored as `externref` in the closure's ref cell. When the inner function accesses `this.#field`, the codegen emits `ref.cast` to cast the externref back to the class struct type, but the cast fails because:

1. The closure capture mechanism boxes `this` to externref via `extern.convert_any`
2. When reading back, `any.convert_extern` + `ref.cast $ClassStruct` is emitted
3. The `any.convert_extern` may produce an `anyref` that doesn't match the expected struct type (e.g., due to wrapping layers)

## Suggested fix

1. When capturing `this` in a closure inside a class method, preserve the concrete struct type instead of boxing to externref
2. Alternatively, use `ref.test` before `ref.cast` and handle the failure case
3. The closure's captured-this ref cell should use the class struct type: `(field $this (mut (ref null $ClassStruct)))` instead of `(field $this (mut externref))`
