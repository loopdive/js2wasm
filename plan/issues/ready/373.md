# #373 -- Object as loop condition / falsy value handling

## Status: open

8 tests with object/function as loop condition (always truthy in Wasm but should check truthiness), 4 with NaN/undefined/null loop conditions.

## Details

```javascript
var obj = {};
while (obj) { break; } // obj is truthy, should enter loop

var x = NaN;
while (x) { break; } // NaN is falsy, should not enter loop

var y = undefined;
while (y) { break; } // undefined is falsy, should not enter loop
```

Objects and functions are always truthy in JavaScript. The current compilation may not handle non-boolean loop conditions correctly when the value is:
1. An object/function reference (always truthy)
2. NaN (falsy)
3. undefined (falsy)
4. null (falsy)

Implementation: ensure the truthiness coercion in loop conditions handles all value types correctly, not just i32/f64/boolean.
