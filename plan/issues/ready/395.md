# #395 — "fn is not a function" in call/apply and array callbacks (70 FAIL)

## Status: open

70 tests fail with "fn is not a function" at runtime. Root cause analysis:

| Pattern | Count |
|---------|-------|
| `.call()/.apply()` on functions | 50 |
| Array callback methods (filter/reduce/map) | 20 |

## Details

**50 call/apply failures**: Functions passed to `.call()` or `.apply()` are not resolved as callable. Related to #342 (Array.prototype.method.call/apply) which handled the array method case but not general function.call().

**20 array callback failures**: `Array.prototype.filter/reduce/map` receive a callback that becomes non-callable — likely the callback function reference is coerced to externref and loses its funcref type.

```javascript
[1,2,3].filter(function(x) { return x > 1; });  // callback becomes non-callable
arr.reduce(function(a, b) { return a + b; }, 0); // same issue
```
