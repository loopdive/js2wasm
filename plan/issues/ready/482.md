# #482 — Well-known Symbol.toPrimitive for type coercion (113 tests)

## Status: open

113 tests use `Symbol.toPrimitive` — the protocol for custom type coercion (`+obj`, `${obj}`, `obj == 42`).

## Approach

Same pattern as #481: compile-time resolution of a well-known symbol.

1. When a class defines `[Symbol.toPrimitive](hint)`, compile it as `__symbol_toPrimitive` struct method
2. In type coercion paths (arithmetic, string concatenation, comparison), check if the object has a `__symbol_toPrimitive` field and call it with the appropriate hint ("number", "string", "default")
3. This replaces the existing valueOf/toString coercion path when toPrimitive is present

## Complexity: M (depends on #481 for the infrastructure)

## Acceptance criteria
- [ ] `class C { [Symbol.toPrimitive](hint) { return 42; } }` compiles
- [ ] `+new C()` calls toPrimitive with hint "number"
- [ ] `${new C()}` calls toPrimitive with hint "string"
