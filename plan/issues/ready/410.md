# #410 -- Stack fallthru mismatch: control flow branches leave wrong stack state

## Status: ready

590 tests fail with "expected N elements on the stack for fallthru, found M". This is a Wasm validation error where control flow branches (if/else, switch cases, try/catch) leave different numbers of values on the operand stack.

## Root cause

When compiling branching constructs, each branch must leave the same number of values on the stack at the merge point. The compiler does not consistently track or reconcile stack depth across branches, especially in:
- Nested if/else with mixed void/value expressions
- Switch cases with fallthrough that accumulate values
- Try/catch where the catch block has different stack layout
- Ternary expressions inside complex expression contexts

## Example failures

- `test/language/expressions/addition/S11.6.1_A2.2_T1.js` -- addition with coercion branches
- `test/language/expressions/conditional/S11.12_A2.1_T1.js` -- ternary stack mismatch
- `test/language/statements/if/S12.5_A11.js` -- if/else branch stack

## Relationship to prior work

#401 (done) addressed some Wasm validation errors including stack issues but focused on `call` argument counts and `struct.new`. This issue targets the remaining 590 cases specific to control flow fallthru mismatches.
