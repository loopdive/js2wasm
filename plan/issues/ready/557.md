# #557 — Performance: repeated instruction tree traversal for index shifting

## Status: open

`shiftLateImportIndices` (expressions.ts:31-117) recursively walks ALL function bodies every time a late import is added. Called 16+ times from different codegen paths.

Complexity: O(K × F × I) where K=late imports, F=functions, I=instructions per function.

For a program with 100 functions, 500 instructions each, and 10 late imports: ~500K recursive visits.

## Approaches

1. **Batch shifts**: Collect all late imports, apply one shift pass at the end
2. **Symbolic references**: Use symbolic function IDs during codegen, resolve to numeric indices in a final pass
3. **Track shifted state**: Each function body tracks its current shift delta, only re-shift when delta changes

Estimated **15-25% overall compile speedup**.
