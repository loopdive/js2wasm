# #725 -- Fix -501 pass regression from #695/#706 interaction

## Status: in-progress

## Problem

After cherry-picking 19 fixes, test262 pass count dropped from 15,232 to 14,731 (-501). The root cause is the interaction between:
- #706: replaced `ref.cast` with `ref.test` + conditional that returns `ref.null` on type mismatch
- #695: added null-guard in `emitNullGuardedStructGet` that returns default values for null refs

The chain: valid object with wrong struct type -> #706 returns `ref.null` -> property access -> #695 returns default (0/null) instead of the real value -> test fails.

## Fix

1. Modified `emitExternrefToStructGet` to save the externref and fall back to `__extern_get(obj, propName)` when `ref.test` fails (type mismatch) instead of returning default values
2. Modified `emitNullGuardedStructGet` to convert to externref and delegate to `emitExternrefToStructGet` when `propName` is provided
3. Fixed `shiftLateImportIndices` and `addUnionImports` to also shift `ctx.pendingInitBody` (module init body not yet in ctx.mod.functions)
4. Fixed import ordering: call `addUnionImports` before `ensureLateImport` to avoid double-shifting
