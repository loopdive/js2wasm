# #798a -- Emit catch_all in try/catch compilation

## Problem
Foreign JS exceptions (from host imports) bypass Wasm `catch $tag` blocks. They need `catch_all` to be captured.

## Implementation

In `src/codegen/statements.ts`, `compileTryStatement`:

1. After the existing `catch $tag` block, add a `catch_all` block
2. In `catch_all`, the exception is implicitly on the stack as externref
3. Store it in the catch variable's local (same local used by `catch $tag`)
4. If no catch variable, just drop the exception

Current pattern:
```wasm
try
  ...body...
catch $tag
  local.set $catchVar  ;; our exceptions
end
```

New pattern:
```wasm
try
  ...body...
catch $tag
  local.set $catchVar  ;; our exceptions
catch_all
  ;; foreign JS exception — need host import to extract it
  call $__get_caught_exception  ;; returns externref
  local.set $catchVar
end
```

Note: `catch_all` doesn't put a value on the stack in the Wasm spec. We need a host import `__get_caught_exception` that returns the most recent JS exception as externref, OR we can use the `exnref` proposal if available.

Check `src/emit/binary.ts` and `src/ir/types.ts` for catch_all encoding support.

## Files
- `src/codegen/statements.ts` — compileTryStatement
- `src/emit/binary.ts` — catch_all encoding (may already exist)
- `src/ir/types.ts` — catch_all instruction type (may need adding)

## Acceptance criteria
- try/catch blocks have catch_all after catch $tag
- Foreign JS exceptions are caught and stored in catch variable
- No regressions on existing exception handling tests
