# #825 -- Null dereference in static private methods (705 runtime failures)

## Problem

705 tests fail at runtime with "dereferencing a null pointer". All are in `language/expressions/class/elements/` and involve static private methods/fields on class expressions. The compiled code runs but dereferences a null Wasm struct reference.

## Pattern

Every failing test follows the pattern of a class expression with static private methods, static private fields, or static private getters/setters:

```js
class C {
  static #method() { return 42; }
  static publicMethod() { return this.#method(); }
}
```

The null pointer likely comes from accessing the class constructor's private method slot before it is initialized, or from the static private method struct being `ref.null` at the point of access.

## Sample files

- test/language/expressions/class/elements/after-same-line-gen-static-private-methods.js
- test/language/expressions/class/elements/after-same-line-method-static-private-methods.js
- test/language/expressions/class/elements/multiple-definitions-static-private-methods.js
- test/language/expressions/class/elements/same-line-gen-static-private-methods.js
- test/language/expressions/class/elements/same-line-method-static-private-methods.js
- test/language/expressions/class/private-static-field-multiple-evaluations-of-class-factory.js
- test/language/expressions/class/private-static-getter-multiple-evaluations-of-class-factory.js

## Root cause

In `src/codegen/index.ts` (class compilation), static private methods on class expressions may not be properly initialized. The struct field for the private method is allocated but left as `ref.null` because:

1. Static private method initialization happens after class body compilation but the reference is used before
2. Class expressions (as opposed to declarations) may have different initialization ordering
3. The private method brand check passes but the actual function ref is null

## Suggested fix

1. In `compileClassBodies` / `collectClassDeclaration`, ensure static private method functions are compiled and their refs stored in the class struct BEFORE any code that references them
2. Check the initialization order for class expressions vs declarations
3. Specifically look at how `#method` references are resolved in static methods -- the `this` binding for static methods on class expressions may not point to the right struct

## Acceptance criteria

- 705 null_deref failures in class/elements tests resolved
- Static private methods, fields, and accessors work on class expressions
