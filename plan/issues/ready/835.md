# #835 -- Error types not registered as extern classes (21 CE)

## Problem

21 tests fail with "Unknown extern class: Error". The compiler special-cases `new Error(msg)` in expressions.ts but doesn't register Error, TypeError, RangeError, etc. as full extern classes in `ctx.externClasses`. When code accesses properties or calls methods on Error instances (e.g. `String.prototype.trim.call(errObj)`), the extern class lookup fails.

## Sample files

- test/built-ins/String/prototype/trim/15.5.4.20-2-50.js
- test/built-ins/String/prototype/trim/15.5.4.20-2-51.js (TypeError)

## Fix

Register Error, TypeError, RangeError, SyntaxError, URIError, EvalError, ReferenceError in `ctx.externClasses` with basic property access support (message, name, stack). Same pattern as Map, Set, RegExp registration.

## Acceptance criteria

- Error types registered as extern classes
- 21 compile errors eliminated
- Property access on Error instances works (`.message`, `.name`)
