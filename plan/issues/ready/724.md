# #724 — Object.defineProperty: throw TypeError for invalid operations (150 FAIL)

## Status: open

## Problem

150 tests expect `assert.throws(TypeError, function() { Object.defineProperty(...) })` — the spec requires TypeError when redefining non-configurable properties, setting writable on non-configurable, etc. Our implementation silently succeeds or ignores the restriction.

## Approach

In the Object.defineProperty compilation path, add checks:
1. If target property is non-configurable and descriptor changes configurable/enumerable → throw TypeError
2. If target property is non-configurable non-writable and descriptor sets value or writable → throw TypeError
3. If target is non-extensible and property doesn't exist → throw TypeError

## Complexity: M
