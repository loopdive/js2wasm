# #626 — Wasm call/call_ref type mismatch (378 CE)

## Status: review

378 tests fail with Wasm validation: call or call_ref arguments don't match the expected function signature. Arguments are the wrong type (externref vs f64, i32 vs ref).

### Fix
1. (Previous commit) Modified coerceType and compileExpression epilogue to handle ref typeIdx mismatches and different struct types.
2. (This commit) Added compileCallArg helper that wraps compileExpression + coerceType as a safety net for all 46+ call-argument compilation sites. Ensures VOID_RESULT and null cases push defaults, and any remaining type mismatches are coerced.

## Implementation Notes
- Added `compileCallArg()` function near `getFuncParamTypes()` in expressions.ts
- Replaced all call-argument `compileExpression()` calls with `compileCallArg()` across direct calls, closure calls, method calls, static calls, constructor calls, rest-param packing, .call/.apply, and tagged template substitutions
- Test: tests/call-arg-coercion.test.ts with 6 scenarios covering f64->externref, externref->f64, method calls, closures, multi-arg, and static methods
