# #745 -- RegExp_new CE regression from lib.d.ts refactoring (~600 tests)

## Problem

After commit 856ed814 (#740) and fix 0689fe69, ~600 test262 RegExp tests that previously passed now fail with "Missing RegExp_new import for regex literal". The lib file iteration fix works for simple cases (`let x = /hello/i` in a function compiles correctly), but fails for tests where:

1. Regex literals are in global-scope variable initializers
2. The real TypeScript `lib.es5.d.ts` has different type shapes for `RegExpConstructor` than the old custom stubs

### Root cause

`collectExternDeclarations` scans lib files for `declare var RegExp: RegExpConstructor`, but the real TypeScript lib uses a more complex type structure with interfaces, overloads, and cross-references between `lib.es5.d.ts` and `lib.es2015.core.d.ts`. The `collectExternFromDeclareVar` function may not handle these correctly.

### Fix approach

1. Debug `collectExternDeclarations` to see if it finds RegExp in the real lib files
2. If not, adjust the type scanning to handle the real TS lib patterns
3. Verify with the specific failing tests

## Acceptance criteria

- 600 previously-passing RegExp tests compile again
- No new regressions
