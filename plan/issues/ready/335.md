# #335 -- Parser comma errors (non-computed-property contexts)

## Status: review

75 test262 tests fail with "',' expected" parser errors outside of computed property contexts. These are TypeScript parser failures on valid JavaScript syntax that TypeScript does not accept.

## Error pattern
- ',' expected

## Likely causes
- BigInt literal syntax not recognized by TypeScript parser
- Generator yield expressions in certain positions confuse the parser
- Complex expressions in array/object positions that TypeScript rejects
- Some tests use syntax features beyond what the TS compiler supports in allowJs mode

## Complexity: M

## Acceptance criteria
- [x] Reduce test262 failures matching this error pattern

## Implementation Summary

### Root cause found
The `renameYieldOutsideGenerators` function in `tests/test262-runner.ts` did not
recognize `async *method()` as a generator method. The `methodRegex` match for
`*method()` checks the preceding context to distinguish from the multiply operator,
but "async" was not in the allowed set of preceding tokens. This caused `yield`
keywords inside async generator method bodies to be renamed to `_yield`, producing
parse errors like "',' expected" and "Property assignment expected".

### Fix
Added `before.endsWith("async")` to the preceding-context check on line 1081 of
`tests/test262-runner.ts`. This allows `async *method()` to be recognized as a
generator method, preserving `yield` as a keyword inside its body.

### Scope of fix
- Fixes 4 test262 comma-error tests directly (async-gen-yield-spread-arr-multiple,
  async-gen-yield-spread-arr-single, async-gen-yield-spread-obj,
  async-gen-yield-identifier-spread-non-strict)
- The remaining 10 comma-error tests are caused by: hashbang syntax (2), Unicode
  16/17 identifiers (3), and top-level-await module syntax (5) -- all TypeScript
  parser limitations that cannot be addressed by preprocessing.
- Many other async generator tests in test262 that use `yield` will also benefit.

### Files changed
- `tests/test262-runner.ts` -- one-line fix in `renameYieldOutsideGenerators`
- `tests/issue-335.test.ts` -- added 2 tests for async generator yield preservation
