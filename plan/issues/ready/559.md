# #559 — Addition/subtraction result not coerced to externref before call (10 CE)

## Status: open

10 tests fail with "call[0] expected type externref, found f64.add/f64.const/i32.const". The compiler passes the raw f64/i32 result of addition/subtraction directly to a function expecting externref (e.g., `assert.sameValue(a + b, expected)`).

The fix: after compiling a binary arithmetic expression whose result is used as a call argument to a function expecting externref, insert `__box_number` coercion.

### Tests affected
- `language/expressions/addition/S11.6.1_A2.2_T2.js` through `S11.6.1_A3.2_T2.4.js` (8 tests)
- `language/expressions/subtraction/S11.6.2_A3_T2.2.js`, `S11.6.2_A3_T2.5.js` (2 tests)

### Root cause
The addition/subtraction codegen emits `f64.add` or `f64.const` results, but the calling context expects externref for the assert helper. The coercion path (`coerceType` or `emitCoercedLocalSet`) is not being invoked for these call arguments.
