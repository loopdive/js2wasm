# #408 — Compiler hangs on for-of with Set mutation during iteration

## Status: open

The compiler enters an infinite loop when compiling for-of loops that mutate a Set during iteration (delete + re-add entries). This causes the test262 runner to hang indefinitely.

## Details

Test: `language/statements/for-of/set-contract-expand.js`

```javascript
var set = new Set();
set.add(0);
set.add(1);

for (var x of set) {
  set.delete(1);
  set.add(1);  // re-insert during iteration
}
```

The `compile()` function is synchronous, so there's no way to timeout — the entire process hangs. Currently worked around by a skip filter in `test262-runner.ts` that detects `set.delete` + `set.add` inside a for-of loop.

Root cause: likely the codegen for for-of with Set enters an analysis loop trying to resolve the iterator state when the collection is mutated.

## Workaround
Skip filter added in test262-runner.ts: matches `set.delete` + `set.add` in for-of-set context.

## Complexity: M

## Acceptance criteria

- [ ] Compiler has a recursion/iteration depth limit to prevent infinite loops
