# Runtime type narrowing for typeof/instanceof guards

When compiling `if (typeof x === "string") { x.length }`, the true branch should use the narrowed type (string) instead of the original (any/externref).

## Approach

In `compileIfStatement` in `src/codegen/statements.ts`:
1. Detect the `typeof x === "type"` guard pattern
2. In the narrowed branch, allocate a new local of the narrowed type
3. Emit unboxing from the AnyValue struct into the new local
4. Temporarily remap `localMap` so accesses to `x` use the narrowed local
5. Restore the original mapping after the branch

Supported narrowings:
- `typeof x === "string"` -> externref (via `__any_unbox_extern`)
- `typeof x === "number"` -> f64 (via `__any_unbox_f64`)
