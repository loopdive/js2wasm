# Runtime type narrowing for typeof/instanceof guards

When compiling `if (typeof x === "string") { x.length }`, the true branch should use the narrowed type (string) instead of the original (any/externref).

## Approach

In `compileIfStatement` in `src/codegen/statements.ts`:
1. Detect the `typeof x === "type"` guard pattern
2. In the narrowed branch, allocate a new local of the narrowed type
3. Emit unboxing from the AnyValue struct into the new local
4. Temporarily remap `localMap` so accesses to `x` use the narrowed local
5. Restore the original mapping after the branch

Supported narrowings:
- `typeof x === "string"` -> externref (via `__any_unbox_extern`)
- `typeof x === "number"` -> f64 (via `__any_unbox_f64`)

## Implementation Summary

### What was done
Added typeof narrowing to `compileIfStatement`. Two new functions:
- `detectTypeofNarrowing` -- pattern-matches `typeof x === "string"/"number"` and `!==` variants
- `applyTypeofNarrowing` -- allocates a narrowed local, emits unbox call, remaps `localMap`

The narrowing is applied at the start of the correct branch (then for `===`, else for `!==`) and the original `localMap` entry is restored after the branch completes.

### Files changed
- `src/codegen/statements.ts` -- new detection/application functions, integration in `compileIfStatement`
- `tests/equivalence/typeof-narrowing.test.ts` -- 5 new tests

### Tests passing
All 5 new equivalence tests pass. No regressions in existing tests.
