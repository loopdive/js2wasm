# #490 — Function/class .name property (576 tests)

## Status: review

576 tests skipped because they access `.name` on functions or classes.

## Approach

Function and class names are statically known at compile time. Store the name as a string field on the function/class struct:

```
struct Function {
  field $__call (ref $functype)
  field $__name (ref string)    // ← add this
}
```

### Implementation
1. Add `__name` field to function and class struct types
2. When compiling function declarations/expressions, set `__name` to the function's identifier (or `""` for anonymous)
3. When compiling class declarations/expressions, set `__name` to the class name
4. Property access `fn.name` resolves to `struct.get $__name`
5. Arrow functions: name is inferred from assignment target (`const foo = () => {}` → name is "foo")
6. Method definitions: name is the method name

Most of the 576 tests just check that `.name` returns the right string — this is compile-time information.

## Implementation Summary

The `.name` property for functions and classes was already implemented in `compilePropertyAccess`
(src/codegen/expressions.ts, line ~16778). The approach resolves names at compile time using the
TypeScript type checker rather than storing a string field in a struct:

1. Named functions/classes: resolved via `objType.getSymbol()?.name`
2. Anonymous function expressions: inferred from assignment target (variable name)
3. Arrow functions: inferred from assignment target
4. Method names: inferred from property access expression
5. Element access (`obj["method"].name`): inferred from string literal argument

Added comprehensive equivalence tests in `tests/equivalence/function-class-name.test.ts` covering:
- Named function declarations
- Named and anonymous function expressions
- Arrow functions (const and let)
- Class declarations
- Named and anonymous class expressions
- Name used in string concatenation and comparison
