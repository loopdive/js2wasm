# #678 — Dynamic prototype chain traversal

## Status: review

#631 added static prototype patterns. ~625 tests need dynamic traversal.

### Approach (implemented)
1. Every class struct gets a `__proto__` externref field at index 1 (after `__tag`)
2. Instance `__proto__` points to the class's own prototype singleton (lazy-initialized)
3. Prototype singleton's `__proto__` points to the parent class's prototype singleton
4. `Object.getPrototypeOf(instance)` reads `__proto__` via `struct.get` at runtime

### Not yet implemented (future work)
- Property access fallback via `__proto__` chain walking
- `for-in` loop prototype chain key collection
- Dynamic `instanceof` via `__proto__` chain (currently uses static `__tag` approach)

## Complexity: L

## Implementation Summary
- Added `__proto__` externref field to root class structs via `fields.unshift` (child classes inherit it)
- Constructor lazy-inits the class prototype singleton and stores it in `__proto__` field
- `Object.getPrototypeOf(instance)` now reads `struct.__proto__` field dynamically
- `emitLazyProtoGet` updated to set prototype's `__proto__` to parent's prototype
- `fixupStructNewArgCounts` updated to recognize `local.get`/`global.get` as stack-push ops
- `compileSuperCall` updated to filter out `__proto__` from assignable parent fields
- Added 5 equivalence tests in `tests/equivalence/dynamic-prototype-chain.test.ts`
- Files changed: `src/codegen/index.ts`, `src/codegen/expressions.ts`
