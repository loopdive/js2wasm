# #619 — Residual undefined .kind crashes after null guard (4,230 CE)

## Status: open

4,230 tests hit "Cannot read properties of undefined (reading 'kind')" even after #611 added null guards. The guards prevent crashes but the tests still compile-error because the underlying AST node is genuinely missing.

### Common patterns (from line numbers)
- L45:1 / L46:1 — class element declarations (200 tests)
- L37:1 / L41:1 — async generator methods (170 tests)
- L31:9 — private field declarations (163 tests)
- L54:27 — nested class with private members (151 tests)
- L40:5 — statement compilation (131 tests)

### Root cause
The compiler doesn't handle certain AST node types:
1. Private class fields with initializers
2. Async generator method parameters
3. Static class blocks
4. Class field declarations (PropertyDeclaration)

### Fix
Add handlers for these missing AST node kinds in compileExpression/compileStatement.
