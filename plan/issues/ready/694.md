# Migrate test262 to vitest with per-test disk cache

## Problem
The existing test262 runner uses a custom standalone script. We want a vitest-native
test file that runs test262 tests with a per-test disk cache for compiled Wasm binaries,
so repeated runs skip recompilation for unchanged tests.

## Plan
1. Create `.test262-cache/` directory (add to .gitignore)
2. Create `tests/test262-vitest.test.ts` with disk-cached compilation
3. Hash compiler source files for cache invalidation
4. Start with first 3 categories as proof of concept (20 tests each)
