# Issue #665: Native Wasm Date implementation

## Problem
Date constructor and methods are not supported. `new Date(ms)`, `new Date()`, `Date.now()`,
and `.getTime()` all fail or produce incorrect results.

## Solution
Represent Date as a WasmGC struct with a single f64 timestamp field (ms since epoch).

- `new Date(ms)` -> struct.new with the f64 timestamp
- `new Date()` -> call `__date_now` host import, then struct.new
- `.getTime()` -> struct.get on the timestamp field
- `Date.now()` -> call `__date_now` host import directly
- Date component methods (getFullYear, getMonth, etc.) -> pure Wasm math using civil date algorithm

## Implementation Summary

### What was done
- Registered `$Date` struct type with single `timestamp: f64` field
- Added `dateTypeIdx` field to `CodegenContext` with lazy registration via `ensureDateType()`
- Added `__date_now` host import registration via `ensureDateNowImport()`
- Added Date usage detection in unified collector (new expressions + Date.now() static calls)
- Added `resolveWasmType` mapping: Date symbol -> ref $Date struct (prevents externref fallback)
- Intercepted Date method calls BEFORE `isExternalDeclaredClass` dispatch (critical -- Date is in lib.d.ts)
- Implemented civil date algorithm (Howard Hinnant) for getFullYear/getMonth/getDate/getDay in pure Wasm i32 arithmetic
- Implemented time component extraction (getHours/getMinutes/getSeconds/getMilliseconds) via modular arithmetic

### What worked
- All 15 tests pass: constructor, static method, getTime, valueOf, and all 8 date component methods
- Howard Hinnant algorithm works correctly for both epoch (0) and modern timestamps
- Structured `if` blocks with `then`/`else` arrays (not flat if/else/end opcodes)

### What didn't work initially
- Flat if/else/end instruction format caused "body is not iterable" errors -- had to use structured format
- Date method dispatch was falling through to extern class handler because Date is in lib.d.ts -- had to intercept before `isExternalDeclaredClass` check
- Variables typed as Date were getting `externref` instead of `ref $Date` -- needed resolveWasmType mapping
- getMilliseconds subtraction operand order was reversed (f64.sub pops a,b and computes a-b)

### Files changed
- `src/codegen/index.ts` -- dateTypeIdx, ensureDateType, ensureDateNowImport, collector, resolveWasmType
- `src/codegen/expressions.ts` -- new Date handler, Date.now() handler, Date method dispatch
- `tests/date-native.test.ts` -- 15 new tests
- `tests/equivalence/helpers.ts` -- __date_now stub
