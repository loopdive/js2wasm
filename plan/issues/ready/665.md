# Issue #665: Native Wasm Date implementation

## Problem
Date constructor and methods are not supported. `new Date(ms)`, `new Date()`, `Date.now()`,
and `.getTime()` all fail or produce incorrect results.

## Solution
Represent Date as a WasmGC struct with a single f64 timestamp field (ms since epoch).

- `new Date(ms)` -> struct.new with the f64 timestamp
- `new Date()` -> call `__date_now` host import, then struct.new
- `.getTime()` -> struct.get on the timestamp field
- `Date.now()` -> call `__date_now` host import directly
- Date component methods (getFullYear, getMonth, etc.) -> pure Wasm math using civil date algorithm

## Scope
Phase 1: new Date(ms), new Date(), .getTime(), Date.now()
Phase 2: getFullYear, getMonth, getDate, getDay, getHours, getMinutes, getSeconds, getMilliseconds
