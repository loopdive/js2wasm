# #665 — Native Wasm Date implementation (not host imports)

## Status: open

~486 Date tests fail. Currently Date methods are partially host-imported. Should be a native Wasm implementation for standalone mode.

### Approach
1. `new Date()` → i64 timestamp (milliseconds since epoch)
2. `new Date(year, month, day, ...)` → compute timestamp from components
3. `.getFullYear()/.getMonth()/.getDate()` → decompose timestamp via arithmetic
4. `.getTime()` → return raw i64
5. `.toISOString()` → format to string via arithmetic + string concat
6. Legacy methods (getYear, setYear, toGMTString) → thin wrappers

Store Date as a struct with one i64 field. All date math is pure arithmetic — no host needed.

### js-host mode fallback
In js-host mode, delegate to JS Date for timezone awareness (Intl, locale). In standalone mode, assume UTC.
