# Issue #254: Private class fields and methods (#field)

## Status: review

## Summary

16 tests fail with "Cannot assign to '#field' because it is a read-only property" combined with ClassDeclaration errors. These tests use private class fields (`#field`) and private methods (`#method()`). Private fields were partially implemented in #127 but some patterns remain broken.

## Root Cause

TypeScript treats private fields as read-only in some contexts, generating a diagnostic error. The actual issue is that private field mutation in constructors or methods may not be properly handled. Additionally, private fields in class expressions (vs declarations) may not work.

## Scope

- `src/codegen/expressions.ts` -- class field initialization
- `src/codegen/index.ts` -- diagnostic suppression
- Tests affected: ~16 compile errors

## Expected Impact

Fixes ~16 compile errors.

## Suggested Approach

1. Suppress the "read-only property" diagnostic for private fields in allowJs mode
2. Ensure private field mutation (assignment) works in constructor and method bodies
3. Compile `this.#field = value` as `struct.set` with the mapped field index

## Acceptance Criteria

- [ ] Private field assignment in constructors compiles
- [ ] Private field assignment in methods compiles
- [ ] At least 10 compile errors resolved

## Implementation Notes

The fix suppresses TypeScript diagnostic codes that incorrectly flag valid JavaScript patterns
involving private class fields:

- **2540**: "Cannot assign to '#X' because it is a read-only property" (18+5 = 23 tests)
- **2803**: "Cannot assign to private method '#X'. Private methods are not writable" (15+4+4 = 23 tests)
- **2806**: "Private accessor was defined without a getter" (5 tests)
- **18030**: "An optional chain cannot contain private identifiers" (4 tests)
- **2729**: "Property '#X' is used before its initialization" (2 tests)
- **18014**: "Private identifier shadowed by another with same spelling" (2 tests)

These diagnostics are TypeScript-specific strictness checks that do not apply in the allowJs
compilation mode used by ts2wasm for test262 conformance.

Added 3 new equivalence tests covering: private field assignment in constructor, private field
mutation in methods, and multiple private fields.
