# #558 — Performance: redundant AST passes + linear type/property lookups

## Status: open

Three related performance issues:

### 1. Multiple AST collection passes (index.ts:465-535)

5-6 separate AST walks before codegen starts:
- `collectConsoleImports`, `collectStringMethodImports`, `collectMathImports`, `collectUnionImports`, etc.

Fix: consolidate into a single visitor that detects all import patterns in one pass. **10-15% speedup**.

### 2. Function type dedup is O(T) (index.ts:6341-6346)

`addFuncType` linearly scans ALL types (structs + arrays + funcs) for each new function type.

Fix: separate `funcTypeMap` keyed by signature string. **5-10% speedup**.

### 3. Property name O(F×P) lookup (expressions.ts:16671)

Object literal compilation does `.find()` per field per property.

Fix: pre-build a `Map<string, Property>` once per literal. **5% speedup on large objects**.

### 4. Double checker calls (index.ts:6774-6880)

`getTypeOfSymbol(prop)` called twice on the same properties in two loops.

Fix: memoize results from the first loop. **5% on struct-heavy code**.
