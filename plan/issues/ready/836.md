# #836 -- Tagged templates with PropertyAccessExpression tag (27 CE)

## Problem

27 tests fail with "Tagged template: unsupported tag expression kind PropertyAccessExpression". The compiler only handles tagged templates where the tag is a simple identifier (e.g. `` tag`...` ``), not a property access (e.g. `` String.raw`...` ``).

## Sample files

- test/built-ins/String/raw/special-characters.js — `` String.raw`\n\t` ``
- test/built-ins/String/raw/zero-literal-segments.js
- test/built-ins/String/raw/template-substitutions-are-appended-on-same-index.js

## Fix

In the tagged template compilation (likely `src/codegen/expressions.ts`), extend the tag resolution to handle `PropertyAccessExpression` — resolve the object, look up the method, and call it with the template args. `String.raw` is the most common case and could be special-cased.

## Acceptance criteria

- Tagged templates with property access tags compile
- `String.raw` works correctly
- 27 compile errors eliminated
