# Remove/narrow stale skip filters

## Problem

Multiple skip filters in `tests/test262-runner.ts` are stale or overly broad, hiding tests
that now pass or would safely compile-error/fail without hanging.

## Filters removed

Based on empirical testing (compile + run all matched files, check for hangs):

| Filter | Tests unblocked | Pass rate | Hangs |
|--------|----------------|-----------|-------|
| arrow returning undefined | 1 | 100% | 0 |
| Math.round large-number precision edge case | 1 | 100% | 0 |
| function expression in catch scope | 26 | 85% | 0 |
| nested function/catch scope with type mismatch | 48 | 25% | 0 |
| this.property at global scope | 26 | 20% | 0 |
| global/arrow this reference | 114 | 15% | 0 |
| arithmetic on objects | 5 | 20% | 0 |
| string comparison with supplementary unicode | 5 | 40% | 0 |
| array index with string concat in loop | 3 | 33% | 0 |
| loose equality between array references | ~0 | 0% (all CE) | 0 |

## Filters narrowed

| Filter | Before | After | Unblocked |
|--------|--------|-------|-----------|
| JSON.stringify replacer/space args | 44 | 7 | 37 |

Changed to strip metadata/comments/strings before checking for "replacer"/"space" keywords.

## Total tests unblocked

~266 tests, with ~40+ estimated new passes.

## Implementation Summary

- Removed 10 skip filters that were verified to produce 0 hangs across all matched test files
- Narrowed JSON.stringify filter to only check executable code (not metadata/comments/strings)
- Files changed: `tests/test262-runner.ts`
- Branch: `issue-544-stale-filters`
