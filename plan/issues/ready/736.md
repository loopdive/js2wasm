# #736 -- SyntaxError detection at compile time (316 tests)

## Status: backlog

## Problem

316 tests expect `SyntaxError` to be thrown (typically via `assert.throws(SyntaxError, function() { eval(...) })`) but the compiler does not detect the syntax error.

### Common patterns
- Duplicate parameter names in strict mode
- `eval` and `arguments` as assignment targets in strict mode
- Reserved words used as identifiers in strict mode
- Octal literals in strict mode
- `delete` of unqualified identifier in strict mode
- Invalid left-hand side in assignment/for-in

### What needs to happen

1. Detect strict mode violations at compile time
2. Emit SyntaxError throws for eval() of invalid code (if eval is supported)
3. Some of these overlap with the negative test handling (#402)
