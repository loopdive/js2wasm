# #622 — Null pointer dereference at runtime (707 FAIL)

## Status: open

707 tests fail with "RuntimeError: dereferencing a null pointer" — struct.get/struct.set on a null reference at runtime.

### Root cause
Variables holding class instances or closure structs are null when accessed. Common triggers:
- Uninitialized class properties accessed before constructor runs
- Optional chaining not emitting null checks
- Function parameters that are undefined at runtime

### Fix
Add null guards before struct.get in hot paths, or ensure variables are properly initialized.
