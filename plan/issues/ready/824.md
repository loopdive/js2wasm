# #824 -- Compilation timeouts: 626 tests exceed 10s compile limit

## Problem

626 tests hit the 10-second compilation timeout. These are counted as compile_error in the results. This is the single largest compile error category (12% of all 5,271 compile errors).

## Root cause analysis

The compiler is doing full Wasm module construction per test. Large test files with many functions, complex class hierarchies, or extensive built-in method usage trigger exponential or high-polynomial time in:

1. **RegExp compilation** (176 tests) -- Complex RegExp patterns with property escapes or unicode sets may trigger expensive string processing during native string helper generation
2. **Object.defineProperty** (20+ tests) -- Descriptor processing may generate excessive helper code
3. **Class compilation** (47 tests) -- Classes with many methods, private fields, and decorators produce large struct types
4. **Built-in method stubs** (100+ tests) -- Tests that exercise many Array/String/Date methods force generation of all helper functions

## Suggested fix

1. Profile compilation of a timeout sample (e.g., `test/built-ins/RegExp/prototype/exec/S15.10.6.2_A1_T16.js`) to find the bottleneck
2. Likely candidates: `ensureNativeStringHelpers` (2,559 lines), `compileClassBodies`, or recursive type resolution
3. Consider lazy helper generation -- only emit helpers actually referenced
4. Consider per-phase time budget to bail early on hopeless cases

## Acceptance criteria

- Timeout count reduced by at least 50% (313+ tests compile within 10s)
- No regressions in pass count
