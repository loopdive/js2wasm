# #824 -- Compilation timeouts: 548 tests exceed 10s compile limit

## Problem

548 tests hit the 10-second compilation timeout. These are counted as compile_error in the results. This is the second largest compile error category.

## Sample files

1. `test/language/expressions/class/dstr/private-meth-static-ary-ptrn-elem-id-init-hole.js`
   - Private static method with destructuring default
2. `test/language/expressions/class/dstr/private-meth-static-dflt-obj-ptrn-id-init-fn-name-gen.js`
   - Private static method with object destructuring default generator
3. `test/language/expressions/class/dstr/private-meth-static-obj-ptrn-prop-obj.js`
   - Private static method with nested object destructuring
4. `test/language/expressions/class/elements/after-same-line-static-async-method-rs-privatename-identifier.js`
   - Static async method with private name identifiers
5. `test/language/expressions/class/elements/after-same-line-static-method-rs-static-async-method-privatename-identifier-alt.js`
   - Static method alongside static async private method

All timeout files involve class expressions with private methods, static methods, destructuring, or combinations thereof.

## Root cause analysis

The compiler is doing full Wasm module construction per test. Large test files with many functions, complex class hierarchies, or extensive built-in method usage trigger exponential or high-polynomial time in:

1. **Class compilation** -- Classes with private methods, static fields, and multiple method kinds produce large struct types and many helper functions
2. **Destructuring in class methods** -- Deeply nested destructuring patterns in method parameters generate large codegen
3. **RegExp compilation** -- Complex RegExp patterns with property escapes trigger expensive string processing
4. **Built-in method stubs** -- Tests that exercise many Array/String/Date methods force generation of all helper functions

## Suggested fix

1. Profile compilation of a timeout sample to find the bottleneck
2. Likely candidates: `ensureNativeStringHelpers`, `compileClassBodies`, or recursive type resolution
3. Consider lazy helper generation -- only emit helpers actually referenced
4. Consider per-phase time budget to bail early on hopeless cases

## Acceptance criteria

- Timeout count reduced by at least 50% (274+ tests compile within 10s)
- No regressions in pass count
