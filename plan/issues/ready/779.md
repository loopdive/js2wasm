# #779 -- Assert failures: tests compile and run but produce wrong values (10,988 tests)

## Problem

Tests fail with "returned 2" (first assertion failed) or other non-1 return values. The code compiles, instantiates, and runs without crashing — but produces incorrect results. This is the single largest failure category.

### History
- 2026-03-25: 8,700 → 7,096 after fixes #780-#787
- 2026-03-28: 10,988 (count increase from unblocking class/elements, String/prototype, directive-prologue, future-reserved-words tests that were previously skipped)

## Root causes (estimated breakdown)

- ~3,500: Property access returns wrong value (missing property descriptors, wrong struct field)
- ~2,500: Missing built-in method behavior (Array, String, Object methods return wrong results)
- ~2,000: Control flow edge cases (scope, hoisting, TDZ, closures)
- ~1,500: Type coercion errors (toString, valueOf, implicit conversions)
- ~1,500: Other (prototype chain, Symbol, iterator protocol)
