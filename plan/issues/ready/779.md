# #779 -- Assert failures: tests compile and run but produce wrong values (8,700 tests)

## Problem

8,700 tests fail with "returned 2" (first assertion failed) or other non-1 return values. The code compiles, instantiates, and runs without crashing — but produces incorrect results.

## Root causes (estimated breakdown)

- ~3,000: Property access returns wrong value (type coercion, wrong struct field)
- ~2,000: Missing built-in method behavior (Array, String, Object methods)
- ~1,500: Control flow edge cases (scope, hoisting, TDZ, closures)
- ~1,200: Type coercion errors (toString, valueOf, implicit conversions)
- ~1,000: Other (prototype chain, Symbol, iterator protocol)

## Fix approach

This is an umbrella issue. Each sub-category needs its own investigation. Priority order:
1. Type coercion errors (most mechanical to fix)
2. Property access wrong values
3. Built-in method edge cases
4. Control flow

## Acceptance criteria

- Triage into sub-issues with specific patterns
- Top 5 sub-categories have fixes
