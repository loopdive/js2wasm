# #779 -- Assert failures: tests compile and run but produce wrong values (10,099 tests)

## Problem

Tests fail with "returned 2" (first assertion failed) or other non-1 return values. The code compiles, instantiates, and runs without crashing -- but produces incorrect results. This is the single largest failure category.

### Sample: assert.sameValue wrong values

**1. Object.defineProperty -- value not set (L19)**
File: `test/built-ins/Object/defineProperty/15.2.3.6-3-126.js`
```js
Object.defineProperty(obj, "property", { value: 100 });
assert.sameValue(obj.property, 100, 'obj.property');
```
Root cause: `Object.defineProperty` does not store the value in the object's property map.

**2. Object.create.length (L10)**
File: `test/built-ins/Object/create/15.2.3.5-0-2.js`
```js
assert.sameValue(Object.create.length, 2, 'Object.create.length');
```
Root cause: Built-in function `.length` property not exposed.

**3. Array.prototype.at type check (L14)**
File: `test/built-ins/Array/prototype/at/index-argument-tointeger.js`
```js
assert.sameValue(typeof Array.prototype.at, 'function', ...);
```
Root cause: `Array.prototype.at` not accessible as a property of the prototype object.

**4. Class computed accessor (L37)**
File: `test/language/expressions/class/accessor-name-inst/computed.js`
```js
assert.sameValue(C.prototype['string'], 'get string');
```
Root cause: Computed property names in class accessors may not be stored correctly.

**5. for-await-of destructuring default values (L32)**
File: `test/language/statements/for-await-of/async-func-decl-dstr-array-elem-init-assignment.js`
```js
assert.sameValue(v2, 2);
assert.sameValue(vNull, null);
assert.sameValue(vHole, 12);
```
Root cause: Destructuring default values in for-await-of not evaluated correctly.

### Sample: assert.throws -- expected exception not thrown

**1. Object.defineProperty on undefined (L9)**
File: `test/built-ins/Object/defineProperty/15.2.3.6-1-1.js`
```js
assert.throws(TypeError, function() {
  Object.defineProperty(undefined, "foo", {});
});
```
Root cause: `Object.defineProperty` does not validate first argument is an object.

**2. Class static prototype generator (L9)**
File: `test/language/computed-property-names/class/static/generator-prototype.js`
```js
assert.throws(TypeError, function() {
  class C { static *['prototype']() {} }
});
```
Root cause: Static 'prototype' property assignment restriction not enforced.

**3. const re-assignment in for-of (L9)**
File: `test/language/statements/const/syntax/const-invalid-assignment-statement-body-for-of.js`
```js
assert.throws(TypeError, function() {
  for (const x of [1, 2, 3]) { x++ }
});
```
Root cause: const reassignment in for-of loop body does not throw TypeError.

### Sample: "returned 3" (second assertion fails)

**1. Mapped arguments nonconfigurable (L17)**
File: `test/language/arguments-object/mapped/mapped-arguments-nonconfigurable-2.js`
```js
a = 2;
assert.sameValue(a, 2);          // passes
assert.sameValue(arguments[0], 2); // FAILS
```
Root cause: Mapped arguments -- setting `a` should also update `arguments[0]` but does not.

**2. Block scope -- for-in property acquisition (L17)**
File: `test/language/block-scope/syntax/for-in/acquire-properties-from-array.js`
```js
assert.sameValue(subject[0], undefined, "[]: first property");
```

Root cause: for-in enumeration returning wrong values.
