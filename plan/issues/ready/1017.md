# #1017 — Null pointer dereferences and illegal casts (504 FAIL)

## Problem

Sub-bucket of #820. 504 test262 failures from Wasm traps:

- `dereferencing a null pointer [in test()]` — 229 FAIL
- `illegal cast [in test()]` — 167 FAIL
- `dereferencing a null pointer [in assert_throws()]` — 108 FAIL

Now with source line numbers (thanks to #1012), these can be traced to specific JS source lines.

## Root cause

The compiler emits `struct.get`, `array.get`, or `ref.cast` on values that are null at runtime. Common scenarios:
1. Uninitialized variables accessed before assignment
2. Optional property access without null check
3. Wrong ref.cast type (value is a different struct type than expected)

## Approach

Sample 10-20 failures with line numbers, group by codegen pattern, fix the top emitters. Likely fixes:
- Add null guards before struct.get/array.get
- Use ref.cast_null instead of ref.cast where the value might be null
- Emit externref fallback path for values that could be null

## Key files
- src/codegen/expressions/ — property access, member expressions
- src/codegen/property-access.ts — struct field access
- src/codegen/index.ts — ref.cast emissions
