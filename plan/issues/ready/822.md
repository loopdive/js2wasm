# #822 -- Wasm type mismatch compile errors (907 CE)

## Problem

907 tests fail with WebAssembly.instantiate type mismatch errors. The compiled Wasm module has type errors that V8 rejects at instantiation time. This is the largest compile error category.

| call expected externref, found f64 | 64 | Missing f64-to-externref boxing at call sites |

## Root cause

Multiple codegen paths in `src/codegen/type-coercion.ts` and `src/codegen/expressions.ts` fail to insert correct type coercions:

1. **Missing unbox**: externref values from array.get need `__unbox_number` before use as f64
2. **Missing box**: f64 values need `__box_number` before passing as externref args
3. **Wrong struct type**: Struct field access uses wrong type index when multiple struct types exist
4. **Incomplete struct.new**: Nested destructuring defaults produce too few arguments for struct.new

## Acceptance criteria

- 907 type mismatch compile errors eliminated
- All sub-patterns addressed with correct coercions

## Implementation Notes (v2 — clean reimplementation after revert)

### Root cause of v1 regression
The original commit (756100cb) accidentally included unrelated deletions:
- 51 lines of for-of destructuring rest element code from statements.ts (#825)
- 14 lines of NewExpression host callback detection from closures.ts (#860)
These were introduced by a stale rebase. The return_call disable was a red herring — #839 already fixed that properly with `canTailCall()` signature validation.

### Changes (v2) — only stack-balance.ts + index.ts
1. **`callArgCoercionInstrs`**: ref.cast_null for ref/ref_null with different struct type indices
2. **`fixBranchType`**: i64 → externref fallthru coercion (drop + ref.null.extern)
3. **`fixCallArgTypesInBody`**: expanded safe coercion set for sub-expression context
4. **`repairBody`**: array.get/set/len repair for externref/extern.convert_any array refs
5. **`fixupExternConvertAny`**: ref.cast_null (not just remove) for struct.new ref fields

### Test results
- Equivalence tests: 1168 pass / 53 fail (matches main baseline — no regressions)
- Does NOT touch statements.ts or closures.ts (avoids accidental reverts)
- Does NOT disable tail-call optimization (handled by #839)
