# #638 — Add reverse typeIdxToStructName map (8 O(N) -> O(1))

## Status: review

8 occurrences of `for (const [name, idx] of ctx.structMap)` doing reverse lookups (find struct name by type index). Each is O(N) where N = number of struct types.

### Fix
Add `typeIdxToStructName: Map<number, string>` to CodegenContext, populated alongside structMap. Replace all reverse scans.

## Complexity: XS

## Implementation Summary

Added `typeIdxToStructName: Map<number, string>` to the `CodegenContext` interface and populated it at all 9 `structMap.set()` call sites (3 wrapper types, class registration, anonymous struct creation, interface registration, etc.).

Replaced 12 reverse-scan loops across 4 files with direct `ctx.typeIdxToStructName.get(typeIdx)` lookups:
- `src/codegen/index.ts` -- 1 site (fixupStructNewArgCounts helper)
- `src/codegen/statements.ts` -- 5 sites (destructuring, for-in compilation)
- `src/codegen/expressions.ts` -- 4 sites (property access, spread, local type resolution)
- `src/codegen/type-coercion.ts` -- 2 sites (ref->externref and ref->f64 coercion)

Net result: -21 lines, all O(N) reverse lookups replaced with O(1) map gets.

Files changed: `src/codegen/index.ts`, `src/codegen/expressions.ts`, `src/codegen/statements.ts`, `src/codegen/type-coercion.ts`
