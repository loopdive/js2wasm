# #740 — Remove duplicated lib.d.ts copies, read from typescript package at runtime

## Status: open

## Problem

`src/checker/lib-*.ts` contains 2.1MB of stringified TypeScript lib files (lib.es5.d.ts, lib.dom.d.ts, etc.) that duplicate what ships with the `typescript` package. They go stale when TS updates.

## Fix

Replace hardcoded string imports with runtime resolution:

### Node.js path
```ts
if (ts.sys?.readFile) {
  const libDir = path.dirname(require.resolve('typescript'));
  return ts.sys.readFile(path.join(libDir, name));
}
```

### Browser path
Fetch from the dev server (vite serves node_modules):
```ts
const resp = await fetch(`/node_modules/typescript/lib/${name}`);
return resp.text();
```

Or use vite's import system with `?raw` suffix.

### Fallback
Keep a minimal set of bundled copies as fallback for environments where neither filesystem nor fetch is available (WASI). But make them lazy-loaded, not imported at module init.

## Benefits
- Remove 2.1MB from repo
- Always in sync with installed TS version
- Faster initial load (no parsing 2.1MB of strings on import)
