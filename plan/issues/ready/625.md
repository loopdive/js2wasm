# #625 -- Wasm local.set type mismatch (552 CE)

## Status: review

552 tests fail with Wasm validation: local.set expects one type but receives another. Common patterns:
- `local.set expected (ref null N), found externref`
- `local.set expected f64, found i32`
- `local.set expected externref, found (ref null N)`

### Root cause
Type coercion not applied before storing to locals in certain code paths (async functions, generators, class methods with forward-declared types).

### Fix
Extend coerceType usage before local.set in affected codepaths.

## Complexity: M

## Implementation Summary

### What was done
1. Added `ref.cast_null` instruction to IR, binary emitter, WAT emitter, object emitter, and dead-elimination
2. Fixed `coerceType()` to handle ref-to-ref conversions with different type indices (previously returned silently, causing Wasm validation failures):
   - `(ref N) -> (ref M)` and `(ref_null N) -> (ref_null M)`: round-trip via extern.convert_any + any.convert_extern + ref.cast
   - `(ref N) -> (ref_null M)`: round-trip via extern.convert_any + any.convert_extern + ref.cast_null
   - `(ref_null N) -> (ref M)`: round-trip via extern.convert_any + any.convert_extern + ref.cast
3. Refactored `emitCoercedLocalSet` into `coerceForLocal` helper that also handles:
   - Different ref typeIdx with neither being AnyValue: widens local to externref + extern.convert_any (avoids runtime trap)
4. Added `emitCoercedLocalTee` for assignment expressions that need the value back on the stack
5. Removed redundant `coerceType` calls before `emitCoercedLocalSet` in variable declaration and for-loop init paths (was causing double-coercion bugs where the second coercion assumed the first succeeded)

### Files changed
- `src/ir/types.ts` -- added `ref.cast_null` to Instr union
- `src/emit/binary.ts` -- binary encoding for ref.cast_null
- `src/emit/wat.ts` -- WAT text for ref.cast_null
- `src/emit/object.ts` -- object file encoding for ref.cast_null
- `src/codegen/dead-elimination.ts` -- track ref.cast_null type usage
- `src/codegen/expressions.ts` -- coerceType fixes, emitCoercedLocalSet refactor, emitCoercedLocalTee
- `src/codegen/statements.ts` -- removed redundant coerceType calls before emitCoercedLocalSet
- `tests/local-set-type-mismatch.test.ts` -- new regression tests

### What worked
- The round-trip approach (extern.convert_any + any.convert_extern + ref.cast) handles struct type mismatches at the Wasm validation level
- Widening locals to externref in emitCoercedLocalSet avoids runtime traps for truly incompatible types
