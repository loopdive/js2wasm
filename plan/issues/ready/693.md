# #693 — Safe compilation speed optimizations

## Status: open

### 1. Skip WAT generation during test runs
`compile()` always generates WAT (string-heavy, ~40% of emit time). Add `skipWat: true` option — only generate binary. WAT only needed for debugging failed tests.

```typescript
// src/compiler.ts
if (!options?.skipWat) {
  result.wat = emitWat(mod);
}
```
**Impact**: ~40% faster emit phase. Zero risk — WAT is never used at runtime.

### 2. Reuse binary emit buffer
`WasmEncoder` allocates a new growing buffer per compilation. Pre-allocate 1MB and reuse:
```typescript
const sharedBuffer = new Uint8Array(1024 * 1024);
// In WasmEncoder: write into sharedBuffer, grow only if needed
// Reset position to 0 between compilations
```
**Impact**: Eliminates ~48K buffer allocations. Reduces GC pressure.

### 3. Persistent TypeScript program across tests
Currently each `compile()` call creates a new `ts.createProgram`. The TS compiler re-parses lib.d.ts (~50KB) every time. Keep a single host with cached libs:
```typescript
const cachedHost = ts.createCompilerHost(options);
// Pre-load lib files once
// For each test: createProgram with cached host, only parse the test source
```
**Impact**: ~30% faster type checking (lib parsing eliminated).

### 4. Preamble template caching
`wrapTest()` generates the same preamble (assert shims, $DONE, etc.) for every test with the same includes. Cache the preamble string keyed by `meta.includes.sort().join(",")`:
```typescript
const preambleCache = new Map<string, string>();
```
**Impact**: Eliminates redundant string building for 48K tests. Negligible memory cost.

### 5. Pre-allocated result objects
Instead of `new Object()` for each of 48K test results, pre-allocate and reuse:
```typescript
const resultObj = { file: "", category: "", status: "", error: "", timing: { ... } };
// Mutate fields instead of allocating new object
```
**Impact**: 48K fewer object allocations. Reduces GC pressure.

### 6. Test prioritization by historical timing
Sort tests so fast ones run first. Load timing data from previous run's JSONL:
```typescript
const timings = loadPreviousTimings(); // file → compileMs
tests.sort((a, b) => (timings[a] ?? 999) - (timings[b] ?? 999));
```
**Impact**: Early results within seconds. Slow tests overlap at end. No correctness risk.

### Combined estimate
| Optimization | Speedup | Risk |
|-------------|---------|------|
| skipWat | ~40% emit | Zero |
| Buffer reuse | ~10% emit | Zero |
| Cached TS host | ~30% typecheck | Zero |
| Preamble cache | ~5% wrap | Zero |
| Result pooling | ~2% GC | Zero |
| Test prioritization | Better UX | Zero |
| **Combined** | **~50% total** | **Zero** |

All are safe — no behavior changes, no shortcuts, no caching of results. Just eliminating redundant work.
