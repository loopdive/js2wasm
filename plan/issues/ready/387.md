# #387 -- Unsupported assignment target (342 CE)

## Status: open

342 tests fail with compile errors because the compiler encounters destructuring assignment to member expressions, computed properties, or other non-simple left-hand side patterns that are not yet supported.

## Details

JavaScript allows destructuring assignment to complex LHS targets:

```javascript
[obj.x, obj.y] = [1, 2];
({a: obj.prop} = source);
[arr[0], arr[1]] = [10, 20];
```

The compiler currently only handles simple identifier targets in destructuring patterns. Member expressions (property access, element access) and computed property names on the LHS are rejected.

Fix:
1. In the destructuring assignment codegen, detect `PropertyAccessExpression` and `ElementAccessExpression` as valid assignment targets
2. Emit the appropriate struct.set / array.set for each target kind
3. Handle nested patterns with mixed simple and complex targets
