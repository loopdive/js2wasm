# #526 — RuntimeError: dereferencing a null pointer (129 FAIL)

## Status: in-progress

129 tests fail with null pointer dereference -- `struct.get` or `struct.set` on a null reference. Need null guards before struct access in codegen.

## Complexity: M

## Root Cause

The `compileForOfDestructuring` function in `statements.ts` handled regular element
extraction and nested binding patterns for vec array destructuring, but had NO handler
for rest elements (`...x`). When `for (const [...x] of [values])` was compiled, the
rest element fell through to regular element access at index 0, producing a scalar
value instead of a vec struct reference. The local stayed null (default), and subsequent
`struct.get` for `x.length` or `x[i]` crashed with "dereferencing a null pointer".

Of the 118 originally failing tests (from the 2026-03-17 run), 108 were `rest-id` (89)
and `rest-ary-rest` (19) patterns that are fixed by adding rest element handling to the
for-of vec array destructuring path. The remaining 10 are:
- 3 `obj-rest` tests (skipped due to Object.getOwnPropertyDescriptor dependency)
- 4 `obj-ptrn-prop-obj-init` tests (nested object destructuring with defaults --
  null guard exists but does not apply default initializer)
- 3 `ary-ptrn-elem-ary-rest-iter` tests (nested array rest with initializer -- more
  complex pattern)

## Implementation Summary

Added rest element handling in `compileForOfDestructuring` for vec array destructuring
(lines ~2475-2525 in statements.ts). Follows the same pattern as the existing variable
destructuring rest handler (lines ~1010-1050):
1. Compute rest length: `max(0, src.length - i)`
2. Create new data array via `array.new_default`
3. Copy remaining elements via `array.copy`
4. Create new vec struct via `struct.new`
5. Store in the rest binding local

### Files changed
- `src/codegen/statements.ts` -- added rest element handler in `compileForOfDestructuring`
- `tests/equivalence/forof-rest-destructuring.test.ts` -- new test covering for-of rest patterns
