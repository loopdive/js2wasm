# #493 — Narrow prototype chain skip filter (233 tests)

## Status: open

233 tests skipped for "prototype chain not supported." Many of these tests just check `instanceof`, constructor identity, or static inheritance — which already work. The skip filter is too broad.

## Approach

Classify the 233 tests:
- Tests that check `instanceof` → should work (already implemented)
- Tests that check `obj.constructor === C` → needs constructor field on structs
- Tests that check `Object.getPrototypeOf(obj) === C.prototype` → needs compile-time prototype ref
- Tests that mutate `__proto__` or use `Object.setPrototypeOf` → genuinely unsupported

### Implementation
1. Narrow the skip filter: only skip tests that use `Object.setPrototypeOf`, `__proto__` assignment, or `Object.create` with prototype argument
2. Add `constructor` as a standard struct field pointing to the class
3. `Object.getPrototypeOf(obj)` → return the class's prototype object (compile-time known)
4. `C.prototype.isPrototypeOf(obj)` → compile as instanceof check

## Complexity: M

## Acceptance criteria
- [ ] Skip filter narrowed to only dynamic prototype mutation
- [ ] `instanceof` and `.constructor` work on class instances
- [ ] Unlock 100+ tests
