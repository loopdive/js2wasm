# Issue #345: Narrow Symbol.iterator skip filter

## Problem

702 tests are skipped because they declare `Symbol.iterator` in their test262
`features` metadata. However, 54 of these tests don't actually use `Symbol` in
their source code body -- they just test basic array destructuring with rest
patterns (e.g., `[, , ...x]`), which the compiler already handles correctly.

The test262 spec tags these tests with `Symbol.iterator` because the ECMAScript
specification defines array destructuring via the iterator protocol, but the
tests themselves only use plain arrays.

## Approach

Move `Symbol.iterator` out of the blanket `UNSUPPORTED_FEATURES` set and add a
source-level check: only skip tests tagged with `Symbol.iterator` if the test
body actually references `Symbol`.

## Acceptance Criteria

- [x] Symbol.iterator skip filter is narrowed to check source body
- [x] 54 tests that don't actually need Symbol semantics are unblocked
- [x] Tests that genuinely use Symbol.iterator in their code remain skipped
- [x] Equivalence tests pass

## Implementation Summary

### What was done
- Removed `Symbol.iterator` from the `UNSUPPORTED_FEATURES` set in `tests/test262-runner.ts`
- Added a source-level check: tests tagged with `Symbol.iterator` feature are only
  skipped if their source body (excluding metadata comments) contains `\bSymbol\b`
- This unblocks 54 tests that are array destructuring tests with rest patterns,
  which the compiler already handles correctly

### Analysis
- 687 tests total were skipped for `unsupported feature: Symbol.iterator`
- 633 of these genuinely use `Symbol` in their code body (custom iterables, assert.throws)
- 54 don't reference Symbol at all -- they just do basic array destructuring
- The existing for-of iterator source filter (lines 182-185) remains as a safety net
