# #828 -- Unexpected undefined AST node in compileExpression (154 compile errors)

## Problem

154 tests fail with compile error "L0:0 unexpected undefined AST node in compileExpression". The error at line 0, column 0 indicates the compiler encounters an undefined AST node where an expression is expected. Most affected tests are in `language/arguments-object/` with async private generator methods.

## Sample files

- test/language/arguments-object/cls-decl-async-private-gen-meth-static-args-trailing-comma-undefined.js
- test/language/arguments-object/cls-decl-async-private-gen-meth-static-args-trailing-comma-spread-operator.js
- test/language/arguments-object/cls-decl-async-private-gen-meth-static-args-trailing-comma-null.js
- test/language/arguments-object/cls-decl-async-private-gen-meth-args-trailing-comma-undefined.js
- test/language/arguments-object/cls-decl-async-private-gen-meth-args-trailing-comma-multiple.js

## Root cause

In `src/codegen/expressions.ts`, `compileExpression` receives an `undefined` node. This happens when the AST walker visits a node that should have children but one is missing. For async private generator methods, the method body AST may not be correctly extracted:

1. The TypeScript AST for `async *#method()` may have a different structure than regular methods
2. The class member visitor in `src/codegen/index.ts` (`collectClassDeclaration`) may not handle async private generator methods, passing `undefined` as the body
3. Alternatively, trailing comma handling in argument lists may produce an extra undefined element

## Suggested fix

1. Add a guard in `compileExpression` to emit a diagnostic instead of crashing on undefined AST nodes
2. In class method collection, handle async private generator methods the same as other method kinds
3. Check `node.body` extraction for all combinations of: async, generator, private, static

## Acceptance criteria

- 154 "unexpected undefined AST node" compile errors resolved
- Async private generator methods compile correctly
