# Issue #653: Timeout epidemic — 6,715 tests timeout

## Problem

The `SKIP_DISABLED = true` flag at the top of `shouldSkip()` in `tests/test262-runner.ts`
bypasses ALL filters, including critical safety filters that prevent:

1. **HANGING_TESTS** — known infinite compilation loops
2. **Temporal** — 3,381 tests for an API that does not exist in Wasm
3. **eval()** — dynamic code execution impossible in Wasm, causes compiler hangs
4. **new Function()** — dynamic code generation impossible in Wasm

Together these account for ~6,715 test timeouts that waste runner time.

## Fix

Move safety/impossibility filters ABOVE the `SKIP_DISABLED` early return so they
are always active. The `SKIP_DISABLED` flag should disable optional filters (feature
gates, harness includes, etc.) but NOT safety filters that prevent hangs or test
APIs that fundamentally cannot exist.

## Implementation Summary

- Moved HANGING_TESTS check above `SKIP_DISABLED` return
- Added Temporal feature check above `SKIP_DISABLED` return
- Moved eval() source check above `SKIP_DISABLED` return (removed duplicate below)
- Moved new Function() check above `SKIP_DISABLED` return (removed duplicate below)
- FIXTURE checks were already above the flag (prior fix)

- `tests/test262-runner.ts` — restructured `shouldSkip()` safety filter ordering
