# Issue #609: Test262 runner missing categories

## Problem
The test262 runner only processed 39,687/53,010 tests (74.9%). 13,323 tests were never run because `TEST_CATEGORIES` in `tests/test262-runner.ts` listed individual subdirectories instead of parent directories, and many top-level directories were entirely missing.

## Implementation Summary

### What was done
- Consolidated 228 granular subdirectory entries (e.g., `built-ins/Math/abs`, `built-ins/Math/ceil`) into 94 parent-level entries (e.g., `built-ins/Math`)
- Added all 22 missing `language/` directories (arguments-object, asi, block-scope, comments, eval-code, identifiers, literals, module-code, etc.)
- Added all 49 missing `built-ins/` directories (RegExp, Date, Function, Error, Symbol, TypedArray, Proxy, Reflect, DataView, etc.)
- Added `annexB/` tests (legacy browser behaviors, standard test262 format)

### What worked
- Using parent directories with `findTestFiles()` recursive walk covers all subdirectories automatically
- No code changes needed to the runner logic itself -- just expanding the category list

### Files changed
- `tests/test262-runner.ts` -- replaced TEST_CATEGORIES array (99 insertions, 246 deletions)

### Coverage improvement
- Before: ~39,687 tests covered
- After: ~48,103 tests covered (language + built-ins + annexB)
- Gain: +8,416 tests (+21.2%)
- Remaining uncovered: intl402/ (3,307 - internationalization) and staging/ (1,484 - experimental)
