# #822 -- f64/externref type mismatches (106 CE)

## Problem

106 tests fail with Wasm compile errors because the codegen emits operations expecting f64 but receives externref, or vice versa. The coercion was missing in several codegen paths.

## Error breakdown

| Pattern | Count | Description |
|---------|-------|-------------|
| `array.set[2] expected f64, found array.get of externref` | 60 | Storing externref into f64-typed arrays |
| `call[1] expected f64, found array.get of externref` | 17 | Bridge call with externref array element |
| `call[0] expected f64, found call of externref` | 9 | Call result type mismatch |
| `struct.new[0] expected f64, found if of externref` | 6 | Destructuring default if-expression |
| `local.set[0] expected f64, found array.get of externref` | 5 | Local assignment mismatch |
| `call[1] expected f64, found local.get of externref` | 4 | Local var as call argument |
| Other patterns | 5 | Various call/local.tee/struct.new mismatches |

## Implementation

Three-pronged fix:

### 1. New `fixArraySetCoercion` pass in stack-balance.ts (60 errors)

Added a new fixup pass that walks instruction streams for `array.set` instructions, infers the value type on the stack from the preceding instruction, and inserts coercion (e.g., `__unbox_number` call) when the value type doesn't match the array's declared element type.

### 2. `ensureCoercionImports` in stack-balance.ts (all patterns)

When `__unbox_number` and `__box_number` imports are missing from the module (because codegen didn't trigger `addUnionImports`), the stack-balance pass now adds them automatically. This involves:
- Adding the imports to the module
- Creating func types if needed
- Shifting all function indices in function bodies, exports, elements, and declaredFuncRefs

### 3. Bridge call coercion in array-methods.ts (17+ errors)

Fixed `buildBridgeCallInstrs` to detect when the array element type is `externref` but the bridge function expects `f64` (the `__call_1_f64` bridge). Now adds `__unbox_number` coercion in this case, with proper handling of function index shifts from `addUnionImports`.

### 4. Relaxed `inSubExpr` restriction in stack-balance.ts

Allowed `__unbox_number` coercion (externref->f64) through the `inSubExpr` guard in `fixCallArgTypesInBody`, since it's a safe single-value conversion that doesn't affect stack layout.

## Files changed

- `src/codegen/stack-balance.ts` — `fixArraySetCoercion`, `ensureCoercionImports`, relaxed `inSubExpr`
- `src/codegen/array-methods.ts` — `buildBridgeCallInstrs` externref->f64 coercion

## Test results

- 0 equivalence test regressions (54 failed / 1167 passed, unchanged)
- All 106 previously-failing compile errors are resolved
- Some tests now show runtime errors (illegal cast, null deref) instead of compile errors
