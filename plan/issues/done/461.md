# #461 — Array of structs: array.new_default fails for non-defaultable ref types

## Status: done

## Problem

Arrays of struct types (e.g., `Task[]` where Task is a class/interface) fail Wasm validation because `array.new_default` requires a defaultable element type. Non-nullable struct refs (`ref $Task`) are not defaultable — they have no null value.

## Details

```typescript
interface Task { id: number; callback: () => void; }
const tasks: Task[] = [];
tasks.push(newTask);
```

The compiler creates `(array (ref $Task))` but `array.new_default` needs `(array (ref null $Task))` since `ref $Task` has no default value.

## Fix

Use `ref_null` (nullable) element types for arrays of struct types:
- `(array (mut (ref null $Task)))` instead of `(array (mut (ref $Task)))`
- When reading elements, use `ref.as_non_null` if a non-null ref is needed

This is the standard WasmGC pattern for arrays of objects.

## Impact

Blocks React scheduler compilation (#455). Also affects any code using arrays of class instances or interface objects.

## Complexity: M

## Implementation Summary

**What was done:** Modified `getOrRegisterArrayType` in `src/codegen/index.ts` to convert non-nullable `ref` element types to `ref_null` when registering array types. This ensures `array.new_default` works for arrays of class/struct instances, since `ref null $T` is defaultable (defaults to null) while `ref $T` is not.

**What worked:** A single 4-line change in `getOrRegisterArrayType` was sufficient. The existing bounds-checked array get (`emitBoundsCheckedArrayGet`) already handles nullable ref types correctly, so no changes were needed in element access code.

**Files changed:**
- `src/codegen/index.ts` — `getOrRegisterArrayType`: convert `ref` to `ref_null` for array element types
- `tests/equivalence/array-of-structs.test.ts` — 4 new equivalence tests covering array literals, `new Array()`, `new Array(n)`, and iteration over arrays of class instances

**Tests now passing:** All 4 new array-of-structs equivalence tests pass. No regressions in existing array or class tests.
