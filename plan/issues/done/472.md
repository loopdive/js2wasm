# #472 -- Async/await support for test262 conformance

1,405 test262 tests were skipped because they use async functions or async iteration.

## Breakdown
- 1,311 tests skipped: "async flag" (test262 async harness)
- 94 tests skipped: "unsupported feature: async-iteration"

## Implementation Summary

**Already resolved** by prior issues:

- **#339** (`efd68865`): Compiled async functions synchronously (async function -> regular function, await -> identity). Removed the `async` flag skip filter from `shouldSkip()` in `tests/test262-runner.ts`, unblocking ~3,300 previously skipped tests. Added `$DONE` shim for async test completion callback.
- **#351** (`005f7ebc`): Removed `async-iteration` from `UNSUPPORTED_FEATURES`, unblocking tests using `for-await-of`.

Both skip filters (the `flags: [async]` check and the `async-iteration` feature filter) no longer exist in the current `tests/test262-runner.ts`. The stale results in `benchmarks/results/test262-results.jsonl` still show "async flag" as a skip reason because that file has not been regenerated since those changes were merged.

### What was verified
- `shouldSkip()` has no async flag check (lines 116-118 are just a comment confirming async tests are now compiled synchronously)
- `UNSUPPORTED_FEATURES` set (lines 67-92) does not include `async-iteration`
- `$DONE` shim exists in `wrapTest()` (line 1429-1435) for async-flagged test262 tests
- No other async-related skip filters exist in the codebase

### Files reviewed (no changes needed)
- `tests/test262-runner.ts` -- confirmed no async skip filters remain
