# Issue #287: Generator function compile errors -- yield in nested contexts

## Status: done

## Summary
Generator functions with yield in loops, conditionals, try/catch, and nested functions failed to compile. Fixed three root causes in the codegen.

## Implementation Summary

### What was done
1. **`collectGeneratorImports` only scanned top-level function declarations**: Added `hasGeneratorInTree()` recursive scanner that detects generator declarations, function expressions, and class methods.

2. **Generator function expressions not supported in closures**: When `FunctionExpression` has `asteriskToken`, register closure name in `ctx.generatorFunctions`, override return type to `externref`, set up `__gen_buffer`, wrap body in block, and call `__create_generator`.

3. **Generator methods in classes not handled**: Added generator detection in method type registration (externref return type) and body compilation (buffer setup, block wrapping).

- `src/codegen/index.ts`
- `src/codegen/expressions.ts`
- `tests/issue-287.test.ts` (13 tests)
