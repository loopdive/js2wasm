# Issue #271: Cannot find name -- missing harness or global declarations

## Status: done

## Summary
~55 tests fail with "Cannot find name X" where X is a test262 harness function (like `assert`) or a global that is not declared. Some tests reference globals that are not in the shim declarations, causing TypeScript compilation to fail.

## Category
Sprint 4 / Group D

## Complexity: S

## Scope
- Audit the test262 shim declarations for missing globals
- Add type declarations for `assert`, `print`, `$ERROR`, `$DONOTEVALUATE`
- Ensure harness functions are properly declared in the compilation context
- Update `tests/test262-runner.ts` shim setup

## Acceptance criteria
- Common harness globals are declared
- At least 20 compile errors resolved

## Implementation Summary

### What was done
After auditing the actual test262 results (20,886 tests), only 4 "Cannot find name" TS-level compile errors were found (3 for `BigInt`, 1 for `Reflect`). The rest of the harness globals (`$ERROR`, `$DONOTEVALUATE`, `Test262Error`) were already being textually replaced by `wrapTest` before compilation, so they never reached the TypeScript compiler.

Changes made:
1. **Added `BigInt` to UNSUPPORTED_FEATURES** -- tests with `features: [BigInt]` are now properly skipped
2. **Added Reflect code-level skip filter** -- tests referencing `Reflect` in executable code (not just feature tags) are now skipped
3. **Added `$ERROR(...)` replacement** -- `$ERROR(msg)` calls are replaced with `return 0;` (same pattern as `throw new Test262Error(...)` replacement)
4. **Added `print` function declaration** -- `function print(_x: number): void {}` added to the preamble so `print(x)` calls compile

### What worked
- The `$ERROR` regex replacement handles the simple `$ERROR("msg")` pattern well
- Adding `BigInt` to the features skip list cleanly handles all BigInt test cases
- The Reflect skip filter correctly strips comments before checking

### What didn't apply
- `$DONOTEVALUATE` was already handled (replaced with `return 0;` in `replaceOtherThrows`)
- `Test262Error` was already handled (replaced via `replaceThrowTest262Error` and removed via `removeAssertThrows`)
- `assert` methods were already handled (renamed to `assert_sameValue` etc. with shim implementations)

### Files changed
- `tests/test262-runner.ts` -- added BigInt to unsupported features, added Reflect skip filter, added $ERROR replacement, added print declaration
- `tests/issue-271.test.ts` -- 9 tests verifying all changes

### Tests now passing
- All 9 new tests in `tests/issue-271.test.ts`
- No regressions in existing test suite
