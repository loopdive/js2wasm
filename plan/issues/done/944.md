# #944 -- Revert #855 + fix #831 regressions (-1,514 pass)

## Problem

Main regressed from 17,782 pass to 16,268 pass (-1,514) between test262 runs `20260403-184313` and `20260404-010934`. Bisection identified two culprit commits.

## Bisect Results

### Culprit 1: `bae201ef` (#855 Promise resolution) — 1,451 regressions

The #855 change made Promise host calls return `externref` instead of unwrapping `Promise<T>`. This breaks:
- **666 tests**: `p.then is not a function` — returned value is opaque externref, not a host Promise
- **504 tests**: Wasm compile errors ("not enough arguments on the stack") — stack balance mismatches in async generator codegen from return type changes
- **215 tests**: `Cannot read properties of null (reading 'then')` — null Promise results
- **62 tests**: Other null deref on Promise results
- **4 tests**: `p.catch`/`p.finally` not a function

Additionally caused 625 `fail→compile_error` transitions (worsened error quality).

**Fix**: Full revert of `bae201ef`. The #855 issue needs a more careful implementation that doesn't break existing async/await/generator Promise handling.

### Culprit 2: `6c3cfc76` (#831 LHS validation) — 140 regressions

The whitelist-based `isInvalidAssignmentTarget` at `src/compiler.ts:466` is called for for-in/for-of LHS validation **without** `allowDestructuring=true`. This rejects valid destructuring patterns:
- `for (let [x] of ...)` — ArrayLiteralExpression falsely rejected
- `for (let {x} of ...)` — ObjectLiteralExpression falsely rejected
- `for await (let [x] of ...)` — same pattern

Additionally caused 293 `fail→compile_error` transitions.

**Fix**: One-line fix — add `allowDestructuring=true` to the for-in/for-of check at line 466.

## Implementation Plan

1. Revert commit `bae201ef` (#855 Promise resolution)
2. Fix `src/compiler.ts:466`: change `isInvalidAssignmentTarget(node.initializer as ts.Expression)` to `isInvalidAssignmentTarget(node.initializer as ts.Expression, true)`
3. Verify with sample tests from both regression categories
4. Run full equivalence + test262 to confirm pass count restored

## Acceptance Criteria

- Pass count ≥ 17,782 (baseline before regression)
- No new compile errors on for-in/for-of with destructuring
- Equivalence tests pass
