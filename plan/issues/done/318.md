# Issue 318: Infer parameter types from call-site arguments for untyped functions

## Status: DONE

## Problem
When functions don't have explicit type annotations on parameters, TypeScript's
checker returns `any` (since `noImplicitAny: false`). The codegen mapped these to
`externref`, causing type mismatches when the function body uses the parameters in
arithmetic or other typed operations. For example:

```ts
function add(a, b) { return a + b; }
add(3, 4); // should return 7
```

The params `a` and `b` were compiled as `externref` instead of `f64`, requiring
unnecessary boxing/unboxing and often producing incorrect results.

## Solution
1. **Call-site inference**: Added `inferParamTypeFromCallSites()` that scans all
   call expressions targeting a function and collects the Wasm types of arguments
   at each parameter position. If all call sites agree on a single concrete type,
   that type is used instead of `externref`.

2. **Applied in function scanning**: The inference runs during the function
   declaration collection phase (inside `collectDeclarations`) for both normal
   and generator functions, only when a parameter lacks an explicit type annotation
   and the checker reports `any`.

3. **Body compilation consistency**: Modified `compileFunctionBody` to read
   parameter types from the already-established function type signature, so the
   body compilation uses the same inferred types as the function declaration.

## Implementation Summary
- Added `inferParamTypeFromCallSites()` helper in `src/codegen/index.ts`
- Modified parameter scanning in the normal and generator function paths to use
  call-site inference for untyped `any` parameters
- Modified `compileFunctionBody` to prefer the function signature's param types
  over re-resolving from TS types
- Added 6 tests covering: single call site, multiple consistent call sites,
  boolean inference, mixed typed/untyped params, helper functions, arithmetic

## Files changed
- `src/codegen/index.ts` -- call-site inference function, scanning integration, body compilation fix
- `tests/issue-318.test.ts` -- 6 new tests
