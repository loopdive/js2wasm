# #125 — Object.defineProperty / property descriptors

## Status: done

## Implementation Summary

### What was done
Enhanced `compileObjectDefineProperty` in expressions.ts to handle more patterns:

1. **Struct name resolution fallbacks**: Added two fallback mechanisms when the TypeScript type is `any` or doesn't directly resolve to a struct:
   - Fallback 1: Resolve struct name from the local variable's Wasm type (for ref/ref_null locals)
   - Fallback 2: Resolve struct name from the variable's declaration initializer (for object literals assigned to `any`-typed vars), with a field-name matching heuristic for anonymous structs

2. **externref-to-struct coercion**: When the object is stored as externref but we know its struct type, emit `any.convert_extern` + `ref.cast` to recover the struct reference before `struct.set`.

3. **Nullable ref support for getters/setters**: Changed getter/setter parameter types from `ref` to `ref_null` so callers with nullable locals don't need `ref.as_non_null`.

4. **Descriptor flags**: `writable`, `enumerable`, `configurable` flags are accepted in the descriptor but ignored (not enforceable in WasmGC structs).

### What works
- Value descriptors on class instances and widened empty objects
- Descriptor flags accepted but ignored
- Getter/setter accessors on class instances (using `this`)
- `writable: true/false` with and without `value`
- Multiple `defineProperty` calls on same object

### Known limitations
- Getter/setter with closure-captured variables from outer scope not supported (needs closure capture in defineProperty-registered accessors)
- `Object.defineProperty` on `any`-typed non-empty object literals (`const obj: any = { x: 0 }`) falls through to externref path for reads
