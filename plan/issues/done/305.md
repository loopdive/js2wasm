# Issue #305: Runtime failures -- computed property names and types/reference

## Status: done

## Summary
3 test262 tests in language/types/reference (S8.7_A1, S8.7_A3, S8.7_A7) failed at runtime because:
1. `new Object()` compiled to `ref.null.extern` instead of a proper struct
2. Variables initialized from `ref_null` locals got `ref` type, causing Wasm validation errors
3. Function parameters for anonymous struct types used `ref` instead of `ref_null`

## Category
Sprint 5 / Group B

## Complexity: XS

## Scope
- Fixed `new Object()` to emit struct.new when widened properties are registered
- Fixed ref/ref_null type mismatches for variable assignments and function parameters
- Extended collectEmptyObjectWidening to handle `new Object()` pattern

## Acceptance criteria
- Both runtime failures resolved

## Implementation Summary

### What was done
1. **`collectEmptyObjectWidening`** (index.ts): Extended to match `var X = new Object()` in addition to `var X = {}`. This allows the widening pre-pass to detect property assignments on `new Object()` variables and register struct types.

2. **`compileNewExpression`** (expressions.ts): When `new Object()` is the initializer of a variable with widened properties, emit `struct.new` with default field values instead of `ref.null.extern`.

3. **Variable declaration ref_null normalization** (statements.ts): When a local variable is initialized from another variable that has `ref_null` type, widen the new local's type to `ref_null` to avoid Wasm validation errors.

4. **Function parameter normalization** (index.ts): For non-class struct parameters (anonymous types from interfaces/type literals), normalize from `ref` to `ref_null` so callers can pass nullable refs from widened variable declarations.

### What worked
- The targeted approach of only normalizing non-class struct types avoids regressions in class method dispatch (which uses hardcoded `ref` for `this` parameters).

### Files changed

- `src/codegen/index.ts` - `collectEmptyObjectWidening` for `new Object()`, function param normalization
- `src/codegen/statements.ts` - Variable declaration ref_null normalization
