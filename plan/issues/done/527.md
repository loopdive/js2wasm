# Issue #527: Add test262 script to package.json

## Problem

The `package.json` does not have a `test262` script, requiring users to type
`npx tsx scripts/run-test262.ts` manually every time.

## Solution

Add a `"test262"` entry to the `scripts` section of `package.json`:

```json
"test262": "npx tsx scripts/run-test262.ts"
```

This allows running `pnpm run test262 -- [categories...]` conveniently.

## Implementation Summary

- Added `"test262": "npx tsx scripts/run-test262.ts"` to package.json scripts
- Used `npx tsx` rather than bare `tsx` because pnpm does not expose tsx to the shell PATH by default
- Verified the script runs successfully with `pnpm run test262`
- Files changed: `package.json`
