# #148 — Element access (bracket notation) on struct types

## Status: open

## Problem
96 test262 compile errors: "Element access on struct type '__anon_0'". Tests use `obj[key]` where `obj` is an object literal and `key` is a string. The compiler only supports dot notation on structs.

Example:
```javascript
var obj = { a: 1, b: 2 };
var key = "a";
assert.sameValue(obj[key], 1);
```

## Fix
For string-literal bracket access (`obj["prop"]`), desugar to dot access at compile time. For dynamic string keys, this requires the hashmap fallback from #130 Phase 4.

## Tests blocked
~96 compile errors
