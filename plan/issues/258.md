# Issue #258: Unsupported call expression -- double/triple nested calls

## Status: review

## Summary
~270 tests fail with two "Unsupported call expression" errors, and ~36 with three. These represent nested call patterns like `fn()()`, `fn()()()`, or deeply chained calls. The call expression compiler needs to handle call expressions where the callee is itself a CallExpression.

## Category
Sprint 4 / Group A

## Complexity: M

## Scope
- Handle call expressions used as arguments to other call expressions
- Handle triple-nested and deeper call chains
- Update argument compilation in `src/codegen/expressions.ts`

## Acceptance criteria
- Nested call expressions (call as argument) compile
- At least 40 compile errors resolved

## Implementation Notes

### Problem
When the callee of a call expression is itself a `CallExpression` (e.g., `fn()(args)`), the compiler fell through to the "Unsupported call expression" error. The key challenge: functions returning closures use `extern.convert_any` to convert the GC struct ref to `externref`, losing type information needed for `call_ref`.

### Solution
Added `compileNestedCallExpression()` in `src/codegen/expressions.ts` and supporting infrastructure:

1. **`funcReturnsClosureType` map** (in `CodegenContext`): Records which closure struct type each function creates. Populated in `compileArrowAsClosure` when a closure is defined inside a function.

2. **Precise closure lookup**: When compiling `fn()()`, first tries `funcReturnsClosureType[fn]` to find the exact closure struct type. Falls back to signature matching against `closureInfoByTypeIdx` for triple-nesting or other patterns.

3. **externref -> ref conversion**: When the inner call returns `externref` (which wraps a GC struct ref), emits `any.convert_extern` + `ref.cast` to recover the typed closure struct reference.

4. **`emitClosureCallRef()` helper**: Shared code for emitting `call_ref` through a closure struct -- stores ref in temp local, pushes self param + args, extracts funcref from field 0, and calls.

### Files Changed
- `src/codegen/expressions.ts`: Added `compileNestedCallExpression()` and `emitClosureCallRef()`; records `funcReturnsClosureType` in `compileArrowAsClosure`
- `src/codegen/index.ts`: Added `funcReturnsClosureType: Map<string, number>` to `CodegenContext`
- `tests/equivalence.test.ts`: Added 4 test cases (fn()(), no-arg, triple nested, closure capture)
