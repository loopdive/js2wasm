# Issue #269: Setter return value diagnostic suppression

## Status: review

## Summary
~42 tests fail with "Setters cannot return a value" combined with ClassDeclaration/new expression errors. While #240 fixed setter return in some cases, the TypeScript diagnostic still blocks compilation in tests that have setters returning values inside class declarations in expression positions.

## Category
Sprint 4 / Group C

## Complexity: S

## Scope
- Suppress "Setters cannot return a value" diagnostic in allowJs mode
- Ensure setter bodies can contain return statements (value is discarded)
- Update diagnostic filtering in `src/compiler.ts`

## Acceptance criteria
- Setter with return statement compiles in allowJs mode
- At least 20 compile errors resolved

## Implementation notes
- TS diagnostic 2408 ("Setters cannot return a value") was already added to DOWNGRADE_DIAG_CODES in `src/compiler.ts` (line 375) by commit d1522c3
- Setter codegen in `src/codegen/index.ts` already sets `returnType: null` for setters, meaning return values are properly discarded
- Added equivalence test confirming setter with `return v` statement compiles and runs correctly
- No additional setter-related diagnostics needed (2808 and 4034-4037 are about module export accessibility, not relevant to test262)
