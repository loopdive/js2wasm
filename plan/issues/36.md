# Issue 36: Static Class Members

## Status: open

## Summary
Add support for `static` methods and `static` properties on classes.

## Motivation
Static members are very common in TypeScript — factory methods, constants, utility methods. Currently the compiler ignores the `static` modifier entirely.

## Design

### Static Methods
Static methods don't receive a `this` parameter. Compile to regular Wasm functions `ClassName_methodName` without the `(ref $struct)` first parameter.

In `collectClassDeclaration`:
1. Check for `StaticKeyword` modifier on method declarations
2. Register as a regular function (no self parameter)
3. Track in a `staticMethodSet` so `ClassName.method()` resolves correctly

### Static Properties
Map to Wasm module globals (`ClassName_propName`), initialized in `__module_init()`.

### Call/Access Resolution
- `MyClass.staticMethod(args)` → call `ClassName_methodName` (no self arg)
- `MyClass.staticProp` → `global.get $ClassName_propName`
- `MyClass.staticProp = val` → `global.set $ClassName_propName`

## Scope
- `src/codegen/index.ts` — detect static modifier, register functions/globals (~50 lines)
- `src/codegen/expressions.ts` — resolve `ClassName.member` for statics (~40 lines)
- `tests/static-members.test.ts` (~80 lines)

## Complexity: M

## Acceptance criteria
- `static method()` compiles without self parameter
- `ClassName.staticMethod()` calls correctly
- `static prop = value` creates module global with initialization
- `ClassName.staticProp` reads and writes the global
- Static and instance members coexist on the same class
- All existing tests still pass
