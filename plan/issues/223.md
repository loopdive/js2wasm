# #223 — Computed property names in class bodies

## Status: open

## Summary

88 compile errors from computed property names in class property declarations. These tests use `[Symbol.xxx]`, `[expr]`, or string/number expressions as class member names.

## Error Pattern

```
A computed property name in a class property declaration must have a simple literal type
```

## Root Cause

Class body compilation only handles static property names. Computed property names like `[Symbol.iterator]` or `[key]` aren't supported in class declarations.

## Fix

For computed property names that resolve to string/number literals at compile time, treat them as regular named properties. For dynamic computed names, either skip them (with a warning) or use the hashmap fallback if available.
