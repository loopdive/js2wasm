# Issue 90: Test262 coverage — built-ins/Array

## Summary

Add `built-ins/Array/prototype/*` method subcategories to the test262 runner. We already compile many array methods via host imports — this validates conformance.

## Motivation

Arrays are the most commonly used data structure. We support push, pop, indexOf, slice, map, filter, forEach, etc. — but have no test262 validation.

## Subcategories to add

- `push`, `pop`, `indexOf`, `lastIndexOf`, `includes`
- `slice`, `concat`, `join`
- `map`, `filter`, `forEach`, `every`, `some`, `find`, `findIndex`
- `fill`, `copyWithin`, `flat`, `flatMap`
- `length.js`, `constructor.js`

Skip (need Symbol.iterator or unsupported):
- `Symbol.iterator`, `Symbol.unscopables`
- `entries`, `keys`, `values` (iterator protocol)
- `sort`, `reverse`, `splice` (may need runtime support)

## Approach

1. Add paths to `TEST_CATEGORIES`
2. Run and triage
3. Fix issues — likely in host import shims (`runtime.ts`)

## Complexity

M — most methods already work, fixes likely in runtime shims

## Dependencies

None
