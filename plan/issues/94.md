# Issue 94: Test262 coverage — language/function-code

## Summary

Add `language/statements/function` and `language/expressions/function` to the test262 runner. Functions are a core language feature.

## Motivation

Function declarations, expressions, scoping, hoisting, and the arguments object are fundamental. We compile functions but have no test262 validation.

## Subcategories to add

- `language/statements/function`
- `language/expressions/function`
- `language/expressions/call`

## Approach

1. Add paths to `TEST_CATEGORIES`
2. Run and triage — likely many skips due to `arguments`, strict mode edge cases
3. Fix what we can

## Complexity

M — many tests will be filtered, but the ones that run validate core function semantics

## Dependencies

None
