# #499 -- with statement (84 tests)

## Status: wont-fix

## Analysis

The JSONL results reported 94 skipped tests, but 8 Promise tests were false positives
from a stale run (the skip filter was narrowed to strip string literals, which eliminates
matches on `"called with (..."` patterns). The actual count is 84 unique tests that
genuinely use the `with` statement.

All 84 tests are flagged `noStrict` (required since `with` is forbidden in strict mode).

### Test breakdown by category

| Category | Count | Nature |
| --- | --- | --- |
| compound-assignment (PutValue semantics) | 44 | `with(scope) { x *= 3 }` with getter+delete edge cases |
| function scope chain | 12 | Functions declared outside `with` must not see `with` scope |
| assignment (PutValue semantics) | 5 | Assignment target resolution with mid-expression mutation |
| prefix-increment | 4 | `++x` inside `with` with getter+delete |
| prefix-decrement | 4 | `--x` inside `with` with getter+delete |
| postfix-increment | 4 | `x++` inside `with` with getter+delete |
| postfix-decrement | 4 | `x--` inside `with` with getter+delete |
| delete inside with | 2 | `delete` behavior on environment object |
| call with base object | 1 | `this` binding via WithBaseObject |
| arrow/closure capture | 1 | Closure captures `with`-resolved identifier |
| try/catch interaction | 1 | `with` interacting with exception handling |
| yield inside with | 1 | Generator yield from `with` block |
| variable binding resolution | 1 | Binding resolution order with `delete` |

### Why static dispatch is infeasible

The original proposal suggested compiling `with(obj) { x }` to
`hasOwnProperty(obj, "x") ? obj.x : outer_x`. This does NOT work for the actual
test262 tests because:

1. **Getter/setter interception** (60+ tests): Scope objects use getters that call
   `delete this.x` mid-evaluation, testing that the Reference created by the initial
   lookup is preserved through PutValue. Static dispatch cannot replicate Reference
   semantics.

2. **Mid-expression scope mutation**: Tests like `x = (delete scope.x, 2)` verify that
   the assignment target was resolved BEFORE the right-hand side deletes the property.
   This requires Reference objects that capture the binding at lookup time.

3. **WithBaseObject for `this` binding**: `method()` inside `with(obj)` must bind
   `this` to `obj`. Static dispatch would need to detect all possible method calls and
   emit conditional `this` binding.

4. **Dynamic property creation**: Some tests verify that writing to an identifier inside
   `with` creates a new property on the scope object if the property was deleted
   mid-expression.

5. **Interaction with delete, generators, closures, try/catch**: Each adds another axis
   of complexity that makes static dispatch exponentially harder.

### Conclusion

The `with` statement fundamentally requires dynamic scope resolution with Reference
semantics. It is deprecated in strict mode (ES5+) and forbidden in ES modules. TypeScript
itself rejects `with` in strict mode. Implementing it would require a runtime object
environment record with dynamic property lookup -- contrary to the project's static
compilation approach.

The 84 tests are a permanent skip and do not represent a meaningful conformance gap for
a TypeScript-to-Wasm compiler.

## False positives fixed

The 8 Promise tests that were previously false-positive matches are no longer skipped by
the current filter (which strips string literals before matching `\bwith\s*\(`). No code
change needed -- the filter was already corrected in a prior commit.
