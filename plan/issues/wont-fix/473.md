# #473 -- Array.prototype method .call/.apply support

852 test262 tests are skipped because they call Array.prototype methods via
`.call()` or `.apply()` with an explicit `this` argument.

## Skip reason
"Array.prototype.method.call/apply" -- tests like:
```js
Array.prototype.push.call(obj, 1, 2, 3);
Array.prototype.indexOf.call(arrayLike, value);
```

## Investigation (2026-03-18)

Analyzed all ~1,215 test files matching the skip filter pattern.

### Receiver analysis

Extracted the first argument to `.call()` across all matching tests:

| Receiver variable | Count | What it actually is |
|---|---|---|
| `obj` | 1,137 | Plain object `{ 0: 11, length: 2 }` |
| `child` | 209 | Prototype-derived object |
| `lengthTracking` | 189 | TypedArray (resizable buffer) |
| `fixedLength` | 174 | TypedArray (resizable buffer) |
| `lengthTrackingWithOffset` | 155 | TypedArray (resizable buffer) |
| `fixedLengthWithOffset` | 148 | TypedArray (resizable buffer) |
| `false` / `true` | 79 / 72 | Boolean primitives |
| `arrayLike` | 74 | Plain object with `.length` |
| `arguments` | 56 | Arguments object |
| `arr` | 64 | Usually plain object despite the name |
| `undefined` / `null` | 27 / 25 | Null/undefined |
| `str` / `'abc'` | 27 / 12 | String primitives |
| Proxy, JSON, Math | 18 / 16 / 15 | Other non-array objects |

Zero tests use `.apply()` (only `.call()`).

### Real array receivers

Only 3 tests pass an actual `[]` array literal to `.call()`:

1. `concat/15.4.4.4-5-b-iii-3-b-1.js` -- uses `Object.defineProperty` (separately unsupported)
2. `lastIndexOf/calls-only-has-on-prototype-after-length-zeroed.js` -- uses Proxy (unsupported)
3. `indexOf/calls-only-has-on-prototype-after-length-zeroed.js` -- uses Proxy (unsupported)

All 3 depend on other unsupported features and would be caught by other skip filters even if this one were removed.

### Conclusion

The skip filter **cannot be meaningfully narrowed**. The fundamental issue is that test262
exercises the ES spec behavior of Array.prototype methods on generic "array-like" objects
(plain objects with `.length` and numeric indices). WasmGC typed arrays cannot serve as
receivers for arbitrary plain objects.

This is an inherent limitation of the WasmGC compilation approach, not a missing feature.
The skip filter is correctly scoped and should remain as-is.

## Original approach (not viable)

1. Implement `.call()` on function references -- dispatch to the underlying function
   with the first argument as `this`
2. Implement `.apply()` similarly but with array spreading
3. For Array.prototype methods specifically, handle the case where `thisArg` is
   an array-like object (has `.length` and numeric indices)
4. Remove the skip filter for these tests

This approach would not help because the tests fundamentally require treating plain objects
as arrays, which is incompatible with WasmGC typed array representation.

## Impact
- No tests can be unblocked by narrowing the filter
- The 852 skipped tests are correctly identified as unsupported
