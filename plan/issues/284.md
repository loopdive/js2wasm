# Issue #284: For-of compile errors -- destructuring and non-array iterables

## Status: review

## Summary
~141 tests fail in language/statements/for-of with compile errors. Many involve destructuring in the for-of head (`for (let [a, b] of arr)`), iterating over non-array iterables, or complex left-hand side assignment targets.

## Category
Sprint 4 / Group D

## Complexity: M

## Scope
- Support destructuring binding in for-of loop variable (`for (let {a, b} of arr)`)
- Handle for-of with array destructuring (`for (let [a, b] of arr)`)
- Support for-of with assignment patterns (without let/const/var)
- Update for-of compilation in `src/codegen/statements.ts`

## Acceptance criteria
- For-of with destructuring patterns compiles
- For-of with assignment targets compiles
- At least 30 compile errors resolved

## Implementation Notes

### Changes made in `src/codegen/statements.ts`:

1. **Nested object destructuring in for-of**: When `element.name` is an `ObjectBindingPattern` or `ArrayBindingPattern` (e.g. `for (const {a: {b}} of items)`), recursively call `compileForOfDestructuring` instead of crashing on `(element.name as ts.Identifier).text`.

2. **Nested array destructuring in for-of**: Same fix for array binding patterns like `for (const [[x, y]] of items)`.

3. **Rest elements in for-of array destructuring**: Handle `element.dotDotDotToken` for patterns like `for (const [first, ...rest] of arr)`. Creates a new vec struct from remaining elements using `array.new_default` + `array.copy`.

4. **Rest elements in regular array destructuring**: Same rest element support for `const [a, b, ...rest] = arr`.

5. **Nested patterns in regular destructuring**: Fixed `compileObjectDestructuring` and `compileArrayDestructuring` to handle nested binding patterns.

6. **Assignment patterns in for-of**: Support `for ([a, b] of arr)` and `for ({a, b} of arr)` without let/const/var, via new `compileForOfAssignDestructuring` function.

### Key implementation details:
- Rest length uses `select(srcLen-i, 0, srcLen>=i)` for max(0, srcLen-i)
- `array.copy` uses `dstTypeIdx`/`srcTypeIdx` (not `destTypeIdx`)
- Rest elements produce a new vec struct with the same type as the source array
