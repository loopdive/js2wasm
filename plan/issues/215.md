# #215 — Bug: Unary plus on empty string and boolean

## Status: open

## Summary

2 tests fail for unary plus operator edge cases.

## Failing Tests

- `11.4.6-2-1.js` — `+""` should be 0
- `S11.4.6_A3_T5.js` — unary plus on boolean

## Root Cause

Unary plus should coerce its operand to a number. Empty string should become 0, boolean true should become 1, false should become 0. The codegen may not handle string-to-number coercion for the unary plus operator.

## Fix

In unary plus codegen, if the operand is a string (externref), call `parseFloat` or `__unbox_number` to coerce. If boolean (i32), convert to f64 directly.
