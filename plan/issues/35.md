# Issue 35: Class Inheritance (extends, super)

## Status: open

## Summary
Add support for class inheritance: `extends` clause, `super()` constructor calls, `super.method()` calls, and method overriding. Map to Wasm GC sub-struct types.

## Motivation
Class inheritance is fundamental to TypeScript/JavaScript OOP. Nearly every non-trivial TS project uses it. Currently the compiler supports classes with constructors, instance methods, and properties, but no subclassing.

## Design

### Wasm GC Mapping
Wasm GC supports structural subtyping via `sub`:

```wat
(type $Animal (struct (field $name (ref extern))))
(type $Dog (sub $Animal (struct (field $name (ref extern)) (field $breed (ref extern)))))
```

A subtype struct must include all parent fields (in order) followed by child-specific fields.

### Struct Type Registration
In `collectClassDeclaration`, when a class has a `HeritageClause` with `extends`:
1. Resolve the parent class name
2. Look up the parent struct type index from `structMap`
3. Collect parent fields first, then child fields
4. Register the struct type with `sub` referencing the parent type index

### Constructor Compilation
When compiling `super(args)` calls inside a child constructor:
1. Detect `SuperKeyword` in call expressions
2. Compile the parent constructor body inline or call the parent constructor function
3. The child constructor creates a struct with all fields (parent + child)

### Method Resolution
- **Override**: If child defines a method with the same name as parent, the child version takes precedence
- **Inherited methods**: If child doesn't override, calls route to parent's method function

### super.method() Calls
When `super.method()` is encountered in a method body:
1. Detect `SuperKeyword` in property access expressions
2. Resolve the parent class name
3. Call `ParentClass_methodName` with `this` as first argument

## Scope
- `src/codegen/index.ts` — extend `collectClassDeclaration` for heritage clauses (~80 lines)
- `src/codegen/expressions.ts` — handle `SuperKeyword` in call and property access (~60 lines)
- `src/ir/types.ts` — extend `StructTypeDef` with optional `superTypeIdx` (~5 lines)
- `src/emit/binary.ts` — emit sub-type declarations (~30 lines)
- `src/emit/wat.ts` — emit sub-type in WAT (~10 lines)
- `tests/inheritance.test.ts` — test cases (~150 lines)

## Complexity: L

## Acceptance criteria
- `class Dog extends Animal` compiles with Dog struct as sub-type of Animal
- `super(args)` in constructor initializes parent fields
- `super.method()` calls parent implementation
- Method override: child method replaces parent method for child instances
- Inherited methods work on child instances
- Property access on inherited fields works correctly
- All existing tests still pass
