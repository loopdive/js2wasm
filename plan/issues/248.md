# Issue #248: Logical operators with object operands returning wrong values

## Status: in-progress

## Summary

2 tests fail: `S11.11.1_A4_T4.js` (logical AND) and `S11.11.2_A3_T4.js` (logical OR). These test logical operators with object/function/undefined operands and expect short-circuit evaluation to return the correct operand value (not just true/false).

## Root Cause

The logical AND/OR codegen may be coercing operands to boolean (i32) and returning 0/1 instead of returning the actual operand value. Per JS spec:
- `x && y` returns x if x is falsy, otherwise y
- `x || y` returns x if x is truthy, otherwise y

The return value should be the original operand, not a boolean.

## Scope

- `src/codegen/expressions.ts` -- logical AND/OR expression codegen
- Tests affected: 2 runtime failures

## Expected Impact

Fixes 2 runtime failures.

## Suggested Approach

1. Review the logical operator codegen to ensure it returns the operand value, not a boolean coercion
2. When operands are externref (objects), the truthiness check should use `ref.is_null` but the returned value should be the original externref
3. May need `select` instruction or block-based control flow to choose between left and right operand

## Acceptance Criteria

- [ ] `undefined && true` returns undefined (not 0 or false)
- [ ] `({}) || undefined` returns the object (not 1 or true)
- [ ] Both logical operator tests pass
