# #484 -- Well-known Symbol.species for constructor delegation (51 tests)

## Status: blocked

Blocked on: #487 (user Symbol as property key), #343 (prototype chain), and other prerequisites.

## Investigation (2026-03-18)

Analyzed all 51 test262 tests that use `Symbol.species` in source code. Key findings:

### All tests are caught by the existing Symbol source filter
The `shouldSkip` function in `test262-runner.ts` already skips these tests via the
`/\bSymbol\b/.test(body)` check (reason: "uses Symbol in source"). No separate skip
filter is needed for species.

### Pattern analysis
Every species test uses `Symbol.species` as a **computed property key**:
```javascript
a.constructor[Symbol.species] = Ctor;
// or
Object.defineProperty(a.constructor, Symbol.species, { ... });
```

This requires:
1. Well-known Symbol.species as a compile-time known value (#481 provides the i32 representation)
2. Using symbols as runtime property keys on objects (`obj[Symbol.species]`) -- requires #487
3. Prototype chain: `a.constructor` traversal -- requires #343
4. Many tests also use `Object.defineProperty` (defineProperty is wont-fix)

### Additional blockers within the 51 tests
- ~10 tests use `Object.defineProperty` -- would remain blocked even with Symbol support
- ~6 tests use `Reflect.construct` -- blocked by Reflect feature
- ~15 tests require prototype chain traversal (`a.constructor[Symbol.species]`)
- All 51 require symbol-keyed property access

### Breakdown by built-in
- Array.prototype.concat: 19 tests
- Array.prototype.slice: 10 tests
- Array.prototype.splice: 8 tests
- Array.prototype.map/filter/flat/flatMap: ~14 tests

### Conclusion
Symbol.species is genuinely blocked on full Symbol property key support (#487) and
prototype chain (#343). Even if those were implemented, the built-in Array methods
(concat, slice, splice, map, filter) would need to be taught the species protocol
internally. This is a large-scope feature with many prerequisites.

Moved to blocked/. Not actionable until #487 and #343 are done.

## Approach

`Symbol.species` is a static getter on built-in classes:
```javascript
class MyArray extends Array {
  static get [Symbol.species]() { return Array; }
}
```

This can be compiled as a static struct field `__symbol_species` that returns a constructor reference. Built-in methods check `this.constructor[Symbol.species]` to determine which constructor to use for the return value.

This is harder than Symbol.iterator because it requires:
1. Constructor references as first-class values
2. Built-in array/promise methods to check the species pattern
3. Runtime symbol-keyed property access
