# #140 — Bug: object computed property names not working at runtime

## Status: open

## Problem
Object literals with computed property names (e.g., `{ [expr]: value }`) compile but produce wrong results. 15 tests in `expr/object` fail with "returned 0".

Example:
```javascript
var x = { [1 + 2]: "three" };
assert.sameValue(x[3], "three");
```

## Root cause
Computed property names in object literals may not correctly map to struct fields when the key is a runtime expression.

## Tests blocked
~15 test262 failures in `expr/object`
