# #1160 — `%Array%.from` codegen error (~730 tests)

## Problem

730 test262 failures report:

```
Codegen error: %Array%.from requires that the property of the first arg...
```

Failures are spread across array method tests: `forEach`, `lastIndexOf`, `map`, `indexOf`, `findIndex`, etc. This indicates `Array.from` is being called during array method setup/polyfill with an argument whose property is not an integer type.

## Root cause hypothesis

`Array.from` codegen path validates that the iterable's length-like property resolves to an integer type. When called with an `arrayLike` object from test harness (e.g., `{0: x, length: 1}`), the `length` property may resolve as `externref` instead of `f64`/`i32`, triggering the error before execution begins.

## Investigation

1. Find `Array.from` codegen path in `src/codegen/array-methods.ts` or `src/codegen/expressions.ts`
2. Identify where "property of the first arg must be integer" is thrown
3. Check what types are being passed at the failing call sites
4. Sample 5 failing tests to confirm they share this root cause

- 730 → 0 `%Array%.from requires that the property` errors
