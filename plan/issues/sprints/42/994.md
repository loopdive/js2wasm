## Root cause analysis

The issue is NOT a compile timeout — compilation takes <1s. The actual failure was **infinite recursion at runtime** causing `WebAssembly.instantiate` to hang forever (reported as "runtime timeout (10s)" or "Maximum call stack size exceeded").

### Root cause: missing static accessor dispatch for `this.#getter` in static methods

In `src/codegen/property-access.ts`, the `compilePropertyAccess` function handles `this` in static methods (where there's no `this` local parameter). It checks `ctx.staticProps` for static field access, but did NOT check `ctx.classAccessorSet` for static accessor (getter/setter) access.

When `this.#f` (a static private getter) was compiled inside a static method:
1. `this` keyword detected with no local `this` parameter → static method path
2. `ctx.staticProps.get("C___priv_f")` → not found (it's a getter, not a field)
3. Fell through to the generic property access fallthrough code
4. Generic fallthrough emitted code that recursively called the enclosing static method

This produced the WAT:
```wat
(func $C_access
  call $__get_undefined  ;; compile `this` as undefined
  return_call $C_access  ;; recursive call to self (infinite loop!)
)
```

### Fix

Added an accessor check after the `staticProps` check in the `this` + static method path (property-access.ts line 945). When `ctx.classAccessorSet` has the accessor key, emit a dummy struct instance and call the getter function directly.

After fix, the WAT is:
```wat
(func $C_access
  i32.const 1            ;; __tag for C
  struct.new $C          ;; dummy struct instance
  call $C_get___priv_f   ;; call getter with dummy self
  return
)
```
