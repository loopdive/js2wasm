# #1206 — Build-pages script misses top-level paths for benchmark JSONs

## Problem

Browser console on the deployed landing page (`https://loopdive.github.io/js2wasm/`)
shows 4 404 errors:

```
GET /frame-nav-sync.js                                          404
GET /benchmarks/results/size-benchmarks.json                    404
GET /benchmarks/results/playground-benchmark-sidebar.json       404
GET /benchmarks/results/loadtime-benchmarks.json                404
```

Cause: `scripts/build-pages.js` lines 311-332 copy these JSONs to
`PLAYGROUND_BENCHMARKS_RESULTS_DIR` (`pages-dist/playground/benchmarks/
results/`) but NOT to the top-level `pages-dist/benchmarks/results/`.
The landing page's `index.html` (lines 2903, 2910, 4264-4286) requests
them at the top-level path.

`frame-nav-sync.js` is in the repo root (`/workspace/frame-nav-sync.js`)
but isn't copied to pages-dist at all — it's referenced from the
landing page header for iframe nav.

Verified live status:

| Path | Top-level | Playground |
|------|-----------|------------|
| `size-benchmarks.json` | 404 | 200 |
| `loadtime-benchmarks.json` | 404 | 200 |
| `playground-benchmark-sidebar.json` | 404 | 404 |
| `frame-nav-sync.js` (root) | 404 | n/a |

(The third is missing from BOTH — likely missing from
`public/benchmarks/results/` source too. Audit needed.)

## Fix

Patch `scripts/build-pages.js`:

```diff
+ // Top-level benchmark JSONs for the landing page
+ copyFileIfExists(
+   join(PUBLIC_BENCH, "size-benchmarks.json"),
+   join(PAGES_DIST, "benchmarks", "results", "size-benchmarks.json"),
+ );
+ copyFileIfExists(
+   join(PUBLIC_BENCH, "loadtime-benchmarks.json"),
+   join(PAGES_DIST, "benchmarks", "results", "loadtime-benchmarks.json"),
+ );
+ copyFileIfExists(
+   join(PUBLIC_BENCH, "playground-benchmark-sidebar.json"),
+   join(PAGES_DIST, "benchmarks", "results", "playground-benchmark-sidebar.json"),
+ );
+
+ // Iframe nav-sync glue (referenced from landing page header)
+ copyFileIfExists(
+   join(ROOT, "frame-nav-sync.js"),
+   join(PAGES_DIST, "frame-nav-sync.js"),
+ );
```

If `playground-benchmark-sidebar.json` doesn't exist in
`public/benchmarks/results/`, it must be generated by an earlier
benchmark step or removed from the landing page references — audit
which.

## Acceptance criteria

1. After deploy, all 4 paths return 200 OR the landing page's
   references are removed (whichever the audit shows is correct).
2. No JS console 404s on `https://loopdive.github.io/js2wasm/` or
   `#goals` / `#dashboard` sub-pages.
3. The `playground-benchmark-sidebar.json` source file is either
   confirmed-existing or filed as a separate bug.

## Out of scope

- Custom domain `js2.loopdive.com` — that's #1188 (separate domain
  setup; orthogonal to whether the JSONs are at the right paths).
- Restructuring the public/ vs pages-dist/ layout. Surgical fix only.

## Time budget

~30 minutes. 5-line script change + a manual deploy verification.

## Test Results

Implementation in `scripts/build-pages.js`: replaced the three
`copyFileIfExists(PUBLIC_BENCH, ...)` calls (which were silent no-ops —
the JSONs live in `benchmarks/results/`, not `public/benchmarks/results/`)
with a loop that resolves each source from `BENCHMARKS_RESULTS_DIR` first,
falls back to `PUBLIC_BENCH`, and copies into BOTH the top-level
`pages-dist/benchmarks/results/` and the playground subdir.

Also added a `copyFileIfExists(ROOT/frame-nav-sync.js, PAGES_DIST/...)`.

Verified after `pnpm run build:pages`:

| Path | Status |
|------|--------|
| `dist/pages/frame-nav-sync.js` | present |
| `dist/pages/benchmarks/results/size-benchmarks.json` | present |
| `dist/pages/benchmarks/results/loadtime-benchmarks.json` | present |
| `dist/pages/benchmarks/results/playground-benchmark-sidebar.json` | present |
| `dist/pages/playground/benchmarks/results/{size,loadtime,sidebar}.json` | present |

Audit note: `playground-benchmark-sidebar.json` is committed in
`benchmarks/results/`, so acceptance criterion 3 is satisfied — the
source file exists; landing page references resolve.
