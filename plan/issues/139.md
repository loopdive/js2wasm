# #139 — Bug: valueOf/toString coercion on arithmetic operators

## Status: open

## Problem
Arithmetic operators (`+`, `-`, `*`, `/`, `%`) do not call `valueOf()` on object operands. Tests with objects that define `valueOf` fail for arithmetic.

## Tests blocked
~8 test262 failures across `expr/subtraction` (1), `expr/multiplication` (1), `expr/modulus` (3), `expr/unary-plus` (2), `expr/unary-minus` (1)

## Fix
In binary/unary arithmetic codegen, when operand is a struct type with `valueOf`, emit a call to it before the arithmetic instruction. Extends #138 approach to arithmetic.
