# Issue 92: Test262 coverage — language/types (coercion)

## Summary

Add `language/types/*` subcategories to the test262 runner. Type coercion rules (ToNumber, ToBoolean, ToString) are implicit in almost all JS operations.

## Motivation

JavaScript's implicit type coercion is a major source of bugs and behavioral differences. Validating our ToNumber/ToBoolean semantics against test262 ensures correctness.

## Subcategories to add

- `language/types/number` — numeric edge cases
- `language/types/boolean` — truthy/falsy semantics
- `language/types/null`, `language/types/undefined`
- `language/types/string` — string coercion
- `language/types/object` — ToPrimitive / valueOf / toString
- `language/types/reference` — reference semantics

## Approach

1. Add paths to `TEST_CATEGORIES`
2. Run and triage — object coercion tests will likely need `tryStaticToNumber` or runtime valueOf support
3. Fix exposed issues

## Complexity

M — many tests will be skipped due to feature requirements, but the ones that run will validate core semantics

## Dependencies

None
