# Issue 98: Proper ToInt32 modular arithmetic for bitwise operations

## Summary

JS bitwise operators use `ToInt32` which wraps values modulo 2^32. Our compiler uses `i32.trunc_sat_f64_s` which clamps instead of wrapping.

## Example

```js
(4294967296 << 0)   // JS: 0        (wraps mod 2^32)
                    // Ours: 2147483647 (clamped to i32 max)

(-2147483649 << 0)  // JS: 2147483647 (wraps)
                    // Ours: -2147483648 (clamped to i32 min)
```

## Fix

Emit the full ToInt32 sequence before bitwise ops:
```wasm
;; ToInt32(x): reduce x modulo 2^32 then truncate
local.get $x
local.get $x
f64.const 4294967296.0
f64.div
f64.floor
f64.const 4294967296.0
f64.mul
f64.sub
i32.trunc_sat_f64_s
```

Or create a helper wasm function `__toInt32` to avoid code bloat.
