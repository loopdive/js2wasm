# Issue #246: For-of object destructuring -- TypeError on primitive coercion

## Status: in-progress

## Summary

5 tests in `language/statements/for-of/dstr/` fail with "TypeError: Cannot convert object to primitive value". These tests destructure objects from arrays in for-of loops, like `for (var {x = 1} of [{y: 2}]) {}`. The runtime attempts to convert the object to a primitive value, which fails.

## Root Cause

When destructuring objects in for-of, the codegen may be attempting to coerce the iterated value to a primitive type before destructuring. The destructuring should operate directly on the struct/object ref, not convert it to a primitive first.

## Scope

- `src/codegen/statements.ts` -- for-of destructuring handling
- Tests affected: 5 runtime failures

## Expected Impact

Fixes 5 runtime failures.

## Suggested Approach

1. In for-of destructuring, when the loop variable is a destructuring pattern:
   - Get the iterated element as a struct ref (not externref)
   - Destructure by accessing struct fields directly
   - Handle missing properties with defaults
2. Ensure no implicit ToPrimitive coercion occurs on the iterated object
3. The issue may be in how the iterated value is typed -- if it is externref, the destructuring code tries to convert it

## Acceptance Criteria

- [ ] For-of with object destructuring does not coerce objects to primitives
- [ ] Missing properties use default values correctly
- [ ] All 5 for-of destructuring tests pass
