# #216 — Bug: Modulus and arithmetic edge cases with special values

## Status: open

## Summary

3 modulus tests and 2 multiplication/subtraction tests fail with special floating-point values (NaN, Infinity, -0).

## Failing Tests

- `S11.5.3_A4_T2.js` — modulus with Infinity
- `S11.5.3_A4_T5.js` — modulus with -0
- `S11.5.3_A4_T6.js` — modulus edge cases
- `S11.6.2_A3_T1.5.js` — subtraction with special values
- `S11.5.1_A3_T1.5.js` — multiplication with special values

## Root Cause

The arithmetic operations may not properly handle IEEE 754 special values. Wasm's `f64.rem_s` (if used for modulus) may behave differently from JS's `%` operator for special values.

## Fix

Use the correct Wasm instruction for modulus that matches JS semantics. For edge cases, may need to call a host import `__js_modulus` or add special-value checks.
