# Issue 42: Comma Operator

## Status: open

## Summary
Support the comma operator `(expr1, expr2)` which evaluates both expressions and returns the last value.

## Motivation
Appears in minified code, for-loop multi-initializers, and some functional patterns. Without support, valid TS code silently fails.

## Design

The comma operator is a `BinaryExpression` with `CommaToken`:
1. Compile the left operand
2. If left produces a value, emit `drop`
3. Compile the right operand (becomes the result)

Chains `(a, b, c)` parse as `((a, b), c)` so recursive handling works.

## Scope
- `src/codegen/expressions.ts` — add `CommaToken` case (~10 lines)
- `tests/comma-operator.test.ts` (~30 lines)

## Complexity: XS

## Acceptance criteria
- `(a(), b())` evaluates both, returns `b()`
- Side effects of left operand are preserved
- Chained commas work
- All existing tests still pass
