# Issue #282: Variable declaration compile errors -- complex initializers

## Status: review

## Summary
~60 tests fail in language/statements/variable with compile errors. These involve variable declarations with complex initializers (function expressions, class expressions, conditional expressions) or multiple declarations in one statement with type mismatches.

## Category
Sprint 4 / Group D

## Complexity: S

## Scope
- Handle variable declarations with class expression initializers
- Support multiple declarations with different inferred types
- Handle variable declarations in for-loop heads with complex patterns
- Update variable declaration compilation in `src/codegen/statements.ts`

## Acceptance criteria
- Variable declarations with complex initializers compile
- Multiple declarations with different types compile
- At least 20 compile errors resolved

## Implementation notes

### Root cause
The `collectStringLiterals` function in `src/codegen/index.ts` only scanned function
declaration bodies for string literals to pre-register. Top-level statements (variable
declarations, expression statements, if/for/while, class declarations, etc.) were never
scanned, so any string literal appearing only in top-level code would fail with
"String literal not registered" during compilation.

### Fix
Extended the scanning loop in `collectStringLiterals` to also visit all non-function
top-level statements (variable statements, expression statements, control flow, class
declarations, etc.). Applied the same fix to `collectForInStringLiterals`,
`collectInExprStringLiterals`, and `collectObjectMethodStringLiterals` for consistency.

### Files changed
- `src/codegen/index.ts` — extended `collectStringLiterals` and three related collector
  functions to scan top-level statements
- `tests/equivalence.test.ts` — added 5 tests for complex variable initializers

### Tests
5 new equivalence tests:
- top-level string literal in variable initializer
- top-level template literal in variable initializer
- top-level conditional with string branches
- multiple variable declarations with mixed types
- top-level typeof in variable initializer
