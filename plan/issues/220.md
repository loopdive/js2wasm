# #220 — ClassDeclaration in more statement positions

## Status: open

## Summary

686 compile errors from "Unsupported statement: ClassDeclaration" — Sprint 1 (#150) added support for ClassDeclaration inside `if` blocks, but many remain in other positions: `switch` cases, labeled statements, `for` loop bodies, `while`/`do-while` bodies, `try`/`catch`/`finally` blocks, etc.

## Error Pattern

```
Unsupported statement: ClassDeclaration
```

## Root Cause

`compileStatement` in statements.ts only handles ClassDeclaration at the top level or inside `if` blocks (added in Sprint 1). Other statement positions still hit the "Unsupported statement" fallback.

## Fix

Extend `compileNestedClassDeclaration` (from Sprint 1) to be called from all statement positions in `compileStatement`. Add the ClassDeclaration case to the main switch statement so it works regardless of parent context.

## Complexity: S (< 150 lines)

## Tests: ~686 compile errors → potentially hundreds of new passes
