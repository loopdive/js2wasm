# #217 — Bug: While/do-while loop with string conditions

## Status: open

## Summary

4 while/do-while tests fail where the loop condition involves string concatenation or string truthiness.

## Failing Tests

- `S12.6.2_A4_T1.js` — while with string condition
- `S12.6.2_A4_T5.js` — while with string mutation
- `S12.6.1_A4_T1.js` — do-while with string condition
- `S12.6.1_A4_T5.js` — do-while with string mutation

## Root Cause

Loop conditions involving strings may not properly coerce to boolean. An empty string should be falsy, a non-empty string should be truthy. The condition evaluation may not handle externref string truthiness.

## Fix

In while/do-while condition compilation, if the condition type is string (externref), emit a truthiness check (string length > 0 or string !== "").
