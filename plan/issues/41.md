# Issue #41: typeof as standalone expression

## Problem

Currently `typeof` only works inside binary comparisons like `typeof x === "number"` (via `compileTypeofComparison`). Using `typeof` as a standalone expression — e.g. `const t = typeof x;` or `return typeof x;` — causes an "Unsupported expression: TypeOfExpression" error.

## Solution

Handle `ts.isTypeOfExpression` in `compileExpressionInner` to emit:

- **Statically known types**: Emit the corresponding string constant from the pool.
  - `f64` (number) → `"number"`
  - `i32` (boolean) → `"boolean"`
  - `externref` string → `"string"`
  - Struct ref → `"object"`
  - `void`/`undefined` → `"undefined"`
- **Unknown / union types (externref)**: Call a `__typeof` host import that returns the JS `typeof` string at runtime.

### Files changed

1. **`src/codegen/expressions.ts`** — Add `ts.isTypeOfExpression` case in `compileExpressionInner`
2. **`src/codegen/index.ts`** — Register typeof result strings in `collectStringLiterals`; add `__typeof` import in `addUnionImports`
3. **`src/runtime.ts`** — Add `__typeof` host function
4. **`src/compiler.ts`** — Add `__typeof` JS glue generation

### Tests

- `tests/typeof-expr.test.ts` — Standalone typeof for number, boolean, string, object, union params
