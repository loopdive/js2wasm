# Issue 38: instanceof Operator

## Status: open

## Summary
Compile the `instanceof` operator for runtime type checking of class instances.

## Motivation
`instanceof` is a common runtime type check used in type guards, error handling, and control flow. Currently not supported.

## Design

### Wasm GC Approach
Use `ref.test` to check if a reference is a subtype:

```wat
ref.test (ref $TargetStruct)  ;; returns i32 (0 or 1)
```

This handles inheritance — a `Dog` instance passes `ref.test (ref $Animal)`.

### Compilation
For `BinaryExpression` with `InstanceOfKeyword`:
1. Compile the left operand (value to test)
2. Resolve the right operand (class name) to its struct type index
3. Emit `ref.test (ref $structTypeIdx)`

### Type Narrowing
After `if (x instanceof Foo)`, emit `ref.cast (ref $Foo)` in the true branch so struct field access works.

### External Classes
For `declare class` types (externref), fall back to a host import that performs JS `instanceof`.

## Scope
- `src/codegen/expressions.ts` — handle `InstanceOfKeyword` + narrowing (~60 lines)
- `src/emit/binary.ts` — ensure `ref.test` opcode emits correctly (~5 lines)
- `tests/instanceof.test.ts` (~60 lines)

## Complexity: S

## Acceptance criteria
- `x instanceof MyClass` returns boolean
- Works with class inheritance (Dog instanceof Animal → true)
- Type narrowing after instanceof allows property access
- All existing tests still pass
