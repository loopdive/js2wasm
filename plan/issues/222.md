# #222 — Unknown identifier scope resolution improvements

## Status: open

## Summary

~200 compile errors from "Unknown identifier" for variables like `x`, `y`, `v`, `length` that should be in scope. Many of these come from destructuring targets, function parameters, or variables declared in patterns not fully supported.

## Error Pattern

```
Unknown identifier: x
Unknown identifier: y
Unknown identifier: v
Unknown identifier: length
```

## Root Cause

The identifier resolution in expressions.ts doesn't find variables declared in:
1. Destructuring patterns (`var {x, y} = obj`)
2. Catch clause parameters (`catch(e)`)
3. For-of/for-in variable declarations
4. Variables declared in nested blocks that should be hoisted

## Fix

Improve variable declaration scanning to cover all declaration sites. Ensure destructuring declarations register all bound names. Check catch clause parameter registration.
