# #208 -- Bug: comparison operator failures with valueOf coercion and edge cases

## Status: backlog

## Summary

24 test262 runtime failures across comparison operators. Includes both valueOf coercion failures and BigInt cross-type comparison issues.

## Motivation

Failure breakdown:
- `greater-than`: 7 failures (4 valueOf coercion + 2 BigInt + 1 edge case)
- `less-than-or-equal`: 6 failures (4 valueOf + 2 BigInt)
- `less-than`: 3 failures (1 edge case + 2 BigInt)
- `greater-than-or-equal`: 2 failures (BigInt)
- `strict-equals`: 2 failures (BigInt)
- `strict-does-not-equals`: 2 failures (BigInt)
- `equals`: 1 failure (BigInt)
- `does-not-equals`: 1 failure (BigInt)

Two distinct root causes:
1. valueOf/toString coercion not called during relational comparison (~8 tests)
2. BigInt vs Number/Boolean cross-type comparison (~12 tests, overlaps with #174)

## Scope
- `src/codegen/expressions.ts` -- comparison operator codegen
- valueOf/toString dispatch for object operands
- BigInt (i64) vs f64 safe conversion

## Complexity
M

## Acceptance criteria
- [ ] Relational operators call valueOf/toString on object operands
- [ ] BigInt comparisons with Number do not trap on Infinity/NaN
- [ ] BigInt === Number returns false (different types)
- [ ] 20+ test262 comparison failures fixed
