# Issue 37: Getter/Setter Properties on User-Defined Classes

## Status: open

## Summary
Compile `get` and `set` accessor declarations in user-defined classes. Currently only handled for external `declare class` types.

## Motivation
Getters and setters are commonly used for computed properties, validation, and encapsulation.

## Design

### Compilation Strategy
- `get propName()` → function `ClassName_get_propName(self: ref $struct): returnType`
- `set propName(val)` → function `ClassName_set_propName(self: ref $struct, val: valType): void`

### Collection Phase
In `collectClassDeclaration`:
1. Detect `GetAccessorDeclaration` and `SetAccessorDeclaration`
2. Register accessor functions with `ClassName_get_propName` / `ClassName_set_propName`
3. Track accessor names in `classAccessorSet` so property access routes through the function

### Expression Compilation
- `obj.prop` where `prop` is a getter → `call $ClassName_get_propName` instead of `struct.get`
- `obj.prop = val` where `prop` has a setter → `call $ClassName_set_propName` instead of `struct.set`

## Scope
- `src/codegen/index.ts` — collect accessor declarations (~40 lines)
- `src/codegen/expressions.ts` — route access/assignment through accessors (~30 lines)
- `tests/getters-setters.test.ts` (~80 lines)

## Complexity: M

## Acceptance criteria
- `get prop()` compiles and is called on property read
- `set prop(val)` compiles and is called on property write
- Getter can compute derived values from struct fields
- Mixed getter-only and getter+setter properties work
- All existing tests still pass
