# Issue #207: Unicode escape sequences in property names

**Status**: review

## Problem

42 test262 tests fail because property names use Unicode escape sequences like
`obj.bre\u0061k = 42`. While TypeScript's parser correctly resolves these escapes
(AST `.text` returns `"break"`), the test262 runner's regex-based preprocessing
in `wrapTest()` operates on raw source text. Patterns like switch widening,
assert routing, and variable detection can be confused by `\uNNNN` sequences
in identifiers.

## Solution

Added `resolveUnicodeEscapes()` in `tests/test262-runner.ts` that converts
`\uNNNN` sequences in identifier positions to their actual characters before
any regex preprocessing. The function splits source into string-literal and
non-string-literal segments, only transforming escapes outside string literals.

This runs as the first transform in `wrapTest()`, after comment stripping but
before all regex-based transforms.

## Files changed

- `tests/test262-runner.ts` — added `resolveUnicodeEscapes()` function and
  call in `wrapTest()`

## Tests

No direct equivalence tests (test262 conformance improvement).
