# Issue #278: Cannot destructure -- not a known struct type

## Status: review

## Summary
~44 tests fail with "Cannot destructure: not a known struct type: __object" errors. Destructuring assignment or declaration on values whose type is not a recognized struct fails. The codegen needs to handle destructuring on dynamically-typed values or externref objects.

## Category
Sprint 4 / Group D

## Complexity: M

## Scope
- Support destructuring on externref/unknown-typed values via property access fallback
- Handle object destructuring where the source type is inferred as a generic object
- Support destructuring in catch clauses and for-of iterations
- Update destructuring compilation in `src/codegen/statements.ts`

## Acceptance criteria
- Destructuring on dynamically-typed values compiles
- At least 25 compile errors resolved

## Implementation Notes

Two changes were made:

1. **`src/codegen/statements.ts` (compileObjectDestructuring)**: Added `ensureStructForType` auto-registration for anonymous object types (`__type`/`__object`), matching the logic already present in `expressions.ts`. Also added externref-to-struct casting via `any.convert_extern` + `ref.cast` when the compiled expression returns externref but a struct is expected.

2. **`src/codegen/expressions.ts` (destructuring assignment)**: Added the same externref-to-struct casting for destructuring assignment patterns (`{ x, y } = expr`).

Both changes follow the existing pattern: when TypeScript infers a type with symbol name `__object` or `__type`, the compiler now calls `ensureStructForType` to register a struct definition, then uses that struct for field extraction via `struct.get`.
