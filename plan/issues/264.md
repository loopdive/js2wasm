# Issue #264: Element access (bracket notation) on struct types

## Status: review

## Summary
88 tests fail with "Element access on struct type X". The compiler does not support bracket notation (`obj[key]`) on struct types. This needs a dynamic field lookup based on the key value, mapping string keys to struct field indices at runtime or compile time.

## Category
Sprint 4 / Group C

## Complexity: M

## Scope
- Support `obj[key]` where obj is a struct and key is a string literal or variable
- For string literal keys, resolve at compile time to struct.get
- For variable keys, generate a switch/dispatch on known field names
- Update element access in `src/codegen/expressions.ts`

## Acceptance criteria
- `obj["field"]` compiles to struct.get
- `obj[variable]` compiles with runtime dispatch for known fields
- At least 60 compile errors resolved

## Implementation Notes

### Changes in `src/codegen/expressions.ts`

**compileElementAccess** (line ~8360): When the bracket key is not resolvable at compile time (not a string/numeric literal, not a const variable, and `resolveConstantExpression` returns undefined), the code now generates a runtime dispatch instead of erroring:
1. Saves the struct ref and key expression to locals
2. Finds the `wasm:js-string` `equals` function for string comparison
3. Builds a nested if/else chain: for each struct field, compares the key against the field name string constant and emits `struct.get` for the matching field
4. Default branch returns 0 (f64) or 0 (i32) depending on field types
5. All field values are coerced to a common result type (f64 if any field is f64)

**compileElementAssignment** (line ~3352): Extended the string-literal-only bracket assignment to also handle numeric literals, const variable keys, and `resolveConstantExpression` results, matching the same resolution logic used in `compileElementAccess`.

### Tests added (5 new tests in `tests/equivalence.test.ts`)
- Bracket notation with string literal keys
- Bracket notation with const variable keys
- Dynamic key via function parameter (runtime dispatch)
- Bracket notation assignment
- Let variable key with reassignment
