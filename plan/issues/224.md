# #224 — Prefix/postfix increment on member expressions

## Status: open

## Summary

~50 compile errors from prefix/postfix increment/decrement on member expressions like `++obj.x` or `arr[i]++`.

## Error Pattern

```
Unsupported prefix increment target
Unsupported postfix increment target
```

## Root Cause

The increment/decrement operators only handle simple identifiers. Member expressions (`obj.prop++`, `arr[i]++`) require a read-modify-write sequence on a property access.

## Fix

Extend the prefix/postfix increment codegen to handle PropertyAccessExpression and ElementAccessExpression targets:
1. Emit the struct.get/element access to read the current value
2. Add/subtract 1
3. Emit the struct.set/element store to write back
4. Return the pre/post value depending on prefix vs postfix

## Complexity: S (< 150 lines)
