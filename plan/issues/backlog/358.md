# #358 -- Dynamic import support

## Status: open

200 skipped tests need `import()` dynamic import expressions. This requires runtime module loading, which is fundamentally challenging in a pure Wasm environment without a JS host.

## Details

Dynamic `import()` returns a Promise that resolves to the module namespace object:
```javascript
const mod = await import('./module.js');
mod.default; // default export
mod.namedExport; // named export
```

Key challenges:
1. **Runtime module resolution** -- Wasm has no built-in module loading; modules are linked at instantiation time
2. **Returns a Promise** -- depends on async/await support (#339)
3. **Dynamic specifiers** -- the module path can be a runtime expression

Possible approaches:
- Pre-link all possible modules at compile time and use a dispatch table
- Implement a minimal module loader in Wasm (very complex)

## Complexity: XL
