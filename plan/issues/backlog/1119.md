# #1119 — SingleNameBinding with class expression default (fn-name-class cluster)

## Problem

**71 test262 null_deref failures** (and 24 runtime_error + 13 type_error + 77 assertion_fail in the broader cluster — 203 total) match the pattern `*-fn-name-class.js`. These tests verify ES2015 spec rule 13.3.3.6 step 6d — the SingleNameBinding "anonymous function naming" rule, where a class expression used as the default value of a destructuring binding gets its `.name` set from the binding identifier.

Example (`test/language/statements/class/dstr/meth-static-ary-ptrn-elem-id-init-fn-name-class.js`):

```js
class C {
  static method([cls = class {}]) {
    // cls.name === 'cls' — per 13.3.3.6 step 6d "If Initializer is present and
    // IsAnonymousFunctionDefinition(Initializer) is true, then SetFunctionName(v, bindingId)"
  }
}
```

Current behavior: null_deref (or type_error / runtime_error depending on call path) during the destructuring-default path.

## Scope

Files matching `*-fn-name-class.js` in `test/language/**/dstr/`:
- 203 total in current baseline (`benchmarks/results/test262-results-20260418-151655.jsonl`)
- 71 `null_deref`, 24 `runtime_error`, 13 `type_error`, 77 `assertion_fail`, 6 `compile_error`, 12 already pass

## Why this needs an architect spec

This is ES2015 feature work, not a mechanical bug fix:
1. Anonymous function naming is spec'd separately from destructuring — 13.3.3.6 step 6d (array SingleNameBinding) and 13.3.3.7 step 5e (object SingleNameBinding)
2. Requires synthesizing a `.name` at the destructuring site, derived from the binding identifier text
3. Must detect `IsAnonymousFunctionDefinition(Initializer)` — applies to `class {}`, `function () {}`, arrow `() => {}`, computed-name classes/functions
4. Multiple touch points: `destructureParamArray`, `destructureParamObject`, `compileForOfDestructuring`, variable destructuring — all need consistent handling
5. Interacts with class expression codegen — we need to know whether the RHS of `=` is an anonymous class expression at destructuring time, then wire its `.name` property to the binding id
