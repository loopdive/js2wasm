# #370 -- WeakMap and WeakSet

## Status: open

10 WeakMap + 8 WeakSet tests. Needs weak reference semantics.

## Details

```javascript
var wm = new WeakMap();
var key = {};
wm.set(key, 'value');
wm.get(key); // 'value'
wm.has(key); // true
// key is held weakly -- if no other references exist, it can be GC'd

var ws = new WeakSet();
ws.add(key);
ws.has(key); // true
```

Implementation challenges:
1. **Weak references** -- WasmGC has `ref.null` but no weak reference primitives
2. **GC interaction** -- weak collections should not prevent GC of their keys
3. **Identity-based keys** -- keys must be objects (not primitives)

Possible approaches:
- Use Wasm GC weak references if available in the target engine
- Fall back to strong references (functionally correct but no GC benefit)
- Use `extern.externalize` + host weak ref support (violates no-JS-host principle)
