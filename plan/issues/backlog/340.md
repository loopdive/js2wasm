# #340 -- Error throwing and try/catch/finally

## Status: open

1,337 tests need exception handling. Currently throw causes hangs. Requires Wasm exception handling proposal (try/catch/throw instructions) or a sentinel-value error propagation scheme.

## Details

The Wasm exception handling proposal adds:
- `try`/`catch`/`catch_all` blocks
- `throw` instruction with a tag
- `rethrow` instruction
- Exception tags declared in the module

Alternative approach without the exception handling proposal:
- Use a global error sentinel + explicit checks after each call
- Every function returns a tagged union: success value or error
- This is slower but works on all Wasm runtimes

The exception handling proposal is preferred as it is now Phase 4 and supported in V8, SpiderMonkey, and WebKit.
