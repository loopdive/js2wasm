# #124 — delete operator via undefined sentinel (232 tests)

## Status: review

Previously labeled "won't implement" — reassessed as achievable.

## Approach

`delete obj.prop` can't remove a WasmGC struct field, but can set it to a sentinel:

1. `delete obj.prop` → `struct.set obj $prop (ref.null)` or undefined sentinel
2. `obj.prop` after delete → returns undefined (check sentinel)
3. `"prop" in obj` → false if sentinel
4. `hasOwnProperty("prop")` → false if sentinel

Works for own properties. Prototype chain deletion not applicable (we don't have dynamic prototype chains).

## Complexity: M

## Implementation Summary

### What was done
- Enhanced `compileDeleteExpression` in `src/codegen/expressions.ts` to actually set struct fields to sentinel values instead of just returning true as a no-op
- Added `emitDeleteSentinel` helper that emits the appropriate sentinel value for each Wasm field type:
  - ref/ref_null: `ref.null` of the field's type index
  - externref: `ref.null.extern`
  - f64: `NaN`
  - i32: `0`
- For `delete obj.prop` (PropertyAccessExpression): resolves struct type and field via `resolveStructName` + `ctx.structFields`, sets the field to the sentinel, returns true
- For `delete obj["prop"]` (ElementAccessExpression with string literal key): same approach
- Falls back to the existing no-op behavior for unresolvable targets
- Only sets sentinel on mutable fields (immutable fields cannot be deleted)

### What worked
- ref.null sentinel for externref/ref fields naturally produces `undefined` on subsequent reads since the null-guard paths already handle ref.null
- The NaN sentinel for f64 matches JS behavior (accessing a deleted numeric property returns undefined, which coerces to NaN in numeric context)

### Files changed
- `src/codegen/expressions.ts` — `compileDeleteExpression`, new `emitDeleteSentinel` helper
- `tests/equivalence/delete-sentinel.test.ts` — 7 new equivalence tests

### Tests now passing
- 7 new tests covering: delete returns true, property becomes undefined after delete, other properties preserved, string property deletion, conditional delete, multiple deletes on same object
