# #376 -- Decorator syntax support

## Status: open

10+ tests use class decorators (@decorator), which is a TC39 stage 3 proposal now part of the ES2024 spec.

## Details

```javascript
function logged(target, context) {
  // decorator logic
}

@logged
class MyClass {
  @logged
  method() {}
}
```

Decorators can be applied to:
1. Classes
2. Class methods
3. Class fields
4. Class accessors (get/set)

Each decorator is a function that receives the decorated value and a context object, and can return a replacement value.

Implementation is complex because:
- TypeScript has legacy decorators (experimentalDecorators) and TC39 decorators (different semantics)
- Need to determine which decorator semantics the test262 tests use
- Requires runtime metadata objects and function wrapping
