# #346 -- Object.defineProperty support

## Status: open

421 tests use Object.defineProperty for property descriptors (configurable, enumerable, writable, get/set). Requires a property descriptor model on top of WasmGC structs.

## Details

Object.defineProperty allows fine-grained control over property behavior:
```javascript
Object.defineProperty(obj, "x", {
  value: 42,
  writable: false,
  enumerable: true,
  configurable: false
});
```

This requires:
1. A property descriptor struct: `{value, writable, enumerable, configurable, get, set}`
2. Per-property metadata storage (separate from the value itself)
3. Property access must check descriptors: writable controls assignment, get/set intercept access
4. Object.getOwnPropertyDescriptor to read descriptors back

For WasmGC structs, this is fundamentally challenging because struct fields have no metadata. Options:
- Side table mapping field names to descriptors
- Wrapper struct per property with metadata fields
- Only support compile-time-known descriptors (const/readonly)
