# #350 -- Symbol type (general)

## Status: open

398 tests need Symbol support (Symbol(), Symbol.toPrimitive, Symbol.species, Symbol.replace, Symbol.match). Core language feature needed for many other features.

## Details

Symbols are unique, immutable primitive values used as property keys and for language protocols.

Implementation approach:
1. **Representation**: Symbols as unique i32 IDs. A global counter allocates new IDs.
2. **Well-known symbols**: Fixed IDs for Symbol.iterator (1), Symbol.toPrimitive (2), Symbol.species (3), etc.
3. **Symbol()**: Allocates a new unique ID
4. **Symbol.for()**: Global symbol registry (string -> ID mapping)
5. **Symbol as property key**: Requires property access to support both string and symbol keys
6. **typeof**: Returns "symbol"

The biggest challenge is using symbols as property keys on WasmGC structs. Options:
- Side table mapping (object_id, symbol_id) -> value
- Special symbol-keyed fields in structs (for well-known symbols only)
- Hybrid: compile-time known symbols use struct fields, runtime symbols use side table
