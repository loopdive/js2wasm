# #339 -- Async function and await support

## Status: open

3,317 tests require async functions and await. Needs either Wasm async support (stack switching proposal) or a CPS transform to compile async/await into state machines in Wasm.

## Details

Async functions return Promises and use await to suspend execution. Possible approaches:
1. **Wasm stack switching proposal** — not yet widely available
2. **CPS/state machine transform** — compile async functions into a state machine with explicit continuation passing, similar to how TypeScript compiles async to generators
3. **JS host bridge** — violates project principles (no JS host dependency)

The state machine approach is most feasible: each await point becomes a state, and the function is compiled as a loop over states with a continuation struct.
