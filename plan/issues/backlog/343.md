# #343 -- Prototype chain support

## Status: open

763 tests rely on prototype chain lookups (Object.getPrototypeOf, instanceof chains, inherited properties). Fundamental change to the object model — WasmGC structs are flat, not chained.

## Details

JavaScript's prototype chain allows property lookup to traverse parent objects. WasmGC structs are statically typed and flat, making prototype chains non-trivial.

Possible approaches:
1. **Prototype pointer in every struct** — add a `$proto` field pointing to the parent struct. Property lookup becomes a chain of struct.get + null checks. Expensive at runtime.
2. **Flattened inheritance** — at compile time, merge all prototype properties into a single struct. Works for static class hierarchies but not for dynamic prototype manipulation.
3. **Vtable approach** — each struct has a vtable pointer; method dispatch goes through the vtable. Similar to how class-based languages compile.

For test262 compliance, approach 1 or 3 is needed since tests dynamically manipulate prototypes.
