# #365 -- Collection mutation during for-of

## Status: open

15 tests mutate collections (Map/Set/Array) during for-of iteration. Currently this causes hangs (infinite loops) because the iterator doesn't account for size changes.

## Details

```javascript
var set = new Set([1, 2, 3]);
var visited = [];
for (var x of set) {
  visited.push(x);
  if (x === 2) set.delete(1); // mutate during iteration
}
// visited should be [1, 2, 3]
```

Per spec, Set/Map iterators must visit elements added during iteration and skip deleted elements, using an internal cursor that advances through insertion order.

Implementation challenges:
1. Iterator state must track a cursor position, not just an index
2. Deletions must not invalidate the cursor
3. Additions after the cursor must be visited
4. This requires linked-list or tombstone-based iteration rather than simple index-based loops
