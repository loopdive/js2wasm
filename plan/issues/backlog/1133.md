# #1133 — `__any_strict_eq` string tag comparison uses identity, not content

## Problem

The `__any_strict_eq` helper in `src/codegen/any-helpers.ts` handles tag-by-tag comparison
for AnyValue-boxed values. For tag 5 (string), it falls through to the catch-all branch
which checks `tag < 2`. Since 5 ≥ 2, this returns `false` for any two string AnyValue
boxes that aren't the exact same struct reference.

Per §7.2.16 (Strict Equality Comparison) step 3, SameValueNonNumber for strings must
compare code-unit sequences, not reference identity.

## Reproduction

```typescript
let a: any = "hello";
let b: any = "hello";
export function test(): boolean {
  return a === b;  // returns false, should return true
}
```

## Spec reference

**§7.2.16** Strict Equality Comparison:
- Step 1: If Type(x) ≠ Type(y), return false
- Step 2: If x is a Number, return Number::equal(x, y)
- Step 3: Return SameValueNonNumber(x, y)

**§7.2.12** SameValueNonNumber for strings: compare code-unit sequences.

## Code location

`src/codegen/any-helpers.ts`, `__any_strict_eq` helper (~line 831-949).
The tag-5 case needs an explicit branch that calls `wasm:js-string equals` (or
`__str_equals` for native strings) on the `externval` fields of both AnyValue structs.

## Fix sketch

Add a tag-5 branch before the catch-all:
```
// tag 5 (string): compare externval fields via string equals
local.get $tagA
i32.const 5
i32.eq
if (result i32)
  local.get $a
  struct.get $AnyValue $externval
  local.get $b
  struct.get $AnyValue $externval
  call $wasm_js_string_equals   ;; or __str_equals for native
else
  ;; catch-all: null/undefined (tag < 2) → both same tag means equal
  local.get $tagA
  i32.const 2
  i32.lt_s
end
```

The same fix should also be applied to `__any_eq` (loose equality same-tag path),
which has the identical bug for tag 5.
