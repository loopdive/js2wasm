# Issue 44: Source Maps

## Status: open

## Summary
Generate source maps that map compiled Wasm instructions back to TypeScript source locations for debugging in browser DevTools.

## Motivation
Zero debugging support exists for compiled Wasm output. Developers must read raw WAT to find problems. Source maps are essential for practical use.

## Design

### Phase 1: Source Position Tracking
Extend `Instr` in `ir/types.ts`:
```typescript
sourcePos?: { file: string; line: number; column: number };
```

During codegen, attach positions from TS AST nodes using `sourceFile.getLineAndCharacterOfPosition()`.

### Phase 2: Binary Offset Tracking
During binary emission, record byte offset of each instruction with a source position.

### Phase 3: Source Map Generation
Generate Source Map v3 JSON from the mapping table. Emit `sourceMappingURL` custom section in the Wasm binary.

### API
```typescript
compile(source, { sourceMap: true }) // returns sourceMap field in CompileResult
```

## Scope
- `src/ir/types.ts` — add `sourcePos` to `Instr` (~5 lines)
- `src/codegen/` — attach source positions (~100 lines across files)
- `src/emit/binary.ts` — track byte offsets (~40 lines)
- `src/emit/sourcemap.ts` — VLQ encoding, source map generation (~150 lines)
- `src/compiler.ts` — wire option through pipeline (~20 lines)
- `tests/sourcemap.test.ts` (~80 lines)

## Complexity: L

## Acceptance criteria
- `compile(source, { sourceMap: true })` returns valid source map JSON
- Source map maps Wasm byte offsets to TS line/column positions
- Wasm binary includes `sourceMappingURL` custom section
- Key constructs have source positions
- All existing tests still pass
