# Issue #259: ClassDeclaration in block/nested scope positions

## Status: review

## Summary
~85 tests fail with just "Unsupported statement: ClassDeclaration". These are class declarations inside blocks (if/else, try/catch, loops) or other nested positions that the codegen currently rejects. The compiler should hoist these class declarations or handle them as block-scoped bindings.

## Category
Sprint 4 / Group B

## Complexity: M

## Scope
- Support `class Foo {}` inside block statements (if, for, try, etc.)
- Treat block-scoped class declarations as local bindings
- Update `compileStatement` in `src/codegen/statements.ts`

## Acceptance criteria
- Class declarations in block scope compile
- At least 30 compile errors resolved

## Implementation Notes

Investigation revealed that the core functionality for nested class declarations was already
implemented in `compileNestedClassDeclaration()` in `src/codegen/statements.ts` (lines 2075-2101),
likely as part of issue #234 in Sprint 3. The handler:

1. Catches `ClassDeclaration` nodes in `compileStatement` (line 265)
2. Calls `collectClassDeclaration()` to register struct type, constructor, and method stubs
3. Calls `compileClassBodies()` to compile constructor and method bodies immediately

All common patterns already work:
- Class inside if/else blocks
- Class inside for/while loops
- Class inside switch case blocks
- Class at function scope (not in a sub-block)
- Multiple classes in same scope
- Class inheritance in nested scope

The remaining 6 test262 "Unsupported statement: ClassDeclaration" errors are all related to
`class-static-block` features (ES2022 `static { ... }` blocks), which is a separate concern.

Added 8 new equivalence tests to verify coverage of all block-scope class patterns.
