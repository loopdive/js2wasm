# #221 — More call expression patterns

## Status: open

## Summary

~200 of the 724 "Unsupported call expression" errors come from patterns that should be addressable: method calls on known objects, chained calls, and calls with complex argument expressions.

## Error Pattern

```
Unsupported call expression
```

## Root Cause

The `compileCallExpression` function doesn't handle all call patterns. Common unsupported patterns include:
- `obj.method.call(thisArg, ...)` — `.call()` on methods
- `(0, foo)()` — indirect call via comma operator
- Method calls on returned values (chained)

## Fix

Add support for the most common unsupported call patterns. Focus on patterns that appear in test262 most frequently:
1. `.call()` and `.apply()` patterns
2. Comma operator indirect calls `(0, foo)()`
3. Method calls on known built-in return types
