# Issue #211: Function.length property and void function comparison

## Status: review

## Problem

7 function statement tests fail in test262:

1. **S13.2.1_A9.1_T1.js** - `__func() !== undefined` where func returns void (function decl)
2. **S13.2.1_A9.1_T2.js** - Same, but with function expression
3. **S13.2.1_A9_T1.js** - `__func() !== undefined` where func has bare `return;` (function decl)
4. **S13.2.1_A9_T2.js** - Same, but with function expression
5. **S13.2.1_A6_T3.js** - Pass-by-value semantics for string params
6. **S13_A19_T1.js** - `typeof __decl` before var/function hoisting
7. **params-dflt-ref-arguments.js** - Default params referencing arguments object

## Root Causes

### Void function comparison to undefined (tests 1-4)
When a void function call result was compared to `null`/`undefined` using `===` or `!==`, the compiler treated the null comparison as applying to a non-externref type (valType was null from VOID_RESULT). It pushed `i32.const 1` for `!== undefined`, but should push `i32.const 0` since void functions return undefined.

### Function.length property (general improvement)
No support existed for accessing the `.length` property on a function-typed identifier to get the formal parameter count. Added static resolution using TypeScript call signatures.

## Changes

### src/codegen/expressions.ts
1. **Void-to-undefined comparison fix**: In the null/undefined comparison shortcut, added explicit handling for `valType === null` (VOID_RESULT case). Void expressions are semantically undefined, so `void_expr === undefined` is true and `void_expr !== undefined` is false.

2. **Function.length property**: Added a handler in `compilePropertyAccess` that checks if the object type has call signatures when `.length` is accessed. Returns the formal parameter count (excluding rest params) as a compile-time f64 constant.

### tests/equivalence.test.ts
Added 4 equivalence tests:
- void function returns undefined (=== undefined)
- void function with bare return returns undefined
- function.length property (formal parameter count)
- pass-by-value semantics for primitives

## Not Fixed (out of scope)
- **S13_A19_T1.js** - Requires global-scope function hoisting before var declarations
- **S13.2.1_A6_T3.js** - Likely related to string `+=` in function params (separate issue)
- **params-dflt-ref-arguments.js** - Requires arguments object in default param initializers
