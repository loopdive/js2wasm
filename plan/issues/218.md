# #218 — Bug: Boolean() with assignment expressions as argument

## Status: open

## Summary

2 Boolean() tests fail when the argument is an assignment expression like `Boolean(x = 0)`.

## Failing Tests

- `S15.6.1.1_A1_T5.js` — `Boolean(x = 0)` should return false, `Boolean(x = 1)` should return true

## Root Cause

The Boolean() function codegen may not handle assignment expressions as arguments. The assignment `x = 0` should first assign, then the result (0) should be coerced to boolean.

## Fix

Ensure assignment expressions used as Boolean() arguments are properly compiled — the assignment side-effect should execute and the resulting value should be used for boolean coercion.
