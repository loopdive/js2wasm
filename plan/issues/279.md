# Issue #279: Arrow function compile errors -- parameter and body patterns

## Status: review

## Summary
~93 tests fail with compile errors in arrow function contexts. These include arrow functions with destructuring parameters, default values in complex positions, or arrow functions used in contexts where the return type is ambiguous (void vs value).

## Category
Sprint 4 / Group A

## Complexity: M

## Scope
- Support arrow functions with destructuring parameters
- Handle arrow functions with complex default parameter expressions
- Fix return type inference for arrow functions in various contexts
- Update arrow function compilation in `src/codegen/expressions.ts`

## Acceptance criteria
- Arrow functions with destructuring params compile
- Arrow functions with complex defaults compile
- At least 30 compile errors resolved

## Implementation notes

### Changes made

1. **`src/codegen/expressions.ts`**:
   - Added `emitArrowParamDestructuring()` function: emits destructuring code for arrow/function expression parameters that use binding patterns (array or object) or default values. Called after param locals are registered but before body compilation, for both `compileArrowAsClosure` and `compileArrowAsCallback`.
   - Fixed parameter type resolution for destructuring params: when a param has an `ArrayBindingPattern` with a type annotation like `number[]`, resolve the vec struct type from the annotation rather than using `getTypeAtLocation(p)` which returns `{}` for destructured params.
   - For untyped destructuring params (test262 JS code), default to `externref`.
   - Registered closures in `closureMap` for assignment expressions (`f = (...) => {...}`) in addition to variable declarations.
   - Fixed `compileClosureCall` to downcast from `externref` to the closure struct ref type when the local was widened (e.g., `let f: (x: number) => number` stores as externref).

2. **`src/codegen/index.ts`**:
   - Added `emitFuncParamDestructuring()` function: handles array and object binding pattern parameters for regular function declarations. Emits struct.get/array.get instructions to extract binding names from the parameter value.

### Test results
- 7 new equivalence tests added and passing
- Arrow-function test262 compile errors reduced from 199 to ~58
- "Unknown identifier" errors for destructuring params reduced from 63 to ~14 (remaining are nested destructuring patterns)
- No regressions in existing tests
