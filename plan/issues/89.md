# Issue 89: Test262 coverage — language/statements

## Summary

Add `language/statements/*` subcategories to the test262 runner and fix compiler failures. Control flow statements are used in every non-trivial program.

## Motivation

We compile if, while, for, switch, do-while, break, continue, try/catch, and return — but have no test262 conformance validation for any of them.

## Subcategories to add

- `if`, `while`, `do-while`, `for`, `for-in`, `for-of`
- `switch`, `break`, `continue`, `return`
- `block`, `empty`, `expression`, `variable`
- `labeled`, `throw`, `try`

Skip:
- `async-function`, `async-generator`, `generators`
- `class`, `const`, `let` (block scoping edge cases)
- `with`, `debugger`, `for-await-of`, `await-using`, `using`

## Approach

1. Add subcategory paths to `TEST_CATEGORIES`
2. Run, triage, fix compiler bugs
3. Update filters as needed

## Complexity

L — control flow edge cases likely expose bugs

## Dependencies

None
