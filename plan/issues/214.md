# #214 -- Bug: member-expr-ident-name unicode escape failures (44 fails in assignment)

## Status: backlog

## Summary

44 test262 runtime failures in `language/expressions/assignment/member-expr-ident-name-*-escaped.js`. These tests use unicode escape sequences in property names which are not resolved during compilation.

## Motivation

All 44 failures follow the same pattern:
- Set a property using an identifier with unicode escape: `obj.bre\u0061k = 42`
- Read the property using bracket notation: `obj['break']`
- The compiler does not decode `\uXXXX` in identifier names, so the property name does not match

This is the single largest cluster of runtime failures. A single fix to decode unicode escapes in identifiers during property access would fix all 44 tests.

Related to #176 (unicode escape in property names).

## Scope
- `src/codegen/expressions.ts` -- property access identifier resolution
- Need to decode `\uXXXX` sequences in identifier names
