# Issue #214: String relational operators fail

**Status:** review
**Type:** bug
**Failures:** 12

## Problem
Relational operators (`<`, `>`, `<=`, `>=`) fail when comparing strings. In non-fast mode, the `string_compare` host import exists but is not provided in the test harness `buildImports`. In fast mode, `compileStringBinaryOp` does not handle relational operators at all (falls through to error).

## Fix
1. Add `string_compare` to the equivalence test harness `buildImports`
2. In fast mode: add an inline `__str_compare` Wasm helper for lexicographic string comparison
3. Handle relational ops in fast mode `compileStringBinaryOp`
