# #177 — Bug: Spread operator in `new` expressions

## Status: backlog

## Summary
`new Ctor(...args)` with spread operator fails at runtime. The spread elements are not correctly expanded into constructor arguments.

## Motivation
5 test262 failures in `language/expressions/new/spread-*`:
- `spread-sngl-literal.js` — `new Ctor(...[3, 4, 5])`
- `spread-sngl-empty.js` — `new Ctor(...[])`
- `spread-mult-literal.js` — `new Ctor(...[a], ...b)`
- `spread-mult-empty.js` — `new Ctor(...[], ...[])`
- `spread-obj-mult-spread.js` — spread with objects

These tests use `arguments` inside the constructor to verify the spread was applied correctly.

## Scope
- `src/codegen/expressions.ts` — NewExpression codegen when arguments contain SpreadElement

## Complexity
M
