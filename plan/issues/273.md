# Issue #273: Unsupported new expression for anonymous class expressions

## Status: review

## Summary
~31 tests fail with "Unsupported new expression for class: __class" as the sole error. These involve `new (class { ... })()` patterns where the class expression is anonymous. The codegen needs to support instantiating anonymous class expressions.

## Category
Sprint 4 / Group B

## Complexity: S

## Scope
- Handle `new (class { ... })()` where the class has no name binding
- Generate a synthetic class name for anonymous class expressions used in new
- Ensure constructor calls work on inline class expressions
- Update `compileNewExpression` in `src/codegen/expressions.ts`

## Implementation Notes

### Changes
1. **`src/codegen/index.ts`**: Added `classExprNodeMap: Map<ts.Node, string>` to `CodegenContext` for mapping ClassExpression AST nodes to their synthetic names. Added `collectInlineClassExpressionsFromNode()` in the collection phase to scan for `new (class { ... })()` patterns and register the inline class before variable types are resolved. Added `compileInlineClassExpressionsFromNode()` in the compilation phase to compile constructor/method bodies for these inline classes.

2. **`src/codegen/expressions.ts`**: Added ClassExpression handling in `compileNewExpression()` — when `new`'s callee unwraps to a ClassExpression, look up the synthetic name from `classExprNodeMap`, fall back to on-demand collection+compilation. Imported `collectClassDeclaration` and `compileClassBodies` from index.ts.

3. **`tests/equivalence.test.ts`**: Added 3 tests for anonymous class expression instantiation (with constructor, with constructor args, with no constructor/field initializer only).

### Key design decision
Classes must be collected during the collection phase (not just at expression-compilation time) so that `resolveWasmType` can map the TS type to the correct struct type when allocating the variable that holds the instance.

## Acceptance criteria
- Anonymous class expression instantiation compiles
- At least 20 compile errors resolved
