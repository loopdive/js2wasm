# #145 — allowJs type flexibility: boolean/string/void as number

## Status: open

## Problem
497 test262 compile errors from TypeScript type checking rejecting valid JS patterns:
- "Argument of type 'boolean' is not assignable to parameter of type 'number'" (277)
- "Argument of type 'string' is not assignable to parameter of type 'number'" (178)
- "Argument of type 'void' is not assignable to parameter of type 'number'" (42)

In allowJs mode, these coercions are valid JavaScript behavior.

## Fix
When compiling with `allowJs`, suppress strict type checking for implicit coercions that JavaScript allows. Emit appropriate coercion instructions:
- boolean -> number: `i32` already, just widen to `f64` if needed
- string -> number: call `parseFloat` or `Number()` host import
- void -> number: emit `f64.const NaN`
