### #1120 â€” int32 fast path for bitwise-coerced numeric loops

`src/codegen/function-body.ts`:
- `collectI32CoercedLocals(decl)` â€” fixpoint pass that detects `let`/`const` locals whose every write is provably int32-safe. Uses bounded depth guard `MAX_I32_SAFE_DEPTH = 64`. Disqualifies:
  - Names declared more than once in the same function (shadowing) â€” by-name candidate model breaks across scopes.
  - Names referenced from any nested function/arrow/method/accessor/constructor â€” closure capture struct fixes the type at construction time.
  - Names that appear as destructuring-assignment targets or for-of/for-in initializers.

`src/codegen/statements/variables.ts`: respect `i32CoercedLocals` when allocating slots â€” drops to i32 if the TS type is NumberLike.

`src/codegen/index.ts` (`walkStmtForLetConst`): same i32 promotion in the let/const hoisting pre-pass so the slot type matches what the codegen will emit.

`src/codegen/binary-ops.ts`:
- `expr | 0` fast path: returns `i32` directly when input is i32 (no `f64.convert_i32_s` round-trip). Callers coerce to f64 via existing `coerceType` paths if needed.
- `arithI32WithToInt32Wrap`: when an arithmetic expression `(a + b)` / `(a - b)` / `(a * b)` is the left operand of `expr | 0` AND both operands are i32 locals, route to the i32 dispatch path (`compileI32BinaryOp`). Walks past parens / `as` casts / non-null asserts when checking the parent.

`src/codegen/context/types.ts`: `i32CoercedLocals?: Set<string>` on FunctionContext, `numericReturnTypes?: Map<string, ValType>` on CodegenContext.

Result: iterative `benchmarks/competitive/programs/fib.js` hot loop collapses from a ~16-instruction f64+ToInt32 round-trip to `local.get a; local.get b; i32.add; local.set next; ...` (8 native i32 instructions). The 4294967296 modular-reduction constant is gone from the loop body.
