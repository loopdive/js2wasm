## Key decisions

- **Rewrote `restoreBuiltins()` from whitelist-based to descriptor-snapshot-based.** The old implementation enumerated ~200 specific named methods across 13 prototypes; any poison outside that set (e.g. prototype additions, accessor drift on non-RegExp prototypes, rare method deletions) leaked into subsequent compilations.
- **Kept hot-path value-assignment for data-property drift** (preserves V8 method ICs — the V8 performance regression that killed the original "#1153 attempt 1" via `defineProperty`). `Object.defineProperty` is only used for deletions (where assignment would wrongly set `enumerable:true`) and accessor drift (where assignment would hit a poisoned setter).
- **Made the restore itself iteration-safe.** My first draft used `for (const [k, v] of snapshot)` over a Map and `for (const key of currentKeys)` over an array — both trigger `Symbol.iterator`, which test262 routinely poisons on Array.prototype. My own smoke probe (scenario 9) caught this: the restore loop threw the poisoned `Error("poison")` before it could clean anything up. Fix: indexed `for (let i = 0; i < arr.length; i++)` loops everywhere in the restore path; stashed `Reflect.ownKeys`, `Object.getOwnPropertyDescriptor`, `Object.defineProperty`, `Object.is` at module load as module-scope constants so even poisoning of those APIs doesn't affect the restore.
- **Non-configurable drift triggers `process.exit(1)` with a clear `FATAL: non-configurable poison on <prototype>[<key>] — restarting worker` log** — matches the existing pool-restart contract for Array.prototype numeric-key and Object.prototype key poisoning.
