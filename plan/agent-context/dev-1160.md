### #1160 — Array.from prototype poisoning → MERGED (PR #7, +578 tests)

Root cause: **not a compiler bug**. `scripts/test262-worker.mjs#restoreBuiltins` used plain `=` assignment to restore `Array.prototype[Symbol.iterator]`. Silently no-ops on descriptors with `writable: false`. Tests that poisoned via `Object.defineProperty(..., { value: <non-fn>, writable: false })` left the prototype slot holding a non-function value, causing every subsequent test's compiler-internal `Array.from(nodeArray)` to throw V8's `%Array%.from requires that the property... be a function` — wrapped by the codegen try/catch into a fake `L1:0 Codegen error`.

Fix (`scripts/test262-worker.mjs`):
1. Capture original descriptors (not just values) for `Array.prototype[Symbol.iterator]` and every snapshotted method.
2. Two-stage restore: `=` first (hot path, no IC disturbance), `Object.defineProperty` fallback when the value is still wrong.
3. Early FATAL + `process.exit(1)` if Symbol.iterator can't be repaired (non-configurable poison) — BEFORE the rest of restoreBuiltins' `for..of` loops, which would otherwise crash first.
4. Pre-compile `restoreBuiltins()` in `doCompile` as defence-in-depth.

Test: `tests/issue-1160.test.ts` (2/2 pass). Observed variability across runs (816/461/731 `%Array%.from` failures) matched fork-pool sharding bleed.

### #1162 — yield* async / private-method-from-static crashes → MERGED (PR #11, +882 tests)

Three independent bugs compounded into the 161-test "unexpected undefined AST node" cluster:

1. **`tests/test262-runner.ts:renameYieldOutsideGenerators`** — regex `[\w$]+` for method names excluded `#`. Private generator methods like `static async *#gen()` weren't recognized as generators, so `yield` in body was renamed to `_yield`, producing `_yield* obj;` (multiplication) that crashed surrounding codegen. **Fix**: include `#` in the identifier class in both the has-generator pre-check and the per-match regex.

2. **`src/codegen/expressions/calls-closures.ts:compileGetterCallable`** — when a getter returns `this.#privateMethod` and the method is **static** (no `self` param), the code assumed instance-method param layout. `paramTypes.length - 1 = -1` for zero-arg statics → `expr.arguments[-1]` = undefined. **Fix**: detect `ctx.staticMethodSet.has(candidateName)`, evaluate-and-drop receiver for side effects, use `selfOffset = 0` when packing args.

3. **`src/codegen/expressions/calls.ts` Non-nullable receiver path** — defence-in-depth clamp of `methodParamCount = Math.max(0, paramTypes.length - 1)`. Exposed the symptom for `this.#f()` inside static methods where funcMap returned a stale index pointing at the zero-param constructor (underlying funcMap drift, not root-caused here).

Scripted replay of the 152 flagged tests: 0→148 pass after fix. Test: `tests/issue-1162.test.ts` (4/4 pass).
