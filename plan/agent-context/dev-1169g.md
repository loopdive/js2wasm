## Lessons / things to know

### Spread call leftover bug (legacy-only)

`f(...[1, 2, 3])` triggers a longstanding **legacy** bug — wasm validation fails with "not enough arguments on the stack for call" at instantiation time. My slice-8a IR widening sidesteps this by expanding spread at compile time. The spread-call cases in `tests/issue-1169g.test.ts` carry a `parity: false` flag for this reason — they assert the IR's correct value but skip the legacy parity check.

Worth filing a follow-up: legacy `compileSpreadCallArgs` emits the wrong arity. Probably easy fix once someone wants to chase it down.

### Baseline drift was a major distraction this PR

The CI regression check flagged 343 regressions, then narrowed to 238 non-timeout (218 compile_error + 20 fail). All 8 sampled regressions reproduced **identically on `main` HEAD** with IR off — pure baseline drift, the `js2wasm-baselines` snapshot was a day stale and ~5 merges behind main. Tech lead approved the merge after I confirmed via `git clone https://github.com/loopdive/js2wasm-baselines.git` and ran sample tests on plain main.

For future devs: when CI flags regressions, before assuming guilt, run **a small sample of the regressing tests on plain main HEAD**. If they fail identically there, it's drift. The baseline lives at `https://github.com/loopdive/js2wasm-baselines.git` (NOT in the main repo's `benchmarks/results/` — that's stale).

Tech lead said they'd refresh the baseline separately after merging this PR.

### Compile_timeout cluster is flake

108 pass→compile_timeout regressions and 159 compile_timeout→pass improvements in this run. Net **+51 fewer** compile_timeouts. Tests on the timeout boundary fluctuate with CI machine load. Treat compile_timeout-only deltas as noise unless the count is enormous AND clusters in one feature area.

### Boilerplate I had to learn

- The pre-commit hook needs `Checklist completed.` literal phrase or it blocks
- Biome's `--write` reformats to 80-char lines; project actually uses Prettier at 120 chars (`.prettierrc`). Use `npx prettier --write` not `npx biome check --write`. Biome only runs `lint` in lint-staged.
- The husky hook runs `prettier --write` then `biome lint --diagnostic-level=error`. Don't bypass.
- Test instantiation needs `instantiateWasm()` from `runtime.ts`, not bare `WebAssembly.instantiate` — the latter doesn't supply the `wasm:js-string` polyfill and string-returning tests spuriously fail.
