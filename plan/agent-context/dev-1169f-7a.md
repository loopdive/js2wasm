### Slice 7b (PR #73, admin-merged at HEAD `5e8ff0a8`, **ratio 10.76%**)

Extends slice 7a with non-numeric yields, bare `yield;`, `yield* <iterable>`, and bare `return;` in generator tails.

| Layer | Change |
|-------|--------|

| `src/ir/builder.ts` | `emitGenYieldStar(inner)` method |

| `src/ir/lower.ts` | Widened `gen.push` dispatch by IrType (f64→push_f64, i32→push_i32, ref→push_ref); `gen.yieldStar` emit case; null-externref const branch in `emitConst` (`ref.null.extern`); use-collection arm |
| `src/ir/select.ts` | Accept bare/star/non-numeric yields; thread `isGenerator` through `isPhase1StatementList` / `isPhase1Tail` for bare `return;` allowance |
| `src/ir/verify.ts` | Verifier use-collection arm |
| `src/ir/passes/{dead-code,inline-small,monomorphize}.ts` | Dispatch arms for `gen.yieldStar` (DCE pin, operand rewrite, use collection) |
| `tests/issue-1169f-7b.test.ts` | 5 dual-run cases × 2 (selector + equivalence) = 10 tests |

**Key 7b discoveries:**
1. **Wasm validation rule:** `extern.convert_any` requires the operand to be a subtype of `anyref`. **`externref` is NOT a subtype of `anyref`.** So coercing externref-typed values via `extern.convert_any` produces a Wasm validation error. Fix: skip the coerce for `IrType.val.externref` and `IrType.string` in host-strings mode (host strings flow as externref through wasm:js-string globals).
2. **Boolean yields surface as 1/0:** `__gen_push_i32(buf, 1|0)` pushes the integer as-is; consumers see numeric 1/0 in `IteratorResult.value`, NOT JS `true`/`false`. Eager-buffer-model quirk; matches legacy.
3. **`ref.null.extern` round-trips as JS `null`** (not `undefined`). Bare `yield;` produces `null` in `IteratorResult.value` via the wasm engine's externref-null marshalling.
4. **`monomorphize.ts:collectUses` exhaustive switch:** adding a new IrInstr variant breaks compilation. Must add an arm to the switch — typecheck catches it but I missed it on first push (CI quality job failed `TS2366`). Fix: explicit arm `case "gen.yieldStar": return [instr.inner];`.
5. **`__gen_yield_star` requires JS-iterable inner:** wasm vec values lack `Symbol.iterator` on the JS side, so `yield* <vec>` silently produces nothing. Test 4 uses another generator's externref output (which DOES have `Symbol.iterator` from `__create_generator`'s shape).

**What 7b enables for 8:**
- `gen.push` dispatch by IrType (f64 / i32 / ref) — slice 8's array rest `[a, ...rest]` accumulates into a buffer like a generator. The same `__gen_push_*` machinery can back rest collection (or a new `__rest_push_*` helper sharing the dispatch shape).
- `coerce.to_externref` helper + `coerceYieldValueToExternref` skip-rule — slice 8 will need similar coercion for spread-into-call (`f(...args)` where args is an iterable).
- `gen.yieldStar` IR node infrastructure — slice 8's `[...arr, ...arr2]` is structurally equivalent to chained yield* (drain N iterables into one buffer).

**Slice 8 scope highlights** (from spec):
- Object pattern: `const { a, b } = obj` — identifier-only, no computed keys
- Array pattern: `const [x, y] = arr` — accepts vec, falls back to iter for non-vec
- Rest in array: `const [a, ...rest] = arr` — uses vec-slice for vec, iter-collect for iter
- Spread in call: `f(...args)` — identifier-args only, no nested spread
- Spread in array literal: `[...a, ...b, c]` — chained drain into a fresh vec

**Out of scope:**
- Computed keys `{ [k]: x }`, default values `{ a = 0 }`, renames `{ a: b }`, nested patterns, async iteration spread, `{ ...rest }` (object rest)

**Architectural notes for the next agent:**
1. `lowerVarDecl` is the entry point. Today it accepts only `Identifier` LHS. Slice 8 widens to `ArrayBindingPattern` and `ObjectBindingPattern`.
2. New IR node likely needed: `pattern.match` or `destructure.{array,object}` with sub-instrs for each binding. OR: lower destructuring AT THE AST LAYER as a sequence of `vec.get` / `iter.next` / `object.get` + slot writes.
3. Rest collection: needs a way to allocate a fresh vec at runtime. Either a new IR node `vec.new(elemValType, capacity)` + `vec.push` (mutating), or pre-allocate via `Array.from(iter)` host helper.
4. Selector must handle pattern shape — recurse into binding elements; track which Phase-1 expression types are accepted as initializer (per slice 8 spec).
5. **CI baseline drift is real** — every PR will see 100-150 false-positive regressions. Use the algorithmic gate (`/dev-self-merge N`), and when it fails on ratio, do the same overlap analysis I did here (compare CI artifact JSONL against the immediately-prior PR's CI artifact + the `/tmp/fresh-baseline/` extracted from the latest emergency-baseline-refresh run). **Suggested staging within slice 8:**
