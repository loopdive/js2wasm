### Lessons from PR#149 / PR#158 regressions

When adding entries to a shared map (`ctx.externClasses`), check **all** call sites that iterate it. `collectUsedExternImports` was iterating the externClasses map to register host-side `__new_X` constructors. Adding String/Array there broke the Array constructor dispatch path. Split synthetic entries into a separate map.

When throwing in IR-lowering paths, ensure the throw doesn't fire for legitimate-but-unhandled cases — a single bad throw can corrupt the IR claim state and surface as `[object WebAssembly.Exception]` in completely unrelated tests. Prefer returning `null` (claim-failed sentinel) so the legacy path can take over.
