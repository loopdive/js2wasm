## Key learnings

1. **Silence protocol**: I was acknowledging team-lead's CI-status pings,
   which the team-lead correctly flagged as "idle messages" disguised as
   replies. Per protocol, devs in ci-wait should be **completely silent**
   until the wait-loop notification fires. Only the system delivers
   meaningful events; intermediate messages are informational.

2. **Background bash is correct**: Foreground Bash caps at 10 min, well
   under typical Test262 Sharded runtime. The `until ... do sleep 60; done`
   loop submitted with `run_in_background: true` is the right pattern.

3. **gh CLI quirk**: `gh pr checks <N>` does not support `--json`; use
   `gh pr view <N> --json statusCheckRollup --jq '.statusCheckRollup'`
   instead. My initial wait loop fired prematurely because the failing
   `gh` call returned empty stdout, jq errored silently, and `until`
   interpreted the error as truthy. Fixed loop pattern documented above.

4. **Per-call dedup pattern**: Both #1302 (global indices) and the
   suspect for #1314 (function indices) are missed-walk bugs in late-import
   shift logic. The `shifted: Set<Instr[]>` only dedupes top-level arrays;
   nested-via-recursion arrays reachable from multiple top-level paths
   need a per-instruction `WeakSet` for true dedup. Worth auditing the
   codebase for similar swap patterns that don't push to `savedBodies`.

5. **Issue scoping**: When an issue tagged "easy" turns out medium-or-up,
   message team-lead with the realistic scope before diving in. I saved
   significant time on #1323 by surfacing the design ambiguity (option A
   vs B) rather than implementing the wrong thing.

## Recommendations for future sessions

- Run `git worktree remove` for stale worktrees of merged PRs at session start.
- Use `gh pr view <N> --json statusCheckRollup` (not `gh pr checks --json`).
- For `[x = g()]`-style array destructure bugs, look at
  `destructureParamArray:730-739` first — manual `fctx.body =` swap pattern
  is a known late-import-shift hazard.
- The `addUnionImports` and similar add-late-import functions all need
  callers to pass `fctx` so the shift walker can reach all relevant body
  arrays. New code paths that emit calls to these imports must take care
  to keep `fctx.savedBodies` accurate.
