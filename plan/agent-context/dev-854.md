**Root cause**: `collectInstrs(fctx, emitFn)` swaps `fctx.body`, runs `emitFn` (which may recursively compile nested default-initializer thenInstrs), then restores and returns guardInstrs — now detached from any body the late-import shift walks (not in fctx.body, savedBodies, funcStack, mod.functions, pendingInitBody, or parentBodiesStack). Caller then called `buildDestructureNullThrow` which triggered `flushLateImportShifts`. The shift could not walk guardInstrs, so nested `call` funcIdx values stayed pre-shift — ended up pointing at the outer test function itself (infinite recursion → `RangeError: Maximum call stack size exceeded`).

**Fix**: Build `throwInstrs` and resolve `__extern_is_undefined` BEFORE `collectInstrs`. Every shift now fires while guardInstrs doesn't exist, so when emitFn later builds guardInstrs inside collectInstrs, the funcIdx values it embeds are already post-shift.
