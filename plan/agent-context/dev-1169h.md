## What landed

Two new declarative IR nodes mirroring the slice-6 forof.vec shape:

- **`IrInstrThrow`** — value-coerced-to-externref, lowers to Wasm
  `throw $__exn`.
- **`IrInstrTry`** — `body` / `catchClause.body` / `finallyBody` are
  self-contained `Instr[]` buffers; lowerer expands to structured
  Wasm `try`/`catch`/`catch_all`.

The lowerer:
- inlines `finallyBody` at try-body normal exit AND at synthesized
  `catch_all` (rethrow 0) for abrupt exit;
- wraps a source-level catch body in an inner `try`/`catch_all` when
  finally is present, so a throw inside catch still runs finally
  before propagating;
- writes externref payload to a pre-allocated slot at handler entry
  (or `drop` for ES2019 optional catch).

Shared `__exn` tag (signature `(externref)`) reused via new
`IrLowerResolver.ensureExnTag()` resolver method, so IR-compiled
throws are catchable by legacy-compiled handlers and vice versa.
Pre-registered in `preregisterExceptionSupport`.

## Out of scope (deferred to slice 9.5+)

- Destructuring catch param (`catch ({message})`) — depends on slice 8.
- Bare `throw;` (no expression) — rare, legacy handles.
- Numeric throws — would need `__box_number` host coerce.
- `return` / `break` / `continue` inside try / catch / finally bodies —
  needs the finally-stack inlining the issue spec describes; selector
  rejects body-position returns so functions fall back to legacy.
- **If-statements at body-position inside try / catch / finally bodies**
  — the body-buffer mechanism (`collectBodyInstrs`) doesn't yet
  support nested control flow because new blocks can't be opened
  inside a body buffer. Selector rejects, function falls back to
  legacy. Removing this restriction would require a new declarative
  `IrInstrIf` instr parallel to `IrInstrTry`, OR teaching the body
  buffer to inline structured `if`/`else` Wasm ops directly.
- `catchRethrowStack` rethrow optimisation.

## Lessons / pitfalls for future devs

1. **Body buffers are linear.** The slice-6 `collectBodyInstrs`
   pattern routes `pushInstr` to a side buffer but does NOT support
   `terminate` / `openBlock`. Any IR feature that needs conditional
   control flow inside a buffer needs either a new declarative IR
   instr (like `IrInstrTry`) OR raw-wasm `if`/`else` ops in the
   buffer.

2. **Exception tag is shared.** Re-using the legacy `__exn` tag means
   IR-compiled and legacy-compiled functions interoperate seamlessly.
   The `ensureExnTag(ctx)` is idempotent — both paths call it.

3. **Wasm catch_all + drop pattern.** Without a source catch but with
   a finally, the `catch_all` runs finally + `rethrow 0`. With BOTH a
   source catch AND a finally, the source catch's body must itself be
   wrapped in an inner try/catch_all so a throw inside catch runs
   finally before propagating. This is the legacy
   `compileTryStatement`'s `cloneFinallyAtDepth(1)` pattern reflected
   in IR-emit form.

4. **CI compile_timeout drift.** `compile_timeout` regressions are
   environmental drift, not real regressions. Project is excluding
   them via #1192/PR #77. When self-merge fails the bucket >50
   threshold solely because of `compile_timeout`, escalate with
   the cross-check evidence (other open PRs showing identical
   pattern + #1192 explicitly addressing it).

5. **Test pattern.** Tests use `let result: number = 0` slot bindings
   to flow values out of try/catch/finally bodies (no early
   returns). Throws are unconditional inside helper fns (no
   if-statements inside try). This minimal subset still exercises
   all the slice-9 emit paths.
