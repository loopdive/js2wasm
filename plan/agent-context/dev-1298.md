1. **Always grep for collisions before filing a new issue ID.** I picked

   alongside other 1335/1337/etc duplicates). The next free was 1353.
   Check `git ls-tree origin/main plan/issues/sprints/50/` against
   the `id:` you want to use.
2. **#1298 fix #3 had a 90-min false start.** The architect-specced
   eager-create + alt-funcref multi-candidate dispatch caused 340
   null_deref test262 regressions (Temporal cluster) on PR #223 v1
   because the rewrite removed the graceful `compile + drop +
   ref.null.extern` exit. The safe re-impl (commit b10a809ef) reverted
   to scan-only at the generic fallback. Then PR #231 re-implemented
   it as `ref.test`-guarded dispatch — this works because the cast
   only fires when ref.test confirms identity. Pattern to remember:
   any time an upstream codegen path commits to a wasm-closure
   dispatch chain, gate it on ref.test BEFORE the cast, with a
   graceful-null else branch. The architect spec will recommend the
   eager-create approach again — push back if so, the v1 lessons
   apply.
3. **Async/Promise codegen is asymmetric.** From #1313: js2wasm async
   closures unwrap their return type at declaration (raw `T` on stack)
   for the common path, BUT identifier-callees registered in
   `ctx.asyncFunctions` get `wrapAsyncReturn` applied, putting a real
   Promise on the stack. await is a passthrough. Fix #1313 (PR #246)
   skips the wrap when parent is `await`. The full bidirectional
   solution (universal Promise lowering with sync unwrap) needs JSPI
   — out of reach in standalone Wasm.
4. **#1239 (object literal accessors) needed 6 layers** — context
   field, hoist pre-pass, variables.ts type override, propertyAccess
   resolver hook, assignment threading, declarations.ts gate for
   `__make_getter_callback` import. Plus a narrowing fix (#69e410380)
   to skip computed-key accessors (`get [Symbol.iterator]()`) which
   were causing 37-test method-definition regressions when routed
   through the new path. Same pattern likely applies to other
   "object-literal-as-host-object" features.
5. **Always check the dev-self-merge gate vs CI conclusion.** PR #240
   first run had `regressions_real: 10, improvements: 41 = 24%`,
   exceeding the 10% threshold but with `max bucket = 1`. Tech lead
   approved physical-impossibility override given drift pattern
