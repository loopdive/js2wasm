## Open follow-up ‚Äî broken ADR links on GitHub Pages (REQUIRES ACTION)

The user reported (just before shutdown):

> `https://loopdive.github.io/js2wasm/docs/adr/0001-hybrid-compilation-strategy.md`
> produces a 404.

> we should also not directly link the .md file but instead a html view of it

**Root cause:** the landing page is served from GitHub Pages at
`loopdive.github.io/js2wasm/`. The `deploy-pages.yml` workflow uploads
only `dist/pages/`, which does NOT include `docs/adr/*.md`. So the
relative `./docs/adr/0NNN-*.md` links from `index.html` 404 in
production. They only "work" when viewing the repo on github.com.

**Recommended fix (next agent ‚Äî quick):** swap the relative `.md`
links for absolute GitHub blob URLs that GitHub renders as HTML:

  `./docs/adr/0001-hybrid-compilation-strategy.md`
  ‚Üí
  `https://github.com/loopdive/js2wasm/blob/main/docs/adr/0001-hybrid-compilation-strategy.md`

Same for the index link:

  `./docs/adr/README.md`
  ‚Üí
  `https://github.com/loopdive/js2wasm/blob/main/docs/adr/README.md`

GitHub's blob view renders Markdown to HTML, so the user gets a proper
HTML view (matching the user's "we should also not directly link the
md file but instead a html view of it" requirement).

**Better fix (separate issue):** add a build step to render
`docs/adr/*.md` ‚Üí `dist/pages/docs/adr/*.html` (via e.g. marked or
remark) and link to those instead. This keeps the URL on the same
origin as the landing page and removes the dependency on GitHub's
Markdown viewer. Suggest filing this as a follow-up issue.

**Affected file (single-file change):** `index.html`, lines ~3118-3133
in the Architecture subsection. About 13 hrefs to update.
