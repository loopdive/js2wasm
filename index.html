JavaScript - just more lightweight and secure.

js2wasm is an ahead-of-time compiler for JavaScript and TypeScript targeting WebAssembly GC. The project aims to keep source code idiomatic, keep modules small, and move execution into a capability-limited Wasm boundary instead of shipping a full JavaScript runtime with every artifact.

The broader goal is npm compatibility for Wasm hosts: taking existing JavaScript packages, compiling them into Wasm modules, and preserving a packaging and integration model developers already use.

Javascript compatible, but more secure and lightweight.

The project is not trying to invent a new source language. It is trying to make familiar JavaScript and TypeScript code viable in places where a JS engine is too heavy, too open, or too expensive to embed per module.

Compile, inspect, and benchmark the actual lowering.

js2wasm parses and checks JavaScript or TypeScript, lowers supported semantics into typed Wasm operations, and keeps host effects isolated at imports and exports. The playground is used as a real development tool for inspecting WAT, profiling codegen, and comparing Wasm against native JS execution.

;; DOM-oriented JS lowers to Wasm plus explicit host imports
(module
  (import "env" "global_document" (func $global_document (result externref)))
  (import "env" "Document_createElement" (func $create (param externref externref) (result externref)))
  (import "env" "Element_appendChild" (func $append (param externref externref) (result externref)))
  (import "string_constants" "div" (global $div externref))
  (func $main (export "main")
    call $global_document
    global.get $div
    call $create
    call $append
    drop))
