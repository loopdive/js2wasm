Javascript compiled to native WebAssembly, no runtime interpreter, no Typescript subset, the real thing. Run JavaScript in WebAssembly without embedding an engine.

The goal of this compiler project is to run Javascript and NPM packages without a runtime as lightweight and sandboxed WebAssembly GC modules.

Compiled ahead of time, no interpreter.

js2wasm parses and checks JavaScript or TypeScript, lowers supported semantics into typed Wasm operations, and keeps host effects isolated at imports and exports. The playground is used as a real development tool for inspecting WAT, profiling codegen, and comparing Wasm against native JS execution.

WAT inspect the exact lowered module
Bench compare Wasm against JS side by side

;; DOM-oriented JS lowers to Wasm plus explicit host imports
(module
  (import "env" "global_document" (func $global_document (result externref)))
  (import "env" "Document_createElement" (func $create (param externref externref) (result externref)))
  (import "env" "Element_appendChild" (func $append (param externref externref) (result externref)))
  (import "string_constants" "div" (global $div externref))
  (func $main (export "main")
    call $global_document
    global.get $div
    call $create
    call $append
    drop))

Safer with module sandboxing.

While JavaScript in the browser is running in a sandbox, dependencies are not isolated from each other and one foul module risks it all, as has been proven by recent supply chain attacks. In Node.js it is worse any program has full access to the file system an network.
