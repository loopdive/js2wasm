---
name: feedback_task_chain
description: Create a task chain for the full sprint sequence at session start — prevents premature stops and missed steps
type: feedback
---

Create top-level tasks for the full sprint pipeline at session start, not just per-dev-dispatch tasks.

**Why:** Without meta-level tasks, it's easy to prematurely signal "done" after one sprint instead of continuing to the next. The Ralph Loop or user expects continuous sprint execution.

**How to apply:** At session start, create a blocked task chain: finish current sprint → retrospective → next sprint → next sprint → etc. Mark each in_progress/completed as you go. Never exit without checking the task list for remaining work.
